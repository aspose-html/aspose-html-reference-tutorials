# Kör JavaScript i Java – Komplett guide för att lägga till element i body

Har du någonsin behövt **execute JavaScript in Java** men varit osäker på hur du ska förena de två världarna? Du är inte ensam. Många Java‑utvecklare stöter på problem när de vill utföra **JavaScript DOM manipulation** utan att starta en fullständig webbläsare. Den goda nyheten? Med Aspose.HTML kan du köra ett skript, **append element to body**, och spara den ändrade markup‑en — allt från ren Java‑kod.

I den här handledningen går vi igenom allt du behöver veta: ladda en HTML‑fil, skriva ett litet skript som **modifies HTML with JavaScript**, köra det skriptet från Java och slutligen spara ändringarna. I slutet har du ett självständigt, körbart exempel som du kan lägga in i vilket projekt som helst.

## Vad du behöver

- **Java 8** eller nyare (API:et fungerar med Java 8+)
- **Aspose.HTML for Java**‑bibliotek (version 22.12 eller senare)
- En enkel HTML‑mallfil (vi skapar en på flygande fot)
- En IDE eller textredigerare — inget avancerat, även en vanlig textredigerare räcker

> Proffstips: Om du använder Maven, lägg till Aspose.HTML‑beroendet i din `pom.xml`. Annars, släng bara JAR‑filerna i din classpath.

## Steg 1 – Ladda HTML‑dokumentet (Förbered för att **execute JavaScript in Java**)

Det första vi gör är att skapa en `HTMLDocument`‑instans som pekar på källfilen. Tänk på detta objekt som bron mellan Java och DOM‑trädet som skriptet senare kommer att manipulera.

```java
import com.aspose.html.HTMLDocument;

// Step 1: Load the HTML document you want to manipulate
HTMLDocument htmlDoc = new HTMLDocument("YOUR_DIRECTORY/template.html");
```

> **Varför detta är viktigt:** `HTMLDocument`‑klassen parsar filen till en DOM‑representation som Aspose‑skriptmotorn kan interagera med. Utan att ladda dokumentet först finns det inget för skriptet att ändra.

## Steg 2 – Skriv JavaScript som **Append Element to Body**

Nu skapar vi ett litet kodstycke som skapar ett `<p>`‑element och lägger till det i `<body>`. Detta är klassisk **JavaScript DOM manipulation**, men vi kör det från Java.

```java
// Step 2: JavaScript that adds a new paragraph to the body
String jsCode = ""
    + "var p = document.createElement('p');"
    + "p.textContent = 'Generated by JavaScript';"
    + "document.body.appendChild(p);";
```

> **Varför detta fungerar:** `document.createElement` skapar en ny nod, `textContent` sätter dess inre text, och `appendChild` fäster den till det befintliga `<body>`‑elementet. Skriptet är ren client‑side JavaScript; Aspose.HTML kör det i en headless‑miljö.

## Steg 3 – Skapa skriptmotorn (Motorn som **Run JavaScript from Java**)

Aspose.HTML levereras med en `ScriptEngine` som kan utvärdera JavaScript mot den laddade DOM‑en. Det är den komponent som faktiskt **run JavaScript from Java**.

```java
import com.aspose.html.scripting.ScriptEngine;

// Step 3: Bind the engine to the document we just loaded
ScriptEngine scriptEngine = new ScriptEngine(htmlDoc);
```

> **Varför du behöver detta:** Motorn isolerar skriptkörningen, så att den inte påverkar din huvud‑Java‑process. Den returnerar också ett `ScriptExecutionResult` så att du kan verifiera att det lyckades.

## Steg 4 – Kör skriptet och kontrollera resultatet

Med allt på plats kör vi slutligen skriptet. `execute`‑metoden returnerar ett resultatobjekt som talar om huruvida skriptet kördes utan fel.

```java
import com.aspose.html.scripting.ScriptExecutionResult;

// Step 4: Run the JavaScript and verify it succeeded
ScriptExecutionResult execResult = scriptEngine.execute(jsCode);
if (execResult.isSuccess()) {
    System.out.println("Script executed successfully.");
} else {
    System.err.println("Script failed: " + execResult.getErrorMessage());
}
```

> **Edge case:** Om skriptet kastar ett undantag (t.ex. ett syntaxfel) blir `isSuccess()` `false` och `getErrorMessage()` ger detaljerna. Kontrollera alltid detta i produktionskod.

## Steg 5 – Spara det modifierade dokumentet (Avsluta **Modify HTML with JavaScript**)

När skriptet har förändrat DOM‑en sparar vi ändringarna tillbaka till disk. Den resulterande filen innehåller nu det nya stycket.

```java
// Step 5: Write the updated DOM back to an HTML file
htmlDoc.save("YOUR_DIRECTORY/modified.html");
System.out.println("Document saved.");
```

> **Result verification:** Öppna `modified.html` i en webbläsare. Du bör se det ursprungliga innehållet plus en ny rad som säger “Generated by JavaScript”.

## Steg 6 – Fullständigt fungerande exempel (Alla steg kombinerade)

Nedan är det kompletta programmet som du kan kopiera‑och‑klistra in i en `JsDomManipulation.java`‑fil. Det innehåller alla imports, felhantering och kommentarer för tydlighet.

```java
import com.aspose.html.HTMLDocument;
import com.aspose.html.scripting.ScriptEngine;
import com.aspose.html.scripting.ScriptExecutionResult;

public class JsDomManipulation {
    public static void main(String[] args) throws Exception {

        // 1️⃣ Load the HTML document to be manipulated
        HTMLDocument htmlDoc = new HTMLDocument("YOUR_DIRECTORY/template.html");

        // 2️⃣ Define JavaScript that adds a new paragraph to the <body>
        String jsCode = "var p = document.createElement('p');"
                      + "p.textContent = 'Generated by JavaScript';"
                      + "document.body.appendChild(p);";

        // 3️⃣ Create a script engine bound to the loaded document
        ScriptEngine scriptEngine = new ScriptEngine(htmlDoc);

        // 4️⃣ Execute the JavaScript and report success
        ScriptExecutionResult execResult = scriptEngine.execute(jsCode);
        if (execResult.isSuccess()) {
            System.out.println("Script executed successfully.");
        } else {
            System.err.println("Script failed: " + execResult.getErrorMessage());
            return; // Stop if the script didn't run
        }

        // 5️⃣ Save the modified document back to HTML
        htmlDoc.save("YOUR_DIRECTORY/modified.html");
        System.out.println("Document saved.");
    }
}
```

### Förväntad output

Kör programmet skriver ut:

```
Script executed successfully.
Document saved.
```

Och `modified.html` innehåller nu något i stil med:

```html
<!DOCTYPE html>
<html>
<head>
    <title>Template</title>
</head>
<body>
    <h1>Original Content</h1>
    <p>Generated by JavaScript</p>
</body>
</html>
```

## Vanliga frågor & fallgropar

| Fråga | Svar |
|----------|--------|
| **Kan jag köra flera skript sekventiellt?** | Ja. Anropa bara `scriptEngine.execute()` igen med en ny skriptsträng. Samma `HTMLDocument`‑instans behåller sitt tillstånd. |
| **Vad händer om HTML‑filen saknar ett `<body>`‑tagg?** | Aspose.HTML kommer automatiskt att skapa ett `<body>`‑element under parsning, så skriptet fungerar ändå. Förutsägbara resultat får du dock genom att börja med en väl‑formad mall. |
| **Är detta tillvägagångssätt kompatibelt med Java 17?** | Absolut. Biblioteket riktar sig mot Java 8+, så nyare runtime‑miljöer fungerar bra. |
| **Behöver jag en fullständig webbläsarmiljö?** | Nej. Aspose.HTML erbjuder en lättviktig, headless‑motor som tolkar standard‑DOM‑API:er utan att starta Chrome eller Firefox. |
| **Hur hanterar jag stora HTML‑filer?** | Läs in dokumentet som en ström (`new HTMLDocument(InputStream)`) för att undvika att hela filen laddas in i minnet på en gång. |
