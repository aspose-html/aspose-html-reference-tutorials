# Thực thi JavaScript trong Java – Hướng dẫn đầy đủ về việc Thêm phần tử vào Body

Bạn đã bao giờ cần **thực thi JavaScript trong Java** nhưng không biết cách kết nối hai môi trường này? Bạn không phải là người duy nhất. Nhiều lập trình viên Java gặp khó khăn khi muốn thực hiện **JavaScript DOM manipulation** mà không khởi chạy một trình duyệt đầy đủ. Tin tốt là gì? Với Aspose.HTML, bạn có thể chạy một script, **thêm phần tử vào body**, và lưu lại markup đã thay đổi—tất cả chỉ bằng mã Java thuần túy.

Trong tutorial này, chúng ta sẽ đi qua mọi thứ bạn cần biết: tải một tệp HTML, viết một đoạn script nhỏ **sửa đổi HTML bằng JavaScript**, thực thi script đó từ Java, và cuối cùng lưu lại các thay đổi. Khi kết thúc, bạn sẽ có một ví dụ tự chứa, có thể chạy được mà bạn có thể đưa vào bất kỳ dự án nào.

## Những gì bạn cần

- **Java 8** trở lên (API hoạt động với Java 8+)
- Thư viện **Aspose.HTML for Java** (phiên bản 22.12 hoặc mới hơn)
- Một tệp mẫu HTML đơn giản (chúng ta sẽ tạo ngay trong quá trình)
- IDE hoặc trình soạn thảo văn bản—không cần gì quá phức tạp, thậm chí một trình soạn thảo văn bản thuần túy cũng đủ

> Mẹo chuyên nghiệp: Nếu bạn dùng Maven, hãy thêm phụ thuộc Aspose.HTML vào `pom.xml`. Nếu không, chỉ cần đặt các file JAR vào classpath của bạn.

## Bước 1 – Tải tài liệu HTML (Chuẩn bị thực thi JavaScript trong Java)

Điều đầu tiên chúng ta làm là tạo một thể hiện `HTMLDocument` trỏ tới tệp nguồn. Hãy nghĩ đối tượng này như một cầu nối giữa Java và cây DOM mà script sẽ thao tác sau này.

```java
import com.aspose.html.HTMLDocument;

// Step 1: Load the HTML document you want to manipulate
HTMLDocument htmlDoc = new HTMLDocument("YOUR_DIRECTORY/template.html");
```

> **Tại sao điều này quan trọng:** Lớp `HTMLDocument` phân tích tệp thành một biểu diễn DOM mà engine scripting của Aspose có thể tương tác. Nếu không tải tài liệu trước, script sẽ không có gì để sửa đổi.

## Bước 2 – Viết JavaScript **Thêm phần tử vào Body**

Bây giờ chúng ta tạo một đoạn mã ngắn tạo một phần tử `<p>` và thêm nó vào `<body>`. Đây là **JavaScript DOM manipulation** cổ điển, nhưng chúng ta sẽ chạy nó từ Java.

```java
// Step 2: JavaScript that adds a new paragraph to the body
String jsCode = ""
    + "var p = document.createElement('p');"
    + "p.textContent = 'Generated by JavaScript';"
    + "document.body.appendChild(p);";
```

> **Tại sao nó hoạt động:** `document.createElement` tạo một nút mới, `textContent` đặt nội dung văn bản bên trong, và `appendChild` gắn nó vào phần tử `<body>` hiện có. Script này hoàn toàn là JavaScript phía client; Aspose.HTML thực thi nó trong môi trường không giao diện (headless).

## Bước 3 – Tạo Script Engine (Engine **Chạy JavaScript từ Java**)

Aspose.HTML đi kèm với một `ScriptEngine` có thể đánh giá JavaScript trên DOM đã tải. Đây là thành phần thực sự **chạy JavaScript từ Java**.

```java
import com.aspose.html.scripting.ScriptEngine;

// Step 3: Bind the engine to the document we just loaded
ScriptEngine scriptEngine = new ScriptEngine(htmlDoc);
```

> **Tại sao bạn cần nó:** Engine cô lập việc thực thi script, ngăn nó ảnh hưởng tới tiến trình Java chính của bạn. Nó cũng trả về một `ScriptExecutionResult` để bạn có thể kiểm tra thành công hay không.

## Bước 4 – Thực thi Script và Kiểm tra Kết quả

Khi mọi thứ đã sẵn sàng, chúng ta cuối cùng mới chạy script. Phương thức `execute` trả về một đối tượng kết quả cho biết script có chạy mà không gặp lỗi hay không.

```java
import com.aspose.html.scripting.ScriptExecutionResult;

// Step 4: Run the JavaScript and verify it succeeded
ScriptExecutionResult execResult = scriptEngine.execute(jsCode);
if (execResult.isSuccess()) {
    System.out.println("Script executed successfully.");
} else {
    System.err.println("Script failed: " + execResult.getErrorMessage());
}
```

> **Trường hợp đặc biệt:** Nếu script ném ra ngoại lệ (ví dụ: lỗi cú pháp), `isSuccess()` sẽ trả về `false` và `getErrorMessage()` cung cấp chi tiết. Hãy luôn kiểm tra điều này trong mã production.

## Bước 5 – Lưu tài liệu đã sửa đổi (Hoàn thành **Sửa đổi HTML bằng JavaScript**)

Sau khi script đã thay đổi DOM, chúng ta ghi lại các thay đổi trở lại đĩa. Tệp kết quả bây giờ chứa đoạn đoạn văn mới.

```java
// Step 5: Write the updated DOM back to an HTML file
htmlDoc.save("YOUR_DIRECTORY/modified.html");
System.out.println("Document saved.");
```

> **Xác minh kết quả:** Mở `modified.html` bằng bất kỳ trình duyệt nào. Bạn sẽ thấy nội dung gốc cộng thêm một dòng mới có nội dung “Generated by JavaScript”.

## Bước 6 – Ví dụ Hoàn chỉnh (Tất cả các bước được kết hợp)

Dưới đây là chương trình đầy đủ mà bạn có thể sao chép‑dán vào file `JsDomManipulation.java`. Nó bao gồm tất cả các import, xử lý lỗi và chú thích để dễ hiểu.

```java
import com.aspose.html.HTMLDocument;
import com.aspose.html.scripting.ScriptEngine;
import com.aspose.html.scripting.ScriptExecutionResult;

public class JsDomManipulation {
    public static void main(String[] args) throws Exception {

        // 1️⃣ Load the HTML document to be manipulated
        HTMLDocument htmlDoc = new HTMLDocument("YOUR_DIRECTORY/template.html");

        // 2️⃣ Define JavaScript that adds a new paragraph to the <body>
        String jsCode = "var p = document.createElement('p');"
                      + "p.textContent = 'Generated by JavaScript';"
                      + "document.body.appendChild(p);";

        // 3️⃣ Create a script engine bound to the loaded document
        ScriptEngine scriptEngine = new ScriptEngine(htmlDoc);

        // 4️⃣ Execute the JavaScript and report success
        ScriptExecutionResult execResult = scriptEngine.execute(jsCode);
        if (execResult.isSuccess()) {
            System.out.println("Script executed successfully.");
        } else {
            System.err.println("Script failed: " + execResult.getErrorMessage());
            return; // Stop if the script didn't run
        }

        // 5️⃣ Save the modified document back to HTML
        htmlDoc.save("YOUR_DIRECTORY/modified.html");
        System.out.println("Document saved.");
    }
}
```

### Kết quả dự kiến

Chạy chương trình sẽ in ra:

```
Script executed successfully.
Document saved.
```

Và `modified.html` sẽ chứa nội dung tương tự:

```html
<!DOCTYPE html>
<html>
<head>
    <title>Template</title>
</head>
<body>
    <h1>Original Content</h1>
    <p>Generated by JavaScript</p>
</body>
</html>
```

## Câu hỏi thường gặp & Những lưu ý

| Câu hỏi | Trả lời |
|----------|--------|
| **Tôi có thể chạy nhiều script liên tiếp không?** | Có. Chỉ cần gọi lại `scriptEngine.execute()` với một chuỗi script mới. Cùng một thể hiện `HTMLDocument` sẽ giữ trạng thái hiện tại. |
| **Nếu tệp HTML thiếu thẻ `<body>` thì sao?** | Aspose.HTML sẽ tự động tạo một phần tử `<body>` trong quá trình phân tích, vì vậy script vẫn hoạt động. Tuy nhiên, để có kết quả dự đoán được, nên bắt đầu với một mẫu template chuẩn. |
| **Phương pháp này có tương thích với Java 17 không?** | Hoàn toàn có. Thư viện nhắm tới Java 8+, vì vậy các runtime mới hơn cũng hoạt động tốt. |
| **Tôi có cần môi trường trình duyệt đầy đủ không?** | Không. Aspose.HTML cung cấp một engine nhẹ, không giao diện, diễn giải các API DOM tiêu chuẩn mà không cần khởi chạy Chrome hay Firefox. |
| **Làm sao xử lý các tệp HTML lớn?** | Tải tài liệu dưới dạng stream (`new HTMLDocument(InputStream)`) để tránh việc nạp toàn bộ tệp vào bộ nhớ cùng một lúc. |

## Các bước tiếp theo

Bây giờ bạn đã biết cách **thực thi JavaScript trong Java**, bạn có thể khám phá các kỹ thuật **JavaScript DOM manipulation** nâng cao hơn:

- **Chèn CSS** để tạo kiểu cho các phần tử mới thêm.
- **Duyệt DOM** để sửa đổi các nút hiện có dựa trên thuộc tính.
- **Chạy script bên ngoài** được lấy từ một URL (sử dụng `scriptEngine.executeFromUrl()` nếu cần).
- Kết hợp kỹ thuật này với **chuyển đổi PDF**—Aspose.HTML có thể render DOM cuối cùng thành PDF chỉ bằng một lời gọi.

Nếu bạn quan tâm tới **chạy JavaScript từ Java** cho mục đích kiểm thử hoặc render phía server, hãy cân nhắc đóng gói logic này vào một lớp tiện ích và tái sử dụng trong các micro‑service.
