# Java で JavaScript を実行する – 要素を Body に追加する完全ガイド

**Java で JavaScript を実行**したいけど、どうやって両者をつなげるか分からないことはありませんか？ あなたは一人ではありません。多くの Java 開発者は、フルブラウザを起動せずに **JavaScript DOM 操作**を行いたいときに壁にぶつかります。朗報です！ Aspose.HTML を使えば、スクリプトを実行し、**要素を body に追加**し、変更されたマークアップを保存することが、純粋な Java コードだけで可能です。

このチュートリアルでは、HTML ファイルの読み込み、**JavaScript で HTML を変更**する小さなスクリプトの作成、Java からそのスクリプトを実行、そして最終的に変更を永続化する手順をすべて解説します。最後まで読めば、任意のプロジェクトに組み込める自己完結型の実行例が手に入ります。

## 必要なもの

- **Java 8** 以上（API は Java 8+ に対応）
- **Aspose.HTML for Java** ライブラリ（バージョン 22.12 以降）
- シンプルな HTML テンプレートファイル（このチュートリアルではその場で作成します）
- IDE またはテキストエディタ – 特別なものは不要、プレーンテキストエディタでも可

> Pro tip: Maven を使用している場合は、`pom.xml` に Aspose.HTML の依存関係を追加してください。そうでなければ、JAR をクラスパスにドロップするだけで構いません。

## Step 1 – HTML ドキュメントをロードする（Java で JavaScript を実行する準備）

最初に行うのは、ソースファイルを指す `HTMLDocument` インスタンスを作成することです。このオブジェクトは、Java と後でスクリプトが操作する DOM ツリーとの橋渡し役になります。

```java
import com.aspose.html.HTMLDocument;

// Step 1: Load the HTML document you want to manipulate
HTMLDocument htmlDoc = new HTMLDocument("YOUR_DIRECTORY/template.html");
```

> **Why this matters:** `HTMLDocument` クラスはファイルを DOM 表現にパースし、Aspose のスクリプトエンジンが操作できるようにします。ドキュメントをロードしなければ、スクリプトが変更できる対象が存在しません。

## Step 2 – **要素を Body に追加**する JavaScript を記述

ここでは、`<p>` 要素を作成し `<body>` に追加する小さなスニペットを作ります。これは典型的な **JavaScript DOM 操作** ですが、Java から実行します。

```java
// Step 2: JavaScript that adds a new paragraph to the body
String jsCode = ""
    + "var p = document.createElement('p');"
    + "p.textContent = 'Generated by JavaScript';"
    + "document.body.appendChild(p);";
```

> **Why this works:** `document.createElement` が新しいノードを生成し、`textContent` が内部テキストを設定、`appendChild` が既存の `<body>` 要素に結び付けます。スクリプトは純粋なクライアントサイド JavaScript で、Aspose.HTML がヘッドレス環境で実行します。

## Step 3 – スクリプトエンジンを作成（**Java から JavaScript を実行**するエンジン）

Aspose.HTML には、ロードした DOM に対して JavaScript を評価できる `ScriptEngine` が同梱されています。これが実際に **Java から JavaScript を実行**する部分です。

```java
import com.aspose.html.scripting.ScriptEngine;

// Step 3: Bind the engine to the document we just loaded
ScriptEngine scriptEngine = new ScriptEngine(htmlDoc);
```

> **Why you need this:** エンジンはスクリプト実行を分離し、メインの Java プロセスに影響を与えないようにします。また、`ScriptExecutionResult` を返すので成功可否を確認できます。

## Step 4 – スクリプトを実行し結果を確認

すべての準備が整ったら、いよいよスクリプトを実行します。`execute` メソッドは、エラーの有無を示す結果オブジェクトを返します。

```java
import com.aspose.html.scripting.ScriptExecutionResult;

// Step 4: Run the JavaScript and verify it succeeded
ScriptExecutionResult execResult = scriptEngine.execute(jsCode);
if (execResult.isSuccess()) {
    System.out.println("Script executed successfully.");
} else {
    System.err.println("Script failed: " + execResult.getErrorMessage());
}
```

> **Edge case:** スクリプトが例外（例: 文法エラー）を投げた場合、`isSuccess()` は `false` になり、`getErrorMessage()` が詳細を提供します。実運用コードでは必ずチェックしてください。

## Step 5 – 変更されたドキュメントを保存（**JavaScript で HTML を変更**する完了）

スクリプトが DOM を変形させた後、変更をディスクに永続化します。生成されたファイルには新しい段落が含まれます。

```java
// Step 5: Write the updated DOM back to an HTML file
htmlDoc.save("YOUR_DIRECTORY/modified.html");
System.out.println("Document saved.");
```

> **Result verification:** 任意のブラウザで `modified.html` を開いてください。元のコンテンツに加えて「Generated by JavaScript」という新しい行が表示されるはずです。

## Step 6 – 完全動作サンプル（全ステップ統合）

以下は `JsDomManipulation.java` ファイルにコピペできる完全プログラムです。インポート文、エラーハンドリング、コメントがすべて含まれています。

```java
import com.aspose.html.HTMLDocument;
import com.aspose.html.scripting.ScriptEngine;
import com.aspose.html.scripting.ScriptExecutionResult;

public class JsDomManipulation {
    public static void main(String[] args) throws Exception {

        // 1️⃣ Load the HTML document to be manipulated
        HTMLDocument htmlDoc = new HTMLDocument("YOUR_DIRECTORY/template.html");

        // 2️⃣ Define JavaScript that adds a new paragraph to the <body>
        String jsCode = "var p = document.createElement('p');"
                      + "p.textContent = 'Generated by JavaScript';"
                      + "document.body.appendChild(p);";

        // 3️⃣ Create a script engine bound to the loaded document
        ScriptEngine scriptEngine = new ScriptEngine(htmlDoc);

        // 4️⃣ Execute the JavaScript and report success
        ScriptExecutionResult execResult = scriptEngine.execute(jsCode);
        if (execResult.isSuccess()) {
            System.out.println("Script executed successfully.");
        } else {
            System.err.println("Script failed: " + execResult.getErrorMessage());
            return; // Stop if the script didn't run
        }

        // 5️⃣ Save the modified document back to HTML
        htmlDoc.save("YOUR_DIRECTORY/modified.html");
        System.out.println("Document saved.");
    }
}
```

### 期待される出力

プログラム実行時に次のように出力されます:

```
Script executed successfully.
Document saved.
```

そして `modified.html` の内容は概ね以下のようになります:

```html
<!DOCTYPE html>
<html>
<head>
    <title>Template</title>
</head>
<body>
    <h1>Original Content</h1>
    <p>Generated by JavaScript</p>
</body>
</html>
```

## よくある質問と落とし穴

| Question | Answer |
|----------|--------|
| **Can I run multiple scripts sequentially?** | Yes. Just call `scriptEngine.execute()` again with a new script string. The same `HTMLDocument` instance retains its state. |
| **What if the HTML file is missing a `<body>` tag?** | Aspose.HTML will auto‑create a `<body>` element during parsing, so the script still works. However, for predictable results, start with a well‑formed template. |
| **Is this approach compatible with Java 17?** | Absolutely. The library targets Java 8+, so newer runtimes are fine. |
| **Do I need a full browser environment?** | No. Aspose.HTML provides a lightweight, headless engine that interprets standard DOM APIs without launching Chrome or Firefox. |
| **How do I handle large HTML files?** | Load the document as a stream (`new HTMLDocument(InputStream)`) to avoid loading the entire file into memory at once. |
