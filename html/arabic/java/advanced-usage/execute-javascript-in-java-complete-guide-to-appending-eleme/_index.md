# تنفيذ JavaScript في Java – دليل كامل لإضافة عناصر إلى الـ Body

هل احتجت يوماً إلى **تنفيذ JavaScript في Java** لكن لم تكن متأكدًا من كيفية ربط العالمين؟ لست وحدك. يواجه العديد من مطوري Java صعوبة عندما يرغبون في إجراء **JavaScript DOM manipulation** دون تشغيل متصفح كامل. الخبر السار؟ مع Aspose.HTML يمكنك تشغيل سكريبت، **append element to body**، وحفظ العلامات المعدلة—كل ذلك من خلال كود Java نقي.

في هذا الدرس سنستعرض كل ما تحتاج معرفته: تحميل ملف HTML، كتابة سكريبت صغير يقوم **modifies HTML with JavaScript**، تنفيذ ذلك السكريبت من Java، وأخيرًا حفظ التغييرات. في النهاية ستحصل على مثال مستقل وقابل للتنفيذ يمكنك إدراجه في أي مشروع.

## ما ستحتاجه

- **Java 8** أو أحدث (API يعمل مع Java 8+)
- مكتبة **Aspose.HTML for Java** (الإصدار 22.12 أو أحدث)
- ملف قالب HTML بسيط (سننشئه في الوقت الفعلي)
- بيئة تطوير متكاملة أو محرر نصوص—لا شيء معقد، حتى محرر نصوص عادي يكفي

> نصيحة احترافية: إذا كنت تستخدم Maven، أضف تبعية Aspose.HTML إلى ملف `pom.xml`. وإلا، فقط ضع ملفات JAR في مسار الـ classpath الخاص بك.

## الخطوة 1 – تحميل مستند HTML (التحضير لتنفيذ JavaScript في Java)

أول ما نقوم به هو إنشاء كائن `HTMLDocument` يشير إلى ملف المصدر. فكر في هذا الكائن كجسر بين Java وشجرة DOM التي سيتعامل معها السكريبت لاحقًا.

```java
import com.aspose.html.HTMLDocument;

// Step 1: Load the HTML document you want to manipulate
HTMLDocument htmlDoc = new HTMLDocument("YOUR_DIRECTORY/template.html");
```

**لماذا هذا مهم:** تقوم فئة `HTMLDocument` بتحليل الملف إلى تمثيل DOM يمكن لمحرك سكريبت Aspose التفاعل معه. بدون تحميل المستند أولاً، لا يوجد شيء لتعديله السكريبت.

## الخطوة 2 – كتابة JavaScript التي **Append Element to Body**

الآن نقوم بإنشاء مقتطف صغير ينشئ عنصر `<p>` ويضيفه إلى `<body>`. هذا هو **JavaScript DOM manipulation** الكلاسيكي، لكننا سننفذه من Java.

```java
// Step 2: JavaScript that adds a new paragraph to the body
String jsCode = ""
    + "var p = document.createElement('p');"
    + "p.textContent = 'Generated by JavaScript';"
    + "document.body.appendChild(p);";
```

**لماذا هذا يعمل:** `document.createElement` ينشئ عقدة جديدة، `textContent` يحدد نصها الداخلي، و`appendChild` يرفقها بعنصر `<body>` الموجود. السكريبت هو JavaScript خالص من جانب العميل؛ Aspose.HTML ينفذه في بيئة headless.

## الخطوة 3 – إنشاء محرك السكريبت (المحرك الذي **Run JavaScript from Java**)

تأتي Aspose.HTML مع `ScriptEngine` يمكنه تقييم JavaScript على الـ DOM المحمل. هو الجزء الذي **run JavaScript from Java** فعليًا.

```java
import com.aspose.html.scripting.ScriptEngine;

// Step 3: Bind the engine to the document we just loaded
ScriptEngine scriptEngine = new ScriptEngine(htmlDoc);
```

**لماذا تحتاج هذا:** يعزل المحرك تنفيذ السكريبت، مما يمنع تأثيره على عملية Java الرئيسية. كما أنه يُعيد `ScriptExecutionResult` لتتمكن من التحقق من النجاح.

## الخطوة 4 – تنفيذ السكريبت والتحقق من النتيجة

بعد إعداد كل شيء، نطلق السكريبت أخيرًا. طريقة `execute` تُعيد كائن نتيجة يُخبرنا ما إذا كان السكريبت قد تم تشغيله دون أخطاء.

```java
import com.aspose.html.scripting.ScriptExecutionResult;

// Step 4: Run the JavaScript and verify it succeeded
ScriptExecutionResult execResult = scriptEngine.execute(jsCode);
if (execResult.isSuccess()) {
    System.out.println("Script executed successfully.");
} else {
    System.err.println("Script failed: " + execResult.getErrorMessage());
}
```

**حالة حافة:** إذا ألقى السكريبت استثناءً (مثل خطأ في الصياغة)، فإن `isSuccess()` سيُعيد `false` و`getErrorMessage()` سيُظهر التفاصيل. تأكد دائمًا من فحص ذلك في كود الإنتاج.

## الخطوة 5 – حفظ المستند المعدل (إنهاء **Modify HTML with JavaScript**)

بعد أن عدل السكريبت الـ DOM، نقوم بحفظ التغييرات إلى القرص. الملف الناتج الآن يحتوي على الفقرة الجديدة.

```java
// Step 5: Write the updated DOM back to an HTML file
htmlDoc.save("YOUR_DIRECTORY/modified.html");
System.out.println("Document saved.");
```

**التحقق من النتيجة:** افتح `modified.html` في أي متصفح. يجب أن ترى المحتوى الأصلي بالإضافة إلى سطر جديد يقرأ “Generated by JavaScript”.

## الخطوة 6 – مثال كامل يعمل (جميع الخطوات مجتمعة)

فيما يلي البرنامج الكامل الذي يمكنك نسخه‑ولصقه في ملف `JsDomManipulation.java`. يتضمن جميع الاستيرادات، معالجة الأخطاء، وتعليقات للتوضيح.

```java
import com.aspose.html.HTMLDocument;
import com.aspose.html.scripting.ScriptEngine;
import com.aspose.html.scripting.ScriptExecutionResult;

public class JsDomManipulation {
    public static void main(String[] args) throws Exception {

        // 1️⃣ Load the HTML document to be manipulated
        HTMLDocument htmlDoc = new HTMLDocument("YOUR_DIRECTORY/template.html");

        // 2️⃣ Define JavaScript that adds a new paragraph to the <body>
        String jsCode = "var p = document.createElement('p');"
                      + "p.textContent = 'Generated by JavaScript';"
                      + "document.body.appendChild(p);";

        // 3️⃣ Create a script engine bound to the loaded document
        ScriptEngine scriptEngine = new ScriptEngine(htmlDoc);

        // 4️⃣ Execute the JavaScript and report success
        ScriptExecutionResult execResult = scriptEngine.execute(jsCode);
        if (execResult.isSuccess()) {
            System.out.println("Script executed successfully.");
        } else {
            System.err.println("Script failed: " + execResult.getErrorMessage());
            return; // Stop if the script didn't run
        }

        // 5️⃣ Save the modified document back to HTML
        htmlDoc.save("YOUR_DIRECTORY/modified.html");
        System.out.println("Document saved.");
    }
}
```

### النتيجة المتوقعة

تشغيل البرنامج يطبع:

```
Script executed successfully.
Document saved.
```

و`modified.html` الآن يحتوي على شيء مشابه لـ:

```html
<!DOCTYPE html>
<html>
<head>
    <title>Template</title>
</head>
<body>
    <h1>Original Content</h1>
    <p>Generated by JavaScript</p>
</body>
</html>
```

## أسئلة شائعة وملاحظات

| السؤال | الإجابة |
|----------|--------|
| **هل يمكنني تشغيل عدة سكريبتات متتالية؟** | نعم. فقط استدعِ `scriptEngine.execute()` مرة أخرى مع سلسلة سكريبت جديدة. تظل حالة كائن `HTMLDocument` نفسها محفوظة. |
| **ماذا لو كان ملف HTML يفتقد وسم `<body>`؟** | ستقوم Aspose.HTML بإنشاء عنصر `<body>` تلقائيًا أثناء التحليل، لذا سيستمر السكريبت في العمل. ومع ذلك، للحصول على نتائج متوقعة، ابدأ بقالب مُشكل بشكل صحيح. |
| **هل هذا النهج متوافق مع Java 17؟** | بالطبع. تستهدف المكتبة Java 8+، لذا فإن بيئات التشغيل الأحدث لا مشكلة فيها. |
| **هل أحتاج إلى بيئة متصفح كاملة؟** | لا. توفر Aspose.HTML محركًا خفيفًا بدون رأس (headless) يفسر واجهات برمجة تطبيقات DOM القياسية دون تشغيل Chrome أو Firefox. |
| **كيف أتعامل مع ملفات HTML الكبيرة؟** | حمّل المستند كتيار (`new HTMLDocument(InputStream)`) لتجنب تحميل الملف بالكامل إلى الذاكرة مرة واحدة. |

## الخطوات التالية

الآن بعد أن عرفت كيفية **execute JavaScript in Java**، يمكنك استكشاف **JavaScript DOM manipulation** المتقدمة:

- **Inject CSS** لتنسيق العناصر المضافة حديثًا.
- **Traverse the DOM** لتعديل العقد الموجودة بناءً على السمات.
- **Run external scripts** التي تُجلب من URL (استخدم `scriptEngine.executeFromUrl()` إذا لزم الأمر).
- دمج هذه التقنية مع **PDF conversion**—يمكن لـ Aspose.HTML تحويل الـ DOM النهائي إلى PDF في استدعاء واحد.

إذا كنت مهتمًا بـ **run JavaScript from Java** للاختبار أو العرض من جانب الخادم، فكر في تغليف المنطق داخل فئة مساعدة وإعادة استخدامها عبر الخدمات المصغرة.
