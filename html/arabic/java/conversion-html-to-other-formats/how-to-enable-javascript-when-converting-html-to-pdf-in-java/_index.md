# كيفية تمكين جافاسكريبت عند تحويل HTML إلى PDF في جافا

هل تساءلت يومًا **كيف يتم تمكين جافاسكريبت** أثناء تحويل HTML إلى PDF؟ ربما حاولت عرض لوحة تحكم، لكن المخططات ظلت فارغة لأن سكريبتات الصفحة لم تُنفّذ أبدًا. هذه مشكلة شائعة—جافاسكريبت يكون معطَّلًا بشكل افتراضي لأسباب أمنية، لذا يُفقد المحتوى الديناميكي.  

في هذا الدرس سنستعرض **كيفية تمكين جافاسكريبت** باستخدام Aspose.HTML for Java، ونوضح **كيفية ضبط مهلة التنفيذ**، وأخيرًا **تحويل html إلى pdf** بصفحة مُ rendered بالكامل. في النهاية ستحصل على مثال جاهز للتنفيذ يحول ملف `.html` ديناميكي إلى PDF مصقول، بالإضافة إلى مجموعة من النصائح لتجنب المشكلات المعتادة.

## المتطلبات المسبقة

- Java 17 (أو أي JDK حديث) مُثبت ومُعد.
- Maven أو Gradle لجلب مكتبة Aspose.HTML for Java.
- ملف HTML بسيط (`dynamic.html`) يحتوي على جافاسكريبت (مثلاً مكتبة مخططات أو سكريبت يُعدِّل الـ DOM).
- إلمام أساسي بصياغة جافا—لا حاجة لمعرفة متقدمة.

> **نصيحة احترافية:** إذا كنت تستخدم بيئة تطوير مثل IntelliJ IDEA، فعّل “auto‑import” لتسمح للمحرر بإضافة عبارات `import` تلقائيًا.

## الخطوة 1 – كيفية تمكين جافاسكريبت في HtmlLoadOptions

أول شيء تحتاج معرفته **كيفية تمكين جافاسكريبت** هو إخبار المحمل بأن السكريبتات مسموح بها. Aspose.HTML يأتي مع `HtmlLoadOptions`، الذي يُعطِّل جافاسكريبت افتراضيًا للسلامة. قم بتبديل الإعداد هكذا:

```java
import com.aspose.html.*;
import com.aspose.html.rendering.*;
import com.aspose.html.converters.*;

public class JsEnabledConversion {
    public static void main(String[] args) throws Exception {

        // Create load options and enable JavaScript execution
        HtmlLoadOptions loadOptions = new HtmlLoadOptions();
        loadOptions.setEnableJavaScript(true);   // <-- this line enables JavaScript
```

لماذا هذا السطر مهم؟ بدونّه يعتبر المحرك كل وسم `<script>` غير فعال، ما يعني أن أي تغييرات في الـ DOM أو طلبات AJAX أو رسم على الـ canvas لن تحدث. تمكين جافاسكريبت يمنح المحول بيئة متصفح صغيرة حيث يمكن للسكريبت أن يعمل كما في Chrome.

## الخطوة 2 – كيفية ضبط مهلة السكريبت لتحويلات موثوقة

الآن بعد أن تم توضيح **كيفية تمكين جافاسكريبت**، ربما تتساءل: “ماذا لو استمر السكريبت إلى الأبد؟” هنا يأتي دور **كيفية ضبط المهلة**. Aspose.HTML يتيح لك تحديد حد أقصى لوقت تنفيذ السكريبت بالميلي ثانية:

```java
        // Define how long scripts may run (milliseconds)
        loadOptions.setScriptTimeout(5000); // 5 seconds is usually enough
```

ضبط المهلة يمنع المحول من التوقف بسبب سكريبتات سيئة الكتابة أو خبيثة. إذا انتهت المهلة، يقوم Aspose بإلغاء السكريبت ومتابعة رسم الصفحة كما هي. يمكنك تعديل القيمة بناءً على تعقيد صفحتك—المخططات الكبيرة قد تحتاج إلى 10 000 ms، بينما النماذج البسيطة تكفيها 2 000 ms.

## الخطوة 3 – تحويل HTML إلى PDF باستخدام الخيارات المُكوَّنة

مع **كيفية تمكين جافاسكريبت** و**كيفية ضبط المهلة** تم الانتهاء، حان الوقت فعليًا **لتحويل html إلى pdf**. طريقة `Converter.convertDocument` تقوم بالعمل الشاق:

```java
        // Convert the HTML page (with JavaScript) to PDF using the configured options
        Converter.convertDocument(
                "YOUR_DIRECTORY/dynamic.html",   // source HTML containing JavaScript
                "YOUR_DIRECTORY/dynamic.pdf",    // destination PDF file
                new PdfSaveOptions(),
                loadOptions);                    // <-- passes the JS‑enabled options

        System.out.println("JS‑enabled PDF created.");
    }
}
```

عند تشغيل البرنامج، يقوم Aspose بتحميل `dynamic.html`، تنفيذ جافاسكريبت (بفضل `setEnableJavaScript(true)` السابق)، احترام مهلة السكريبت التي حُدِّدت بـ 5 ثوانٍ، وأخيرًا كتابة `dynamic.pdf`. افتح ملف PDF—you should see the chart, table, or any other dynamic element that was originally generated by JavaScript.

### النتيجة المتوقعة

```text
JS‑enabled PDF created.
```

وسيحتوي `dynamic.pdf` الناتج على صفحة مُ rendered بالكامل، مع كل المحتوى الذي يولده السكريبت مرئيًا.

## الاختلافات الشائعة وحالات الحافة

### 1. تحويل عدة صفحات دفعة واحدة
إذا احتجت **لتحويل html إلى pdf** لمجموعة من الملفات، ما عليك سوى تكرار الحلقة على مسارات المصدر وإعادة استخدام نفس كائن `HtmlLoadOptions`. هذا يُجنبك تكلفة إنشاء الخيارات في كل مرة.

### 2. التعامل مع الصفحات التي تعتمد على AJAX
للصفحات التي تجلب بيانات عبر AJAX، قد تحتاج إلى زيادة **مهلة السكريبت** أو توفير `NetworkRequestHandler` مخصص لتقليد استجابات API. وإلا قد ينتهي التحويل قبل وصول البيانات، تاركًا أماكن احتياطية في الـ PDF.

### 3. اعتبارات الأمان
تمكين جافاسكريبت يفتح سطح هجوم صغير. احرص دائمًا على التحقق أو عزل الـ HTML الذي تُدخله إلى المحول، خاصة إذا كان المصدر من مستخدمين غير موثوقين. ضبط **مهلة السكريبت** بشكل معقول هو الخط الدفاعي الأول.

### 4. تغيير صيغ الإخراج
Aspose.HTML لا يقتصر على PDF فقط. يمكنك استبدال `new PdfSaveOptions()` بـ `new PngDevice()` أو `new JpegDevice()` للحصول على صور نقطية، أو حتى `new XpsSaveOptions()` لملفات XPS. خطوات **كيفية تمكين جافاسكريبت** و**كيفية ضبط المهلة** تبقى نفسها.

## ملخص خطوة‑بخطوة (مرجع سريع)

| الخطوة | ما تقوم به | سطر الكود الرئيسي |
|--------|------------|-------------------|
| 1 | إنشاء `HtmlLoadOptions` وتفعيل جافاسكريبت | `loadOptions.setEnableJavaScript(true);` |
| 2 | تحديد حد أقصى لتنفيذ السكريبت | `loadOptions.setScriptTimeout(5000);` |
| 3 | استدعاء `Converter.convertDocument` مع `PdfSaveOptions` | `Converter.convertDocument(..., new PdfSaveOptions(), loadOptions);` |

## نصائح احترافية لتجربة سلسة

- **قم بتخزين الخيارات** إذا كنت تحول ملفات عديدة؛ هذا يقلل من ضغط الـ GC.
- **سجِّل زمن التحويل** لتحديد الصفحات التي تتجاوز المهلة باستمرار—قد تحتاج إلى تحسينها.
- **اختبر باستخدام متصفحات headless** (مثل Chrome Headless) أولًا لتقدير مدة تشغيل السكريبتات؛ ثم عكس تلك المدة في `setScriptTimeout`.
- **استخدم مسارات مطلقة** أو موارد على class‑path لـ `dynamic.html` لتجنب مفاجآت “الملف غير موجود” عند تشغيل الـ JAR من دليل مختلف.

## الأسئلة المتكررة

**س: هل يعمل هذا مع Java 11؟**  
ج: نعم. Aspose.HTML يدعم Java 8 وما فوق، لذا Java 11 مناسب. فقط تأكد من أن نسخة المكتبة تتوافق مع نسخة JDK الخاصة بك.

**س: ماذا لو أردت تعطيل جافاسكريبت لصفحة معينة؟**  
ج: أنشئ كائن `HtmlLoadOptions` منفصل دون استدعاء `setEnableJavaScript(true)`. مرّر هذا الكائن إلى `Converter.convertDocument` للصفحات الآمنة.

**س: هل يمكن تغيير المهلة لكل صفحة؟**  
ج: بالتأكيد. فقط عدّل `loadOptions.setScriptTimeout(...)` قبل كل استدعاء للتحويل.

## الخلاصة

غطّينا **كيفية تمكين جافاسكريبت** في Aspose.HTML for Java، وأظهرنا **كيفية ضبط المهلة**، وتناولنا سير عمل كامل لـ **تحويل html إلى pdf**. عبر تفعيل `setEnableJavaScript(true)` وضبط `setScriptTimeout` بدقة، ستحصل على مخرجات PDF موثوقة حتى من أكثر الصفحات ويب ديناميكية.

هل أنت مستعد للخطوة التالية؟ جرّب التحويل إلى صيغ أخرى، جرب قيم مهلة مختلفة، أو دمج هذا المقتطف في خدمة مصغرة تُنشئ تقارير عند الطلب. السماء هي الحد عندما تتحكم في تنفيذ جافاسكريبت والرندرة.
