# Executar JavaScript em Java – Guia Completo para Anexar Elementos ao Body

Já precisou **executar JavaScript em Java** mas não sabia como conectar os dois mundos? Você não está sozinho. Muitos desenvolvedores Java encontram um obstáculo quando querem fazer **manipulação de DOM com JavaScript** sem abrir um navegador completo. A boa notícia? Com Aspose.HTML você pode rodar um script, **anexar elemento ao body**, e salvar o markup alterado — tudo a partir de código Java puro.

Neste tutorial vamos percorrer tudo o que você precisa saber: carregar um arquivo HTML, escrever um pequeno script que **modifica HTML com JavaScript**, executar esse script a partir de Java e, finalmente, persistir as alterações. Ao final você terá um exemplo autocontido e executável que pode ser inserido em qualquer projeto.

## O que você vai precisar

- **Java 8** ou superior (a API funciona com Java 8+)
- Biblioteca **Aspose.HTML for Java** (versão 22.12 ou posterior)
- Um arquivo de modelo HTML simples (vamos criar um na hora)
- Uma IDE ou editor de texto — nada sofisticado, até um editor de texto simples serve

> Dica de especialista: se você usa Maven, adicione a dependência Aspose.HTML ao seu `pom.xml`. Caso contrário, basta colocar os JARs no seu classpath.

## Etapa 1 – Carregar o Documento HTML (Prepare-se para Executar JavaScript em Java)

A primeira coisa que fazemos é criar uma instância de `HTMLDocument` que aponta para o arquivo fonte. Pense neste objeto como a ponte entre Java e a árvore DOM que o script tocará mais tarde.

```java
import com.aspose.html.HTMLDocument;

// Step 1: Load the HTML document you want to manipulate
HTMLDocument htmlDoc = new HTMLDocument("YOUR_DIRECTORY/template.html");
```

> **Por que isso importa:** A classe `HTMLDocument` analisa o arquivo em uma representação DOM que o motor de script da Aspose pode interagir. Sem carregar o documento primeiro, não há nada para o script modificar.

## Etapa 2 – Escrever JavaScript que **Anexa Elemento ao Body**

Agora criamos um pequeno trecho que cria um elemento `<p>` e o adiciona ao `<body>`. Isso é a clássica **manipulação de DOM com JavaScript**, mas vamos executá‑lo a partir de Java.

```java
// Step 2: JavaScript that adds a new paragraph to the body
String jsCode = ""
    + "var p = document.createElement('p');"
    + "p.textContent = 'Generated by JavaScript';"
    + "document.body.appendChild(p);";
```

> **Por que isso funciona:** `document.createElement` cria um novo nó, `textContent` define seu texto interno, e `appendChild` o anexa ao elemento `<body>` existente. O script é JavaScript puro do lado do cliente; Aspose.HTML o executa em um ambiente sem cabeça.

## Etapa 3 – Criar o Motor de Script (O Motor que **Executa JavaScript a partir de Java**)

Aspose.HTML vem com um `ScriptEngine` que pode avaliar JavaScript contra o DOM carregado. É a peça que realmente **executa JavaScript a partir de Java**.

```java
import com.aspose.html.scripting.ScriptEngine;

// Step 3: Bind the engine to the document we just loaded
ScriptEngine scriptEngine = new ScriptEngine(htmlDoc);
```

> **Por que você precisa disso:** O motor isola a execução do script, impedindo que ele afete seu processo Java principal. Ele também devolve um `ScriptExecutionResult` para que você possa verificar o sucesso.

## Etapa 4 – Executar o Script e Verificar o Resultado

Com tudo configurado, finalmente dispararmos o script. O método `execute` devolve um objeto de resultado que indica se o script foi executado sem erros.

```java
import com.aspose.html.scripting.ScriptExecutionResult;

// Step 4: Run the JavaScript and verify it succeeded
ScriptExecutionResult execResult = scriptEngine.execute(jsCode);
if (execResult.isSuccess()) {
    System.out.println("Script executed successfully.");
} else {
    System.err.println("Script failed: " + execResult.getErrorMessage());
}
```

> **Caso extremo:** Se o script lançar uma exceção (por exemplo, um erro de sintaxe), `isSuccess()` será `false` e `getErrorMessage()` fornecerá os detalhes. Sempre verifique isso em código de produção.

## Etapa 5 – Salvar o Documento Modificado (Concluir **Modificação de HTML com JavaScript**)

Depois que o script alterou o DOM, persistimos as mudanças de volta ao disco. O arquivo resultante agora contém o novo parágrafo.

```java
// Step 5: Write the updated DOM back to an HTML file
htmlDoc.save("YOUR_DIRECTORY/modified.html");
System.out.println("Document saved.");
```

> **Verificação do resultado:** Abra `modified.html` em qualquer navegador. Você deverá ver o conteúdo original mais uma nova linha que diz “Generated by JavaScript”.

## Etapa 6 – Exemplo Completo (Todas as Etapas Combinadas)

Abaixo está o programa completo que você pode copiar‑colar em um arquivo `JsDomManipulation.java`. Ele inclui todas as importações, tratamento de erros e comentários para clareza.

```java
import com.aspose.html.HTMLDocument;
import com.aspose.html.scripting.ScriptEngine;
import com.aspose.html.scripting.ScriptExecutionResult;

public class JsDomManipulation {
    public static void main(String[] args) throws Exception {

        // 1️⃣ Load the HTML document to be manipulated
        HTMLDocument htmlDoc = new HTMLDocument("YOUR_DIRECTORY/template.html");

        // 2️⃣ Define JavaScript that adds a new paragraph to the <body>
        String jsCode = "var p = document.createElement('p');"
                      + "p.textContent = 'Generated by JavaScript';"
                      + "document.body.appendChild(p);";

        // 3️⃣ Create a script engine bound to the loaded document
        ScriptEngine scriptEngine = new ScriptEngine(htmlDoc);

        // 4️⃣ Execute the JavaScript and report success
        ScriptExecutionResult execResult = scriptEngine.execute(jsCode);
        if (execResult.isSuccess()) {
            System.out.println("Script executed successfully.");
        } else {
            System.err.println("Script failed: " + execResult.getErrorMessage());
            return; // Stop if the script didn't run
        }

        // 5️⃣ Save the modified document back to HTML
        htmlDoc.save("YOUR_DIRECTORY/modified.html");
        System.out.println("Document saved.");
    }
}
```

### Saída Esperada

Executar o programa imprime:

```
Script executed successfully.
Document saved.
```

E `modified.html` agora contém algo como:

```html
<!DOCTYPE html>
<html>
<head>
    <title>Template</title>
</head>
<body>
    <h1>Original Content</h1>
    <p>Generated by JavaScript</p>
</body>
</html>
```

## Perguntas Frequentes & Armadilhas

| Pergunta | Resposta |
|----------|----------|
| **Posso executar vários scripts sequencialmente?** | Sim. Basta chamar `scriptEngine.execute()` novamente com uma nova string de script. A mesma instância de `HTMLDocument` mantém seu estado. |
| **E se o arquivo HTML não contiver a tag `<body>`?** | Aspose.HTML criará automaticamente um elemento `<body>` durante a análise, então o script ainda funciona. Contudo, para resultados previsíveis, comece com um modelo bem‑formado. |
| **Esta abordagem é compatível com Java 17?** | Absolutamente. A biblioteca tem como alvo Java 8+, então runtimes mais recentes funcionam sem problemas. |
| **Preciso de um ambiente de navegador completo?** | Não. Aspose.HTML fornece um motor leve e sem cabeça que interpreta as APIs padrão de DOM sem lançar Chrome ou Firefox. |
| **Como lidar com arquivos HTML grandes?** | Carregue o documento como um stream (`new HTMLDocument(InputStream)`) para evitar carregar todo o arquivo na memória de uma vez. |
