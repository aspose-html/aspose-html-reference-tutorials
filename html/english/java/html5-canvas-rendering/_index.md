## Tips & Best Practices
- **Pro tip:** Use the `CanvasRenderingContext2D` API to draw vector shapes before exporting; vector data scales better in PDF.
- **Performance tip:** Render the canvas at the target PDF resolution to avoid extra scaling steps.
- **Debug tip:** Save the intermediate canvas as a PNG (using `saveAs`) to verify the visual output before PDF conversion.

## Frequently Asked Questions

**Q: Can I export a Canvas that contains images from external URLs?**  
A: Yes. Aspose.HTML for Java can download and embed external images during the rendering process, provided the URLs are publicly accessible.

**Q: Is it possible to draw custom shapes like polygons or bezier curves?**  
A: Absolutely. The Canvas 2D API supports `beginPath()`, `moveTo()`, `lineTo()`, and `bezierCurveTo()` methods, all of which are fully rendered by Aspose.HTML.

**Q: Does the library support high‑DPI PDF output?**  
A: Yes. You can specify the DPI when creating the PDF rendering options to ensure crisp, print‑ready results.

**Q: What if I need to add text overlay after exporting the canvas?**  
A: You can either draw the text on the canvas before export or use Aspose.PDF to add a text layer on the generated PDF.

**Q: Are there any licensing restrictions for using the library in a cloud service?**  
A: A commercial license is required for production deployments, including cloud services. A free trial can be used for development and testing.
