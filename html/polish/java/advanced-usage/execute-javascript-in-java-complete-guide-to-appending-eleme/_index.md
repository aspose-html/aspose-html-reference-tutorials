# Wykonywanie JavaScript w Javie – Kompletny przewodnik po dodawaniu elementów do elementu body

Czy kiedykolwiek potrzebowałeś **wykonać JavaScript w Javie**, ale nie wiedziałeś, jak połączyć te dwa światy? Nie jesteś sam. Wielu programistów Javy napotyka trudności, gdy chcą wykonać **manipulację DOM przy użyciu JavaScript** bez uruchamiania pełnej przeglądarki. Dobra wiadomość? Dzięki Aspose.HTML możesz uruchomić skrypt, **dodać element do body**, i zapisać zmodyfikowany kod – wszystko z czystego kodu Java.

W tym samouczku przeprowadzimy Cię przez wszystko, co musisz wiedzieć: ładowanie pliku HTML, napisanie małego skryptu, który **modyfikuje HTML przy użyciu JavaScript**, wykonanie tego skryptu z Javy oraz ostateczne zapisanie zmian. Po zakończeniu będziesz mieć samodzielny, uruchamialny przykład, który możesz wkleić do dowolnego projektu.

## Czego będziesz potrzebować

- **Java 8** lub nowsza (API działa z Java 8+)
- **Aspose.HTML for Java** library (wersja 22.12 lub nowsza)
- Prosty plik szablonu HTML (utworzymy go w locie)
- IDE lub edytor tekstu — nic specjalnego, nawet zwykły edytor tekstowy wystarczy

> Pro tip: Jeśli używasz Maven, dodaj zależność Aspose.HTML do swojego `pom.xml`. W przeciwnym razie po prostu umieść pliki JAR w classpath.

## Krok 1 – Załaduj dokument HTML (Przygotowanie do wykonywania JavaScript w Javie)

Pierwszą rzeczą, którą robimy, jest stworzenie instancji `HTMLDocument`, która wskazuje na plik źródłowy. Traktuj ten obiekt jako most między Javą a drzewem DOM, które później będzie modyfikowane przez skrypt.

```java
import com.aspose.html.HTMLDocument;

// Step 1: Load the HTML document you want to manipulate
HTMLDocument htmlDoc = new HTMLDocument("YOUR_DIRECTORY/template.html");
```

> **Dlaczego to ważne:** Klasa `HTMLDocument` parsuje plik do reprezentacji DOM, z którą silnik skryptowy Aspose może współpracować. Bez wcześniejszego załadowania dokumentu nie ma nic, co skrypt mógłby zmodyfikować.

## Krok 2 – Napisz JavaScript, który **dodaje element do body**

Teraz tworzymy mały fragment kodu, który tworzy element `<p>` i dodaje go do `<body>`. To klasyczna **manipulacja DOM przy użyciu JavaScript**, ale uruchomimy to z Javy.

```java
// Step 2: JavaScript that adds a new paragraph to the body
String jsCode = ""
    + "var p = document.createElement('p');"
    + "p.textContent = 'Generated by JavaScript';"
    + "document.body.appendChild(p);";
```

> **Dlaczego to działa:** `document.createElement` tworzy nowy węzeł, `textContent` ustawia jego wewnętrzny tekst, a `appendChild` dołącza go do istniejącego elementu `<body>`. Skrypt jest czystym JavaScript po stronie klienta; Aspose.HTML wykonuje go w środowisku bezgłowym.

## Krok 3 – Utwórz silnik skryptów (Silnik, który **uruchamia JavaScript z Javy**)

Aspose.HTML dostarcza `ScriptEngine`, który może ocenić JavaScript względem załadowanego DOM. To właśnie on **uruchamia JavaScript z Javy**.

```java
import com.aspose.html.scripting.ScriptEngine;

// Step 3: Bind the engine to the document we just loaded
ScriptEngine scriptEngine = new ScriptEngine(htmlDoc);
```

> **Dlaczego tego potrzebujesz:** Silnik izoluje wykonanie skryptu, zapobiegając wpływowi na główny proces Javy. Zwraca także `ScriptExecutionResult`, dzięki czemu możesz zweryfikować sukces.

## Krok 4 – Wykonaj skrypt i sprawdź wynik

Mając wszystko skonfigurowane, w końcu uruchamiamy skrypt. Metoda `execute` zwraca obiekt wyniku, który informuje, czy skrypt wykonał się bez błędów.

```java
import com.aspose.html.scripting.ScriptExecutionResult;

// Step 4: Run the JavaScript and verify it succeeded
ScriptExecutionResult execResult = scriptEngine.execute(jsCode);
if (execResult.isSuccess()) {
    System.out.println("Script executed successfully.");
} else {
    System.err.println("Script failed: " + execResult.getErrorMessage());
}
```

> **Przypadek brzegowy:** Jeśli skrypt zgłosi wyjątek (np. błąd składni), `isSuccess()` będzie `false`, a `getErrorMessage()` poda szczegóły. Zawsze sprawdzaj to w kodzie produkcyjnym.

## Krok 5 – Zapisz zmodyfikowany dokument (Zakończenie **modyfikacji HTML przy użyciu JavaScript**)

Po tym, jak skrypt zmienił DOM, zapisujemy zmiany na dysku. Powstały plik zawiera teraz nowy akapit.

```java
// Step 5: Write the updated DOM back to an HTML file
htmlDoc.save("YOUR_DIRECTORY/modified.html");
System.out.println("Document saved.");
```

> **Weryfikacja wyniku:** Otwórz `modified.html` w dowolnej przeglądarce. Powinieneś zobaczyć oryginalną treść oraz nową linię z napisem „Generated by JavaScript”.

## Krok 6 – Pełny działający przykład (Wszystkie kroki połączone)

Poniżej znajduje się kompletny program, który możesz skopiować i wkleić do pliku `JsDomManipulation.java`. Zawiera wszystkie importy, obsługę błędów i komentarze dla przejrzystości.

```java
import com.aspose.html.HTMLDocument;
import com.aspose.html.scripting.ScriptEngine;
import com.aspose.html.scripting.ScriptExecutionResult;

public class JsDomManipulation {
    public static void main(String[] args) throws Exception {

        // 1️⃣ Load the HTML document to be manipulated
        HTMLDocument htmlDoc = new HTMLDocument("YOUR_DIRECTORY/template.html");

        // 2️⃣ Define JavaScript that adds a new paragraph to the <body>
        String jsCode = "var p = document.createElement('p');"
                      + "p.textContent = 'Generated by JavaScript';"
                      + "document.body.appendChild(p);";

        // 3️⃣ Create a script engine bound to the loaded document
        ScriptEngine scriptEngine = new ScriptEngine(htmlDoc);

        // 4️⃣ Execute the JavaScript and report success
        ScriptExecutionResult execResult = scriptEngine.execute(jsCode);
        if (execResult.isSuccess()) {
            System.out.println("Script executed successfully.");
        } else {
            System.err.println("Script failed: " + execResult.getErrorMessage());
            return; // Stop if the script didn't run
        }

        // 5️⃣ Save the modified document back to HTML
        htmlDoc.save("YOUR_DIRECTORY/modified.html");
        System.out.println("Document saved.");
    }
}
```

### Oczekiwany wynik

Uruchomienie programu wypisuje:

```
Script executed successfully.
Document saved.
```

A `modified.html` teraz zawiera coś w stylu:

```html
<!DOCTYPE html>
<html>
<head>
    <title>Template</title>
</head>
<body>
    <h1>Original Content</h1>
    <p>Generated by JavaScript</p>
</body>
</html>
```

## Częste pytania i pułapki

| Pytanie | Odpowiedź |
|----------|--------|
| **Czy mogę uruchamiać wiele skryptów kolejno?** | Tak. Po prostu wywołaj ponownie `scriptEngine.execute()` z nowym ciągiem skryptu. Ta sama instancja `HTMLDocument` zachowuje swój stan. |
| **Co jeśli plik HTML nie zawiera tagu `<body>`?** | Aspose.HTML automatycznie utworzy element `<body>` podczas parsowania, więc skrypt nadal zadziała. Jednak dla przewidywalnych rezultatów warto zacząć od dobrze sformowanego szablonu. |
| **Czy to podejście jest kompatybilne z Java 17?** | Absolutnie. Biblioteka jest przeznaczona dla Java 8+, więc nowsze środowiska działają bez problemu. |
| **Czy potrzebuję pełnego środowiska przeglądarki?** | Nie. Aspose.HTML zapewnia lekkie, bezgłowe środowisko, które interpretuje standardowe API DOM bez uruchamiania Chrome czy Firefox. |
| **Jak obsłużyć duże pliki HTML?** | Załaduj dokument jako strumień (`new HTMLDocument(InputStream)`), aby uniknąć wczytywania całego pliku do pamięci naraz. |

- **Wstrzyknij CSS**, aby stylizować nowo dodane elementy.
- **Przejdź po DOM**, aby modyfikować istniejące węzły na podstawie atrybutów.
- **Uruchom zewnętrzne skrypty** pobrane z URL (użyj `scriptEngine.executeFromUrl()`, jeśli potrzebne).
- Połącz tę technikę z **konwersją do PDF** — Aspose.HTML może renderować finalny DOM do PDF w jednym wywołaniu.

Jeśli interesuje Cię **uruchamianie JavaScript z Javy** w celach testowych lub renderowania po stronie serwera, rozważ opakowanie logiki w klasę pomocniczą i ponowne wykorzystanie jej w różnych mikroserwisach.
