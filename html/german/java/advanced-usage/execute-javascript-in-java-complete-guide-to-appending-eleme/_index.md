# JavaScript in Java ausführen – Vollständige Anleitung zum Anhängen von Elementen an den Body

Haben Sie jemals **JavaScript in Java** ausführen müssen, wussten aber nicht, wie Sie die beiden Welten verbinden? Sie sind nicht allein. Viele Java‑Entwickler stoßen an Grenzen, wenn sie **JavaScript DOM‑Manipulation** durchführen wollen, ohne einen vollständigen Browser zu starten. Die gute Nachricht? Mit Aspose.HTML können Sie ein Skript ausführen, **ein Element an den Body anhängen** und das geänderte Markup speichern – alles aus reinem Java‑Code.

In diesem Tutorial führen wir Sie durch alles, was Sie wissen müssen: Laden einer HTML‑Datei, Schreiben eines kleinen Skripts, das **HTML mit JavaScript modifiziert**, Ausführen dieses Skripts aus Java und schließlich das Persistieren der Änderungen. Am Ende haben Sie ein eigenständiges, ausführbares Beispiel, das Sie in jedes Projekt einbinden können.

## Was Sie benötigen

- **Java 8** oder neuer (die API funktioniert mit Java 8+)
- **Aspose.HTML for Java** Bibliothek (Version 22.12 oder später)
- Eine einfache HTML‑Vorlagendatei (wir erstellen sie on the fly)
- Eine IDE oder ein Texteditor – nichts Besonderes, sogar ein einfacher Texteditor reicht

> Pro Tipp: Wenn Sie Maven verwenden, fügen Sie die Aspose.HTML‑Abhängigkeit zu Ihrer `pom.xml` hinzu. Andernfalls legen Sie die JARs einfach in Ihren Klassenpfad.

## Schritt 1 – Laden des HTML‑Dokuments (Vorbereitung zum Ausführen von JavaScript in Java)

Das Erste, was wir tun, ist eine `HTMLDocument`‑Instanz zu erstellen, die auf die Quelldatei zeigt. Betrachten Sie dieses Objekt als Brücke zwischen Java und dem DOM‑Baum, den das Skript später berühren wird.

```java
import com.aspose.html.HTMLDocument;

// Step 1: Load the HTML document you want to manipulate
HTMLDocument htmlDoc = new HTMLDocument("YOUR_DIRECTORY/template.html");
```

> **Warum das wichtig ist:** Die Klasse `HTMLDocument` parsed die Datei in eine DOM‑Repräsentation, mit der die Aspose‑Scripting‑Engine interagieren kann. Ohne das Dokument zuerst zu laden, gibt es nichts, das das Skript ändern könnte.

## Schritt 2 – Schreiben von JavaScript, das **Element an den Body anhängt**

Jetzt erstellen wir ein winziges Snippet, das ein `<p>`‑Element erzeugt und es zum `<body>` hinzufügt. Das ist klassische **JavaScript DOM‑Manipulation**, aber wir führen es aus Java aus.

```java
// Step 2: JavaScript that adds a new paragraph to the body
String jsCode = ""
    + "var p = document.createElement('p');"
    + "p.textContent = 'Generated by JavaScript';"
    + "document.body.appendChild(p);";
```

> **Warum das funktioniert:** `document.createElement` erzeugt einen neuen Knoten, `textContent` setzt dessen Innen­text, und `appendChild` hängt ihn an das bestehende `<body>`‑Element an. Das Skript ist reines client‑seitiges JavaScript; Aspose.HTML führt es in einer headless‑Umgebung aus.

## Schritt 3 – Erstellen der Script‑Engine (Die Engine, die **JavaScript aus Java ausführt**)

Aspose.HTML liefert eine `ScriptEngine`, die JavaScript gegen das geladene DOM auswerten kann. Sie ist das Bauteil, das tatsächlich **JavaScript aus Java ausführt**.

```java
import com.aspose.html.scripting.ScriptEngine;

// Step 3: Bind the engine to the document we just loaded
ScriptEngine scriptEngine = new ScriptEngine(htmlDoc);
```

> **Warum Sie das benötigen:** Die Engine isoliert die Skriptausführung, sodass sie Ihren Haupt‑Java‑Prozess nicht beeinflusst. Sie liefert außerdem ein `ScriptExecutionResult`, mit dem Sie den Erfolg prüfen können.

## Schritt 4 – Ausführen des Skripts und Prüfen des Ergebnisses

Nachdem alles eingerichtet ist, starten wir schließlich das Skript. Die Methode `execute` gibt ein Ergebnisobjekt zurück, das anzeigt, ob das Skript fehlerfrei ausgeführt wurde.

```java
import com.aspose.html.scripting.ScriptExecutionResult;

// Step 4: Run the JavaScript and verify it succeeded
ScriptExecutionResult execResult = scriptEngine.execute(jsCode);
if (execResult.isSuccess()) {
    System.out.println("Script executed successfully.");
} else {
    System.err.println("Script failed: " + execResult.getErrorMessage());
}
```

> **Randfall:** Wenn das Skript eine Ausnahme wirft (z. B. einen Syntaxfehler), ist `isSuccess()` `false` und `getErrorMessage()` liefert die Details. Prüfen Sie das immer im Produktionscode.

## Schritt 5 – Speichern des modifizierten Dokuments (Abschluss von **HTML mit JavaScript modifizieren**)

Nachdem das Skript den DOM verändert hat, speichern wir die Änderungen zurück auf die Festplatte. Die resultierende Datei enthält nun den neuen Absatz.

```java
// Step 5: Write the updated DOM back to an HTML file
htmlDoc.save("YOUR_DIRECTORY/modified.html");
System.out.println("Document saved.");
```

> **Ergebnis‑Verifizierung:** Öffnen Sie `modified.html` in einem beliebigen Browser. Sie sollten den ursprünglichen Inhalt plus eine neue Zeile sehen, die „Generated by JavaScript“ lautet.

## Schritt 6 – Vollständiges funktionierendes Beispiel (Alle Schritte kombiniert)

Unten finden Sie das komplette Programm, das Sie in eine `JsDomManipulation.java`‑Datei kopieren können. Es enthält alle Importe, Fehlerbehandlung und Kommentare zur Klarheit.

```java
import com.aspose.html.HTMLDocument;
import com.aspose.html.scripting.ScriptEngine;
import com.aspose.html.scripting.ScriptExecutionResult;

public class JsDomManipulation {
    public static void main(String[] args) throws Exception {

        // 1️⃣ Load the HTML document to be manipulated
        HTMLDocument htmlDoc = new HTMLDocument("YOUR_DIRECTORY/template.html");

        // 2️⃣ Define JavaScript that adds a new paragraph to the <body>
        String jsCode = "var p = document.createElement('p');"
                      + "p.textContent = 'Generated by JavaScript';"
                      + "document.body.appendChild(p);";

        // 3️⃣ Create a script engine bound to the loaded document
        ScriptEngine scriptEngine = new ScriptEngine(htmlDoc);

        // 4️⃣ Execute the JavaScript and report success
        ScriptExecutionResult execResult = scriptEngine.execute(jsCode);
        if (execResult.isSuccess()) {
            System.out.println("Script executed successfully.");
        } else {
            System.err.println("Script failed: " + execResult.getErrorMessage());
            return; // Stop if the script didn't run
        }

        // 5️⃣ Save the modified document back to HTML
        htmlDoc.save("YOUR_DIRECTORY/modified.html");
        System.out.println("Document saved.");
    }
}
```

### Erwartete Ausgabe

Das Ausführen des Programms gibt aus:

```
Script executed successfully.
Document saved.
```

Und `modified.html` enthält nun etwa Folgendes:

```html
<!DOCTYPE html>
<html>
<head>
    <title>Template</title>
</head>
<body>
    <h1>Original Content</h1>
    <p>Generated by JavaScript</p>
</body>
</html>
```

## Häufige Fragen & Stolperfallen

| Frage | Antwort |
|----------|--------|
| **Kann ich mehrere Skripte nacheinander ausführen?** | Ja. Rufen Sie einfach `scriptEngine.execute()` erneut mit einem neuen Skript‑String auf. Die gleiche `HTMLDocument`‑Instanz behält ihren Zustand. |
| **Was passiert, wenn die HTML‑Datei kein `<body>`‑Tag enthält?** | Aspose.HTML erstellt während des Parsens automatisch ein `<body>`‑Element, sodass das Skript weiterhin funktioniert. Für vorhersehbare Ergebnisse sollten Sie jedoch mit einer gut‑geformten Vorlage beginnen. |
| **Ist dieser Ansatz mit Java 17 kompatibel?** | Absolut. Die Bibliothek zielt auf Java 8+ ab, sodass neuere Laufzeiten problemlos funktionieren. |
| **Benötige ich eine vollständige Browserumgebung?** | Nein. Aspose.HTML stellt eine leichte, headless‑Engine bereit, die standardmäßige DOM‑APIs interpretiert, ohne Chrome oder Firefox zu starten. |
| **Wie gehe ich mit großen HTML‑Dateien um?** | Laden Sie das Dokument als Stream (`new HTMLDocument(InputStream)`) um zu vermeiden, dass die gesamte Datei gleichzeitig im Speicher liegt. |
