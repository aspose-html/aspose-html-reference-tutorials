# 将动态 HTML 转换为 PDF（使用 Aspose HTML for Java）

是否曾经需要**将动态 html pdf 转换**，但你处理的页面依赖 JavaScript 来构建表格、图表或菜单？你并不孤单。在许多真实项目中，收到的 HTML 并不是静态的——脚本会运行，DOM 节点出现，只有这样页面才会呈现出你期望的样子。

在本教程中，我们将演示一个完整、可运行的示例，**加载带脚本的 HTML**、**等待 JavaScript 执行**、使用 **query selector 表格行** 检查最终的 DOM，最后使用 Aspose HTML for Java 库 **将动态 HTML 转换为 PDF**。完成后，你将拥有一个可以直接放入任何 Maven 或 Gradle 项目并立即运行的单个 Java 类。

> **为什么要在意？**  
> 在脚本执行完毕之前进行转换，往往会得到空白的 PDF 或缺失的表格。本文展示的方法确保 PDF 完全反映用户在浏览器中看到的内容。

---

## 你需要准备的环境

- **Java 17**（或任意较新的 JDK；API 支持 Java 8 以上，但更新的 JDK 性能更佳）  
- **Aspose.HTML for Java** 23.10（或更高版本）——可从 Maven Central 获取  
- 一个 **动态 HTML 文件**（这里使用包含构建表格脚本的 `dynamic.html`）  
- 对 Java I/O 与 Maven/Gradle 有基本了解（不需要深入的专业知识）

如果你已经有 Maven `pom.xml`，请添加 Aspose 依赖：

```xml
<dependency>
    <groupId>com.aspose</groupId>
    <artifactId>aspose-html</artifactId>
    <version>23.10</version>
</dependency>
```

---

## 第一步：加载启用脚本的 HTML

首先需要告诉 Aspose，即将加载的 HTML 可能包含需要运行的 JavaScript。这通过 `HTMLLoadOptions` 完成——将 **enableScripts** 标志设为 `true`。

```java
import com.aspose.html.*;
import com.aspose.html.dom.*;
import java.nio.file.Paths;

/**
 * Demonstrates loading an HTML file that contains JavaScript.
 */
public class RunJsBeforeConversion {

    public static void main(String[] args) throws Exception {

        // --------------------------------------------------------------------
        // 1️⃣ Load the HTML document with JavaScript execution enabled
        // --------------------------------------------------------------------
        // The `true` argument tells Aspose to enable script execution.
        HTMLLoadOptions loadOptions = new HTMLLoadOptions(true);

        // Adjust the path to point at your own dynamic.html file.
        HTMLDocument htmlDocument = new HTMLDocument(
                Paths.get("YOUR_DIRECTORY/dynamic.html").toUri(),
                loadOptions);
```

> **小技巧：** 将 HTML 文件放在源码文件夹旁边或使用绝对路径；否则 `toUri()` 调用会抛出 `FileNotFoundException`。

---

## 第二步：等待 JavaScript 执行

Aspose 在轻量级引擎上运行脚本，但仍需给页面一点时间完成执行。在生产系统中，你可能会挂钩到 DOM‑ready 事件，但在快速演示中，一个简单的暂停就足够了。

```java
        // --------------------------------------------------------------------
        // 2️⃣ Wait for JavaScript execution (simple pause for demo purposes)
        // --------------------------------------------------------------------
        // 2 seconds is usually enough for tiny demos; increase if your script
        // does heavy work or makes async network calls.
        Thread.sleep(2000);
```

> **为什么要暂停？** 库同步执行脚本，但某些操作（如 `setTimeout`）会被加入队列。睡眠可以确保这些队列在我们检查 DOM 之前被清空。

---

## 第三步：使用 Query Selector 查询表格行——验证 DOM

脚本运行完毕后，我们可以像在浏览器控制台中一样使用 CSS 选择器获取元素。这里我们定位 `id="report"` 的表格，并打印其包含的行数。

```java
        // --------------------------------------------------------------------
        // 3️⃣ Inspect the DOM after script execution
        // --------------------------------------------------------------------
        // Using a CSS selector to find the table generated by JavaScript.
        Element reportTable = htmlDocument.querySelector("table#report");

        // Guard against a missing table – helpful when the script fails.
        if (reportTable == null) {
            System.out.println("⚠️ No table with id='report' found.");
            return;
        }

        // The Element API gives us direct access to rows collection.
        int rowCount = reportTable.getRows().getLength();
        System.out.println("Rows after script execution: " + rowCount);
```

示例 `dynamic.html` 的典型输出如下：

```
Rows after script execution: 5
```

如果看到的行数不同，请再次检查 HTML 中的脚本——可能是条件生成了行。

---

## 第四步：将最终的 DOM 状态转换为 PDF

在验证完 DOM 后，最后一步只需一行代码：将 `HTMLDocument` 交给 `Converter.convert`。生成的 PDF 完全对应脚本执行完毕后浏览器的渲染效果。

```java
        // --------------------------------------------------------------------
        // 4️⃣ Convert the final DOM state to a PDF file
        // --------------------------------------------------------------------
        Converter.convert(htmlDocument,
                Paths.get("YOUR_DIRECTORY/dynamic.pdf").toUri());

        System.out.println("✅ PDF generated at YOUR_DIRECTORY/dynamic.pdf");
    }
}
```

使用任意阅读器打开 `dynamic.pdf`——你应该能看到完整填充的表格、样式以及脚本注入的所有图片。

## 完整工作示例（所有步骤合并）

下面是完整的源文件，可直接复制到 `src/main/java/RunJsBeforeConversion.java`。记得将 `YOUR_DIRECTORY` 替换为实际存放 `dynamic.html` 的文件夹路径。

```java
import com.aspose.html.*;
import com.aspose.html.dom.*;
import java.nio.file.Paths;

/**
 * End‑to‑end demo: load a dynamic HTML page, let its JavaScript run,
 * query the generated table, and convert the result to PDF.
 *
 * Requires Aspose.HTML for Java 23.10+ on the classpath.
 */
public class RunJsBeforeConversion {
    public static void main(String[] args) throws Exception {

        // 1️⃣ Load HTML with scripts enabled
        HTMLLoadOptions loadOptions = new HTMLLoadOptions(true);
        HTMLDocument htmlDocument = new HTMLDocument(
                Paths.get("YOUR_DIRECTORY/dynamic.html").toUri(),
                loadOptions);

        // 2️⃣ Wait for JavaScript execution (2 seconds)
        Thread.sleep(2000);

        // 3️⃣ Query selector table rows – verify script output
        Element reportTable = htmlDocument.querySelector("table#report");
        if (reportTable == null) {
            System.out.println("⚠️ No table with id='report' found.");
            return;
        }
        int rowCount = reportTable.getRows().getLength();
        System.out.println("Rows after script execution: " + rowCount);

        // 4️⃣ Convert the final DOM to PDF
        Converter.convert(htmlDocument,
                Paths.get("YOUR_DIRECTORY/dynamic.pdf").toUri());

        System.out.println("✅ PDF generated at YOUR_DIRECTORY/dynamic.pdf");
    }
}
```

运行方式：

```bash
mvn compile exec:java -Dexec.mainClass=RunJsBeforeConversion
```

或使用等效的 Gradle 命令。

## 常见问题及解决方案

| 问题 | 产生原因 | 解决办法 |
|------|----------|----------|
| **PDF 空白** | 脚本被禁用（`HTMLLoadOptions(false)`）或暂停时间太短。 | 确保 `enableScripts` 为 `true`，并在页面使用异步调用时延长 `Thread.sleep`。 |
| **`reportTable` 为 null** | 选择器错误或脚本未创建该元素。 | 检查 HTML 中的 `<script>` 是否真的添加了 `<table id="report">`。使用 Chrome DevTools 检查最终 DOM。 |
| **缺少字体或 CSS** | HTML 引用了不可访问的外部样式表。 | 使用绝对 URL，或将 CSS 文件复制到本地并使用 `file://` 路径引用。 |
| **大页面导致内存激增** | Aspose 将整个 DOM 加载到内存中。 | 使用 `HTMLLoadOptions.setMaxMemoryUsage` 限制内存消耗，或将转换拆分为多个块。 |

---

## 扩展方案

- **多页处理** – 如果动态页面使用分页，在更新 URL 片段（`#page=2` 等）后，在循环中调用 `Converter.convert`。  
- **无头浏览器替代** – 对于极其复杂的脚本（例如 WebGL），可以考虑集成真实的无头浏览器如 **Playwright**，并将渲染后的 HTML 交给 Aspose。  
- **自定义渲染选项** – `PdfSaveOptions` 允许设置页面大小、边距和图像质量。例如：`new PdfSaveOptions(PdfPageSize.A4, PdfPageOrientation.Portrait)`。
