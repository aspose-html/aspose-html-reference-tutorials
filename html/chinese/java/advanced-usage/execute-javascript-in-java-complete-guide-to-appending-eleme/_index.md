# 在 Java 中执行 JavaScript – 完整的向 Body 追加元素指南

是否曾需要 **在 Java 中执行 JavaScript**，却不知如何在两者之间搭桥？你并非唯一。许多 Java 开发者在想要进行 **JavaScript DOM 操作** 而不启动完整浏览器时会卡住。好消息是？使用 Aspose.HTML，你可以运行脚本、**向 body 追加元素**，并保存修改后的 markup——全部在纯 Java 代码中完成。

在本教程中，我们将逐步讲解所有必备内容：加载 HTML 文件、编写一个 **使用 JavaScript 修改 HTML** 的小脚本、从 Java 执行该脚本，最后持久化更改。完成后，你将拥有一个自包含、可直接运行的示例，能够放入任何项目中使用。

## 你需要的环境

- **Java 8** 或更高（API 支持 Java 8+）
- **Aspose.HTML for Java** 库（版本 22.12 或更高）
- 一个简单的 HTML 模板文件（我们将在运行时创建）
- IDE 或文本编辑器——不需要花哨的工具，普通纯文本编辑器即可

> 小技巧：如果使用 Maven，请在 `pom.xml` 中添加 Aspose.HTML 依赖。否则，只需将 JAR 包放入 classpath 即可。

## 第一步 – 加载 HTML 文档（为在 Java 中执行 JavaScript 做准备）

首先我们创建一个指向源文件的 `HTMLDocument` 实例。把这个对象想象成 Java 与脚本后续将触及的 DOM 树之间的桥梁。

```java
import com.aspose.html.HTMLDocument;

// Step 1: Load the HTML document you want to manipulate
HTMLDocument htmlDoc = new HTMLDocument("YOUR_DIRECTORY/template.html");
```

> **为什么这很重要：** `HTMLDocument` 类会将文件解析为 DOM 表示，供 Aspose 脚本引擎交互。若不先加载文档，脚本将无对象可修改。

## 第二步 – 编写 **向 Body 追加元素** 的 JavaScript

现在我们编写一个小片段，创建一个 `<p>` 元素并将其添加到 `<body>` 中。这是经典的 **JavaScript DOM 操作**，但我们将在 Java 中运行它。

```java
// Step 2: JavaScript that adds a new paragraph to the body
String jsCode = ""
    + "var p = document.createElement('p');"
    + "p.textContent = 'Generated by JavaScript';"
    + "document.body.appendChild(p);";
```

> **工作原理：** `document.createElement` 创建新节点，`textContent` 设置其内部文本，`appendChild` 将其附加到已有的 `<body>` 元素。该脚本是纯客户端 JavaScript；Aspose.HTML 在无头环境中执行它。

## 第三步 – 创建脚本引擎（能够 **从 Java 运行 JavaScript** 的引擎）

Aspose.HTML 附带一个 `ScriptEngine`，可以在已加载的 DOM 上评估 JavaScript。它正是实际 **从 Java 运行 JavaScript** 的核心组件。

```java
import com.aspose.html.scripting.ScriptEngine;

// Step 3: Bind the engine to the document we just loaded
ScriptEngine scriptEngine = new ScriptEngine(htmlDoc);
```

> **为什么需要它：** 引擎将脚本执行隔离，防止影响主 Java 进程。同时它会返回 `ScriptExecutionResult`，让你能够验证执行是否成功。

## 第四步 – 执行脚本并检查结果

一切就绪后，我们最终触发脚本。`execute` 方法返回一个结果对象，告知脚本是否无错误运行。

```java
import com.aspose.html.scripting.ScriptExecutionResult;

// Step 4: Run the JavaScript and verify it succeeded
ScriptExecutionResult execResult = scriptEngine.execute(jsCode);
if (execResult.isSuccess()) {
    System.out.println("Script executed successfully.");
} else {
    System.err.println("Script failed: " + execResult.getErrorMessage());
}
```

> **边缘情况：** 若脚本抛出异常（例如语法错误），`isSuccess()` 将返回 `false`，`getErrorMessage()` 提供详细信息。生产代码中务必进行此检查。

## 第五步 – 保存修改后的文档（完成 **使用 JavaScript 修改 HTML**）

脚本修改了 DOM 后，我们将更改持久化回磁盘。生成的文件现在包含了新的段落。

```java
// Step 5: Write the updated DOM back to an HTML file
htmlDoc.save("YOUR_DIRECTORY/modified.html");
System.out.println("Document saved.");
```

> **结果验证：** 在任意浏览器中打开 `modified.html`。你应当看到原始内容外加一行新文字：“Generated by JavaScript”。

## 第六步 – 完整工作示例（所有步骤合并）

下面是完整的程序代码，可直接复制粘贴到 `JsDomManipulation.java` 文件中。它包含所有 import、错误处理以及注释，便于阅读。

```java
import com.aspose.html.HTMLDocument;
import com.aspose.html.scripting.ScriptEngine;
import com.aspose.html.scripting.ScriptExecutionResult;

public class JsDomManipulation {
    public static void main(String[] args) throws Exception {

        // 1️⃣ Load the HTML document to be manipulated
        HTMLDocument htmlDoc = new HTMLDocument("YOUR_DIRECTORY/template.html");

        // 2️⃣ Define JavaScript that adds a new paragraph to the <body>
        String jsCode = "var p = document.createElement('p');"
                      + "p.textContent = 'Generated by JavaScript';"
                      + "document.body.appendChild(p);";

        // 3️⃣ Create a script engine bound to the loaded document
        ScriptEngine scriptEngine = new ScriptEngine(htmlDoc);

        // 4️⃣ Execute the JavaScript and report success
        ScriptExecutionResult execResult = scriptEngine.execute(jsCode);
        if (execResult.isSuccess()) {
            System.out.println("Script executed successfully.");
        } else {
            System.err.println("Script failed: " + execResult.getErrorMessage());
            return; // Stop if the script didn't run
        }

        // 5️⃣ Save the modified document back to HTML
        htmlDoc.save("YOUR_DIRECTORY/modified.html");
        System.out.println("Document saved.");
    }
}
```

### 预期输出

运行程序后会打印：

```
Script executed successfully.
Document saved.
```

并且 `modified.html` 将包含类似以下内容：

```html
<!DOCTYPE html>
<html>
<head>
    <title>Template</title>
</head>
<body>
    <h1>Original Content</h1>
    <p>Generated by JavaScript</p>
</body>
</html>
```

## 常见问题与注意事项

| 问题 | 解答 |
|----------|--------|
| **可以顺序运行多个脚本吗？** | 可以。只需再次调用 `scriptEngine.execute()` 并传入新的脚本字符串。相同的 `HTMLDocument` 实例会保留其状态。 |
| **如果 HTML 文件缺少 `<body>` 标签怎么办？** | Aspose.HTML 在解析时会自动创建 `<body>` 元素，脚本仍能正常工作。不过，为了得到可预期的结果，建议使用结构完整的模板。 |
| **此方法兼容 Java 17 吗？** | 完全兼容。该库面向 Java 8+，因此更高版本运行没有问题。 |
| **需要完整的浏览器环境吗？** | 不需要。Aspose.HTML 提供轻量级、无头的引擎，能够解释标准 DOM API，而无需启动 Chrome 或 Firefox。 |
| **如何处理大型 HTML 文件？** | 使用流式加载文档（`new HTMLDocument(InputStream)`）以避免一次性将整个文件读入内存。 |

- **遍历 DOM** 根据属性修改已有节点。
- **运行外部脚本**（如从 URL 获取），如有需要可使用 `scriptEngine.executeFromUrl()`。
- 将此技术与 **PDF 转换** 结合——Aspose.HTML 能在一次调用中将最终 DOM 渲染为 PDF。

如果你想在测试或服务器端渲染中 **从 Java 运行 JavaScript**，可以将上述逻辑封装为工具类，并在微服务之间复用。
