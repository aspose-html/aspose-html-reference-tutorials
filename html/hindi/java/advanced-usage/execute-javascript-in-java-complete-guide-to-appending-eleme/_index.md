# Java में JavaScript निष्पादित करें – बॉडी में एलिमेंट जोड़ने की पूरी गाइड

क्या आपको कभी **Java में JavaScript निष्पादित** करने की ज़रूरत पड़ी है लेकिन दो दुनियाओं को जोड़ने का तरीका नहीं पता था? आप अकेले नहीं हैं। कई Java डेवलपर्स को तब रुकावट आती है जब वे **JavaScript DOM manipulation** को पूरी ब्राउज़र खोले बिना करना चाहते हैं। अच्छी खबर? Aspose.HTML के साथ आप एक स्क्रिप्ट चला सकते हैं, **बॉडी में एलिमेंट जोड़ सकते** हैं, और बदले हुए मार्कअप को सहेज सकते हैं—सभी शुद्ध Java कोड से।

इस ट्यूटोरियल में हम वह सब कवर करेंगे जो आपको जानना आवश्यक है: HTML फ़ाइल लोड करना, एक छोटा स्क्रिप्ट लिखना जो **JavaScript के साथ HTML को संशोधित** करता है, उस स्क्रिप्ट को Java से चलाना, और अंत में बदलावों को स्थायी बनाना। अंत तक आपके पास एक स्व-निहित, चलाने योग्य उदाहरण होगा जिसे आप किसी भी प्रोजेक्ट में डाल सकते हैं।

## What You’ll Need

- **Java 8** या नया (API Java 8+ के साथ काम करता है)
- **Aspose.HTML for Java** लाइब्रेरी (वर्ज़न 22.12 या बाद वाला)
- एक सरल HTML टेम्पलेट फ़ाइल (हम इसे तुरंत बनाएँगे)
- एक IDE या टेक्स्ट एडिटर—कुछ भी चल जाएगा, यहाँ तक कि साधारण टेक्स्ट एडिटर भी ठीक है

> Pro tip: यदि आप Maven का उपयोग कर रहे हैं, तो `pom.xml` में Aspose.HTML डिपेंडेंसी जोड़ें। अन्यथा, JAR फ़ाइलों को अपने क्लासपाथ में डाल दें।

## Step 1 – Load the HTML Document (Prepare to Execute JavaScript in Java)

पहला कदम है `HTMLDocument` इंस्टेंस बनाना जो स्रोत फ़ाइल की ओर इशारा करता है। इस ऑब्जेक्ट को Java और उस DOM ट्री के बीच पुल समझें जिस पर स्क्रिप्ट बाद में काम करेगी।

```java
import com.aspose.html.HTMLDocument;

// Step 1: Load the HTML document you want to manipulate
HTMLDocument htmlDoc = new HTMLDocument("YOUR_DIRECTORY/template.html");
```

> **Why this matters:** `HTMLDocument` क्लास फ़ाइल को DOM प्रतिनिधित्व में पार्स करता है जिससे Aspose स्क्रिप्टिंग इंजन इंटरैक्ट कर सकता है। दस्तावेज़ लोड किए बिना स्क्रिप्ट के पास संशोधित करने के लिए कुछ नहीं रहेगा।

## Step 2 – Write JavaScript that **Append Element to Body**

अब हम एक छोटा स्निपेट तैयार करेंगे जो `<p>` एलिमेंट बनाता है और उसे `<body>` में जोड़ता है। यह क्लासिक **JavaScript DOM manipulation** है, लेकिन हम इसे Java से चलाएँगे।

```java
// Step 2: JavaScript that adds a new paragraph to the body
String jsCode = ""
    + "var p = document.createElement('p');"
    + "p.textContent = 'Generated by JavaScript';"
    + "document.body.appendChild(p);";
```

> **Why this works:** `document.createElement` एक नया नोड बनाता है, `textContent` उसका अंदरूनी टेक्स्ट सेट करता है, और `appendChild` उसे मौजूदा `<body>` एलिमेंट से जोड़ता है। स्क्रिप्ट शुद्ध क्लाइंट‑साइड JavaScript है; Aspose.HTML इसे हेडलेस वातावरण में चलाता है।

## Step 3 – Create the Script Engine (The Engine That **Run JavaScript from Java**)

Aspose.HTML एक `ScriptEngine` प्रदान करता है जो लोड किए गए DOM के विरुद्ध JavaScript का मूल्यांकन कर सकता है। यही वह घटक है जो वास्तव में **Java से JavaScript चलाता** है।

```java
import com.aspose.html.scripting.ScriptEngine;

// Step 3: Bind the engine to the document we just loaded
ScriptEngine scriptEngine = new ScriptEngine(htmlDoc);
```

> **Why you need this:** इंजन स्क्रिप्ट निष्पादन को अलग करता है, जिससे यह आपके मुख्य Java प्रोसेस को प्रभावित नहीं करता। यह एक `ScriptExecutionResult` भी लौटाता है जिससे आप सफलता की पुष्टि कर सकते हैं।

## Step 4 – Execute the Script and Check the Result

सब कुछ सेटअप हो जाने के बाद, हम अंततः स्क्रिप्ट को चलाते हैं। `execute` मेथड एक रिज़ल्ट ऑब्जेक्ट लौटाता है जो बताता है कि स्क्रिप्ट बिना त्रुटियों के चली या नहीं।

```java
import com.aspose.html.scripting.ScriptExecutionResult;

// Step 4: Run the JavaScript and verify it succeeded
ScriptExecutionResult execResult = scriptEngine.execute(jsCode);
if (execResult.isSuccess()) {
    System.out.println("Script executed successfully.");
} else {
    System.err.println("Script failed: " + execResult.getErrorMessage());
}
```

> **Edge case:** यदि स्क्रिप्ट कोई अपवाद फेंकती है (जैसे सिंटैक्स एरर), `isSuccess()` `false` रहेगा और `getErrorMessage()` विवरण देगा। प्रोडक्शन कोड में हमेशा इसे चेक करें।

## Step 5 – Save the Modified Document (Finish **Modify HTML with JavaScript**)

स्क्रिप्ट द्वारा DOM बदलने के बाद, हम बदलावों को डिस्क पर सहेजते हैं। परिणामी फ़ाइल में अब नया पैराग्राफ शामिल होगा।

```java
// Step 5: Write the updated DOM back to an HTML file
htmlDoc.save("YOUR_DIRECTORY/modified.html");
System.out.println("Document saved.");
```

> **Result verification:** `modified.html` को किसी भी ब्राउज़र में खोलें। आपको मूल कंटेंट के साथ एक नई लाइन दिखनी चाहिए जिसमें लिखा हो “Generated by JavaScript”।

## Step 6 – Full Working Example (All Steps Combined)

नीचे पूरा प्रोग्राम दिया गया है जिसे आप `JsDomManipulation.java` फ़ाइल में कॉपी‑पेस्ट कर सकते हैं। इसमें सभी इम्पोर्ट, एरर हैंडलिंग और स्पष्टता के लिए टिप्पणियाँ शामिल हैं।

```java
import com.aspose.html.HTMLDocument;
import com.aspose.html.scripting.ScriptEngine;
import com.aspose.html.scripting.ScriptExecutionResult;

public class JsDomManipulation {
    public static void main(String[] args) throws Exception {

        // 1️⃣ Load the HTML document to be manipulated
        HTMLDocument htmlDoc = new HTMLDocument("YOUR_DIRECTORY/template.html");

        // 2️⃣ Define JavaScript that adds a new paragraph to the <body>
        String jsCode = "var p = document.createElement('p');"
                      + "p.textContent = 'Generated by JavaScript';"
                      + "document.body.appendChild(p);";

        // 3️⃣ Create a script engine bound to the loaded document
        ScriptEngine scriptEngine = new ScriptEngine(htmlDoc);

        // 4️⃣ Execute the JavaScript and report success
        ScriptExecutionResult execResult = scriptEngine.execute(jsCode);
        if (execResult.isSuccess()) {
            System.out.println("Script executed successfully.");
        } else {
            System.err.println("Script failed: " + execResult.getErrorMessage());
            return; // Stop if the script didn't run
        }

        // 5️⃣ Save the modified document back to HTML
        htmlDoc.save("YOUR_DIRECTORY/modified.html");
        System.out.println("Document saved.");
    }
}
```

### Expected Output

प्रोग्राम चलाने पर यह प्रिंट करेगा:

```
Script executed successfully.
Document saved.
```

और `modified.html` अब कुछ इस प्रकार दिखेगा:

```html
<!DOCTYPE html>
<html>
<head>
    <title>Template</title>
</head>
<body>
    <h1>Original Content</h1>
    <p>Generated by JavaScript</p>
</body>
</html>
```

## Common Questions & Gotchas

| Question | Answer |
|----------|--------|
| **Can I run multiple scripts sequentially?** | हाँ। बस `scriptEngine.execute()` को फिर से नए स्क्रिप्ट स्ट्रिंग के साथ कॉल करें। वही `HTMLDocument` इंस्टेंस अपनी स्थिति बनाए रखता है। |
| **What if the HTML file is missing a `<body>` tag?** | Aspose.HTML पार्सिंग के दौरान स्वचालित रूप से एक `<body>` एलिमेंट बना देगा, इसलिए स्क्रिप्ट फिर भी काम करेगी। हालांकि, पूर्वानुमानित परिणामों के लिए एक सही‑फ़ॉर्मेटेड टेम्पलेट से शुरू करना बेहतर है। |
| **Is this approach compatible with Java 17?** | बिल्कुल। लाइब्रेरी Java 8+ को टार्गेट करती है, इसलिए नए रनटाइम भी ठीक काम करेंगे। |
| **Do I need a full browser environment?** | नहीं। Aspose.HTML एक हल्का, हेडलेस इंजन प्रदान करता है जो मानक DOM API को इंटरप्रेट करता है बिना Chrome या Firefox लॉन्च किए। |
| **How do I handle large HTML files?** | दस्तावेज़ को स्ट्रीम (`new HTMLDocument(InputStream)`) के रूप में लोड करें ताकि पूरी फ़ाइल को एक बार में मेमोरी में लोड करने से बचा जा सके। |

## Next Steps

अब जब आप **Java में JavaScript निष्पादित** करना जानते हैं, तो आप अधिक उन्नत **JavaScript DOM manipulation** की खोज कर सकते हैं:

- **Inject CSS** करके नए जोड़े गए एलिमेंट्स को स्टाइल करें।
- **Traverse the DOM** करके एट्रिब्यूट्स के आधार पर मौजूदा नोड्स को संशोधित करें।
- **Run external scripts** जो URL से प्राप्त हों (`scriptEngine.executeFromUrl()` का उपयोग करके)।
- इस तकनीक को **PDF conversion** के साथ मिलाएँ—Aspose.HTML एक ही कॉल में अंतिम DOM को PDF में रेंडर कर सकता है।

यदि आप **Java से JavaScript चलाने** को टेस्टिंग या सर्वर‑साइड रेंडरिंग के लिए उपयोग करना चाहते हैं, तो इस लॉजिक को एक यूटिलिटी क्लास में रैप करके माइक्रो‑सर्विसेज़ में पुनः उपयोग करने पर विचार करें।
