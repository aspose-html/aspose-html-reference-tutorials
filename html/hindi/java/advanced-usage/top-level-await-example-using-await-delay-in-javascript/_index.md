# टॉप लेवल अवेट उदाहरण – जावास्क्रिप्ट में await delay का उपयोग

क्या आप कभी सोचते थे कि सभी को async IIFE में लपेटे बिना मॉड्यूल निष्पादन को कैसे रोकें? यही वह चीज़ है जो **टॉप लेवल अवेट उदाहरण** आपको करने देता है। इस ट्यूटोरियल में हम एक छोटा वेब पेज देखेंगे जो `await delay javascript` का उपयोग करके काम को स्थगित करता है, फिर एक `increment counter class` बनाता है जो **public class fields javascript** का लाभ उठाता है। अंत तक आपके पास एक पूर्ण, कॉपी‑पेस्ट करने योग्य स्निपेट होगा जो किसी भी आधुनिक ब्राउज़र में चल सके।

हम सब कुछ कवर करेंगे—एक क्लास को सार्वजनिक फ़ील्ड के साथ परिभाषित करने से लेकर एक साधारण प्रॉमिस‑आधारित डिले हेल्पर को वायर करने तक। कोई बाहरी लाइब्रेरी नहीं, कोई बिल्ड स्टेप नहीं—सिर्फ साधारण HTML, एक `<script type="module">`, और नवीनतम ECMAScript फीचर। यदि आप पहले से async फ़ंक्शन्स से परिचित हैं, तो आप देखेंगे कि टॉप‑लेवल अवेट क्यों एक प्राकृतिक विस्तार जैसा लगता है। यदि आप **javascript class public field** सिंटैक्स में नए हैं, तो चिंता न करें—हम प्रत्येक लाइन के पीछे का कारण समझाते हैं।

## आप क्या सीखेंगे

- कैसे एक **टॉप लेवल अवेट उदाहरण** मॉड्यूल स्क्रिप्ट के अंदर काम करता है।
- `await delay javascript` हेल्पर का पैटर्न जो `setTimeout` को प्रॉमिस के साथ नकल करता है।
- कैसे एक **increment counter class** लिखें जो एक सार्वजनिक फ़ील्ड (`count`) और एक static initialization block का उपयोग करता है।
- अपेक्षित कंसोल आउटपुट और यह कैसे सत्यापित करें कि static block बाकी स्क्रिप्ट से पहले चल रहा है।
- सामान्य समस्याओं के समाधान के टिप्स, जैसे पुराने ब्राउज़र जो टॉप‑लेवल अवेट को सपोर्ट नहीं करते।

> **Pro tip:** आधुनिक ब्राउज़र (Chrome 106+, Firefox 102+, Edge 106+) पहले से ही टॉप‑लेवल अवेट को सपोर्ट करते हैं। यदि आपको पुराने वातावरण को सपोर्ट करना है, तो Vite या Babel जैसे टूल के साथ बंडल करने पर विचार करें।

## चरण 1 – सार्वजनिक फ़ील्ड और static block के साथ एक क्लास परिभाषित करें

हमारे डेमो का पहला भाग एक क्लास है जो एक संख्यात्मक काउंटर को स्टोर करता है। `count = 0;` लाइन पर ध्यान दें—यह **public class fields javascript** सिंटैक्स है जो ES2022 में पेश किया गया था। यह बिना कंस्ट्रक्टर के एक इंस्टेंस प्रॉपर्टी बनाता है।

```html
<!DOCTYPE html>
<html>
<head>
    <title>ECMAScript 2022 Demo</title>
</head>
<body>
<img src="placeholder.png" alt="top level await example screenshot" title="top level await example">

<script type="module">
    // Step 1: Class definition
    class Counter {
        // Public field holds the current count
        count = 0;

        // Static block runs once when the class is first evaluated
        static {
            console.log('Counter class initialized');
        }

        // Simple method that increments the public field
        increment() {
            this.count++;
        }
    }
```

**Static block क्यों?** यह हमें एक‑बार की इनिशियलाइज़ेशन लॉजिक (जैसे लॉगिंग) चलाने देता है जब मॉड्यूल पार्स किया जाता है, *await* चलने से पहले। इससे यह सुनिश्चित होता है कि संदेश कंसोल में सबसे पहले दिखे, यह साबित करता है कि क्लास जल्दी मूल्यांकित हुई।

## चरण 2 – `await delay javascript` हेल्पर बनाएं

अब हमें एक छोटा प्रॉमिस‑आधारित डिले फ़ंक्शन चाहिए। यह मूलतः `setTimeout` के चारों ओर एक रैपर है, लेकिन प्रॉमिस लौटाने से यह await‑योग्य बन जाता है।

```javascript
    // Step 2: Promise‑based delay helper
    const delay = ms => new Promise(resolve => setTimeout(resolve, ms));
```

**Edge case:** यदि `ms` नकारात्मक या संख्या नहीं है, तो `setTimeout` इसे `0` मान लेता है। आप वैलिडेशन जोड़ सकते हैं, लेकिन डेमो के लिए यह एक‑लाइनर कोड पढ़ने में आसान रखता है।

## चरण 3 – टॉप‑लेवल अवेट का उपयोग करके निष्पादन को रोकें

अब आता है शो का सितारा: टॉप‑लेवल अवेट। क्योंकि हमारी स्क्रिप्ट एक मॉड्यूल है (`type="module"`), हम `await` को सीधे टॉप स्कोप में रख सकते हैं। यह मॉड्यूल को तब तक रोक देता है जब तक प्रॉमिस रिजॉल्व नहीं हो जाता, जिससे हम साइड इफ़ेक्ट्स के क्रम को नियंत्रित कर सकते हैं।

```javascript
    // Step 3: Top‑level await – delay half a second
    await delay(500);   // Wait 500 ms so the static block logs first
```

**आम सवाल:** “क्या मैं सामान्य `<script>` टैग में टॉप‑लेवल अवेट का उपयोग कर सकता हूँ?” नहीं—केवल मॉड्यूल स्क्रिप्ट्स इसे सपोर्ट करती हैं। यदि आप `type="module"` भूल जाते हैं, तो आपको सिंटैक्स एरर मिलेगा।

## चरण 4 – क्लास को इंस्टैंशिएट करें, इन्क्रीमेंट करें, और परिणाम लॉग करें

अंत में हम सब कुछ एक साथ जोड़ते हैं: एक इंस्टेंस बनाते हैं, `increment()` कॉल करते हैं, और अंतिम काउंट आउटपुट करते हैं। यह **increment counter class** को कार्रवाई में दिखाता है।

```javascript
    // Step 4: Work with the Counter instance
    const counterInstance = new Counter();
    counterInstance.increment();               // count becomes 1
    console.log('Final count:', counterInstance.count);
</script>
</body>
</html>
```

### अपेक्षित कंसोल आउटपुट

```
Counter class initialized
Final count: 1
```

यदि आप पेज को DevTools में खोलते हैं, तो आपको static‑block संदेश **डिले समाप्त होने से पहले** दिखाई देना चाहिए, जिससे पुष्टि होती है कि क्लास पहले मूल्यांकित हुई।

## चरण 5 – कंस्ट्रक्टर की बजाय सार्वजनिक क्लास फ़ील्ड क्यों उपयोग करें?

आप सोच सकते हैं कि हमने यह क्यों नहीं लिखा:

```javascript
class Counter {
    constructor() {
        this.count = 0;
    }
}
```

दोनों तरीके काम करते हैं, लेकिन **public class fields javascript** आपको एक साफ़, अधिक घोषणात्मक सिंटैक्स देता है। फ़ील्ड एक बार परिभाषित होती है, हर कंस्ट्रक्टर कॉल में नहीं, जिससे क्लास बड़े होने पर पढ़ना आसान हो जाता है। अतिरिक्त रूप से, static blocks को कंस्ट्रक्टर के अंदर नहीं रखा जा सकता, इसलिए इनिशियलाइज़ेशन लॉजिक को अलग करना अधिक स्वाभाविक बन जाता है।

## चरण 6 – वास्तविक‑दुनिया के ऐप्स के लिए उदाहरण को अनुकूलित करना

प्रोडक्शन में आपको अक्सर एक से अधिक काउंटर की जरूरत पड़ेगी। यहाँ कुछ त्वरित विचार हैं:

- **एकाधिक काउंटर:** उन्हें `Map` में स्टोर करें, जहाँ कुंजी पहचानकर्ता हो।
- **स्थायी स्थिति:** `console.log` को `localStorage` लिखने से बदलें।
- **Async इनिशियलाइज़ेशन:** static block का उपयोग करके सर्वर से कॉन्फ़िगरेशन फ़ेच करें, इससे पहले कि कोई इंस्टेंस बनाया जाए।

इन सभी पैटर्न को अभी भी टॉप‑लेवल अवेट से लाभ मिलता है, क्योंकि आप मॉड्यूल लोड समय पर डेटा फ़ेच कर सकते हैं और यह सुनिश्चित कर सकते हैं कि यह हर कंज्यूमर के लिए तैयार है।

## अक्सर पूछे जाने वाले प्रश्न

**प्रश्न: क्या टॉप‑लेवल अवेट अन्य मॉड्यूल्स को ब्लॉक करता है?**  
उत्तर: नहीं। प्रत्येक मॉड्यूल स्वतंत्र रूप से चलता है। केवल वह मॉड्यूल जो await रखता है, वह रुका रहता है; अन्य मॉड्यूल समानांतर में लोड होते रहते हैं।

**प्रश्न: यदि डिले प्रॉमिस रेजेक्ट हो जाए तो क्या होगा?**  
उत्तर: अनहैंडल्ड रेजेक्शन मॉड्यूल मूल्यांकन को रोक देगा और कंसोल में एरर के रूप में दिखेगा। यदि आपको ग्रेसफ़ुल फ़ॉलबैक चाहिए, तो await को `try…catch` में रैप करें।

**प्रश्न: क्या `static` कीवर्ड आवश्यक है?**  
उत्तर: काउंटर के लिए नहीं, लेकिन static block यह दर्शाने का एक सुविधाजनक तरीका है कि कोड *एक बार* लोड समय पर चलता है—लॉगिंग या फीचर‑डिटेक्शन के लिए बढ़िया।

## निष्कर्ष

आपने अभी एक **टॉप लेवल अवेट उदाहरण** बनाया है जो `await delay javascript`, एक **increment counter class**, और **public class fields javascript** की शक्ति को दर्शाता है। पूर्ण, चलने योग्य स्निपेट ऊपर मौजूद है, और कंसोल आउटपुट यह साबित करता है कि static block डिले वाले कोड चलने से पहले फायर होता है।

अब आप अधिक जटिल async फ्लो के साथ प्रयोग कर सकते हैं, डिले को वास्तविक API कॉल से बदल सकते हैं, या क्लास में अतिरिक्त मेथड जोड़ सकते हैं। याद रखें, आधुनिक जावास्क्रिप्ट आपको मॉड्यूल्स को प्रथम‑श्रेणी के async एंटिटी के रूप में ट्रीट करने देता है—कोई अतिरिक्त रैपर नहीं चाहिए।

हैप्पी कोडिंग, और अपने वैरिएशन कमेंट्स में शेयर करें। यदि आपको यह गाइड उपयोगी लगा, तो इसे उस टीममेट के साथ शेयर करें जो अभी भी async पैटर्न से जूझ रहा है!
