# Java에서 JavaScript 실행 – Body에 요소 추가하기 완전 가이드

Java에서 **JavaScript를 실행**하고 싶지만 두 환경을 연결하는 방법을 몰라 고민한 적 있나요? 당신만 그런 것이 아닙니다. 많은 Java 개발자들이 **JavaScript DOM 조작**을 전체 브라우저를 띄우지 않고 수행하려 할 때 난관에 부딪히곤 합니다. 좋은 소식은? Aspose.HTML을 사용하면 스크립트를 실행하고, **요소를 body에 추가**하고, 변경된 마크업을 저장할 수 있습니다—모두 순수 Java 코드만으로 가능합니다.

이 튜토리얼에서는 HTML 파일을 로드하고, **JavaScript로 HTML을 수정**하는 작은 스크립트를 작성하고, Java에서 해당 스크립트를 실행한 뒤, 변경 사항을 저장하는 전체 과정을 단계별로 살펴봅니다. 마지막에는 어떤 프로젝트에든 바로 넣어 사용할 수 있는 독립 실행형 예제가 완성됩니다.

## 준비물

- **Java 8** 이상 (API는 Java 8+에서 동작)
- **Aspose.HTML for Java** 라이브러리 (버전 22.12 이상)
- 간단한 HTML 템플릿 파일 (예제에서는 코드로 생성)
- IDE 또는 텍스트 편집기—특별한 것이 필요 없으며, 일반 텍스트 편집기만 있어도 됩니다

> Pro tip: Maven을 사용한다면 `pom.xml`에 Aspose.HTML 의존성을 추가하세요. 그렇지 않다면 JAR 파일을 클래스패스에 넣으면 됩니다.

## Step 1 – Load the HTML Document (Prepare to Execute JavaScript in Java)

첫 번째로 `HTMLDocument` 인스턴스를 생성해 소스 파일을 가리키게 합니다. 이 객체는 Java와 스크립트가 나중에 접근할 DOM 트리 사이의 다리 역할을 합니다.

```java
import com.aspose.html.HTMLDocument;

// Step 1: Load the HTML document you want to manipulate
HTMLDocument htmlDoc = new HTMLDocument("YOUR_DIRECTORY/template.html");
```

> **왜 중요한가:** `HTMLDocument` 클래스는 파일을 DOM 형태로 파싱해 Aspose 스크립팅 엔진이 조작할 수 있게 합니다. 문서를 먼저 로드하지 않으면 스크립트가 수정할 대상이 없습니다.

## Step 2 – Write JavaScript that **Append Element to Body**

이제 `<p>` 요소를 만들고 `<body>`에 추가하는 작은 스니펫을 작성합니다. 전형적인 **JavaScript DOM 조작**이지만 Java에서 실행합니다.

```java
// Step 2: JavaScript that adds a new paragraph to the body
String jsCode = ""
    + "var p = document.createElement('p');"
    + "p.textContent = 'Generated by JavaScript';"
    + "document.body.appendChild(p);";
```

> **왜 동작하는가:** `document.createElement`는 새 노드를 만들고, `textContent`는 내부 텍스트를 설정하며, `appendChild`는 기존 `<body>` 요소에 붙입니다. 이 스크립트는 순수 클라이언트‑사이드 JavaScript이며, Aspose.HTML은 헤드리스 환경에서 이를 실행합니다.

## Step 3 – Create the Script Engine (The Engine That **Run JavaScript from Java**)

Aspose.HTML에는 로드된 DOM에 대해 JavaScript를 평가할 수 있는 `ScriptEngine`이 포함되어 있습니다. 바로 이 엔진이 **Java에서 JavaScript를 실행**합니다.

```java
import com.aspose.html.scripting.ScriptEngine;

// Step 3: Bind the engine to the document we just loaded
ScriptEngine scriptEngine = new ScriptEngine(htmlDoc);
```

> **왜 필요한가:** 엔진은 스크립트 실행을 격리시켜 메인 Java 프로세스에 영향을 주지 않게 합니다. 또한 `ScriptExecutionResult`를 반환해 성공 여부를 확인할 수 있습니다.

## Step 4 – Execute the Script and Check the Result

모든 준비가 끝났으면 스크립트를 실행합니다. `execute` 메서드는 오류 여부를 알려주는 결과 객체를 반환합니다.

```java
import com.aspose.html.scripting.ScriptExecutionResult;

// Step 4: Run the JavaScript and verify it succeeded
ScriptExecutionResult execResult = scriptEngine.execute(jsCode);
if (execResult.isSuccess()) {
    System.out.println("Script executed successfully.");
} else {
    System.err.println("Script failed: " + execResult.getErrorMessage());
}
```

> **예외 상황:** 스크립트가 예외를 발생시키면(예: 구문 오류) `isSuccess()`가 `false`가 되고 `getErrorMessage()`가 상세 정보를 제공합니다. 운영 코드에서는 항상 확인하세요.

## Step 5 – Save the Modified Document (Finish **Modify HTML with JavaScript**)

스크립트가 DOM을 변경한 뒤, 변경 내용을 디스크에 저장합니다. 이제 결과 파일에는 새 단락이 포함됩니다.

```java
// Step 5: Write the updated DOM back to an HTML file
htmlDoc.save("YOUR_DIRECTORY/modified.html");
System.out.println("Document saved.");
```

> **결과 확인:** `modified.html`을 브라우저에서 열면 원본 내용에 “Generated by JavaScript”라는 새로운 줄이 추가된 것을 볼 수 있습니다.

## Step 6 – Full Working Example (All Steps Combined)

아래는 `JsDomManipulation.java` 파일에 복사‑붙여넣기 할 수 있는 전체 프로그램입니다. 모든 import, 오류 처리, 주석이 포함되어 있습니다.

```java
import com.aspose.html.HTMLDocument;
import com.aspose.html.scripting.ScriptEngine;
import com.aspose.html.scripting.ScriptExecutionResult;

public class JsDomManipulation {
    public static void main(String[] args) throws Exception {

        // 1️⃣ Load the HTML document to be manipulated
        HTMLDocument htmlDoc = new HTMLDocument("YOUR_DIRECTORY/template.html");

        // 2️⃣ Define JavaScript that adds a new paragraph to the <body>
        String jsCode = "var p = document.createElement('p');"
                      + "p.textContent = 'Generated by JavaScript';"
                      + "document.body.appendChild(p);";

        // 3️⃣ Create a script engine bound to the loaded document
        ScriptEngine scriptEngine = new ScriptEngine(htmlDoc);

        // 4️⃣ Execute the JavaScript and report success
        ScriptExecutionResult execResult = scriptEngine.execute(jsCode);
        if (execResult.isSuccess()) {
            System.out.println("Script executed successfully.");
        } else {
            System.err.println("Script failed: " + execResult.getErrorMessage());
            return; // Stop if the script didn't run
        }

        // 5️⃣ Save the modified document back to HTML
        htmlDoc.save("YOUR_DIRECTORY/modified.html");
        System.out.println("Document saved.");
    }
}
```

### Expected Output

프로그램을 실행하면 다음과 같이 출력됩니다:

```
Script executed successfully.
Document saved.
```

그리고 `modified.html` 파일은 대략 다음과 같은 내용을 갖게 됩니다:

```html
<!DOCTYPE html>
<html>
<head>
    <title>Template</title>
</head>
<body>
    <h1>Original Content</h1>
    <p>Generated by JavaScript</p>
</body>
</html>
```

## Common Questions & Gotchas

| Question | Answer |
|----------|--------|
| **Can I run multiple scripts sequentially?** | Yes. Just call `scriptEngine.execute()` again with a new script string. The same `HTMLDocument` instance retains its state. |
| **What if the HTML file is missing a `<body>` tag?** | Aspose.HTML will auto‑create a `<body>` element during parsing, so the script still works. However, for predictable results, start with a well‑formed template. |
| **Is this approach compatible with Java 17?** | Absolutely. The library targets Java 8+, so newer runtimes are fine. |
| **Do I need a full browser environment?** | No. Aspose.HTML provides a lightweight, headless engine that interprets standard DOM APIs without launching Chrome or Firefox. |
| **How do I handle large HTML files?** | Load the document as a stream (`new HTMLDocument(InputStream)`) to avoid loading the entire file into memory at once. |
