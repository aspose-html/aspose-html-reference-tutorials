# Aspose HTML for Java를 사용한 동적 HTML PDF 변환

동적 HTML 페이지가 JavaScript를 사용해 테이블, 차트 또는 메뉴를 생성한다면 **동적 HTML PDF 변환**이 필요했던 적이 있나요? 혼자가 아닙니다. 실제 프로젝트에서는 HTML이 정적이지 않고 스크립트가 실행되면서 DOM 노드가 생성된 뒤에야 페이지가 원하는 형태가 됩니다.  

이 튜토리얼에서는 **스크립트가 포함된 HTML을 로드하고**, **JavaScript 실행을 기다린 뒤**, **쿼리 셀렉터로 테이블 행을 검사**하고, 마지막으로 **Aspose HTML for Java 라이브러리**를 사용해 **동적 HTML을 PDF로 변환**하는 전체 실행 가능한 예제를 단계별로 살펴봅니다. 최종적으로 Maven 또는 Gradle 프로젝트에 바로 넣어 실행할 수 있는 단일 Java 클래스를 제공할 것입니다.

> **왜 중요한가요?**  
> 스크립트 실행이 끝나기 전에 페이지를 변환하면 빈 PDF나 누락된 테이블이 생성됩니다. 여기서 소개하는 방법은 브라우저에서 사용자가 보는 그대로 PDF에 반영됩니다.

---

## 준비 사항

- **Java 17** (또는 최신 JDK; API는 Java 8+에서도 동작하지만 최신 JDK가 더 나은 성능을 제공합니다)  
- **Aspose.HTML for Java** 23.10 (이후 버전) – Maven Central에서 가져올 수 있습니다  
- **동적 HTML 파일** (`dynamic.html`에 스크립트로 테이블을 생성하도록 포함)  
- Java I/O와 Maven/Gradle에 대한 기본 지식 (깊은 전문 지식은 필요 없음)

이미 Maven `pom.xml`이 있다면 Aspose 의존성을 추가하세요:

```xml
<dependency>
    <groupId>com.aspose</groupId>
    <artifactId>aspose-html</artifactId>
    <version>23.10</version>
</dependency>
```

## 1단계: 스크립트 활성화된 HTML 로드

먼저 Aspose에 로드할 HTML에 JavaScript가 포함될 수 있음을 알려야 합니다. `HTMLLoadOptions`를 사용해 **enableScripts** 플래그를 `true`로 설정합니다.

```java
import com.aspose.html.*;
import com.aspose.html.dom.*;
import java.nio.file.Paths;

/**
 * Demonstrates loading an HTML file that contains JavaScript.
 */
public class RunJsBeforeConversion {

    public static void main(String[] args) throws Exception {

        // --------------------------------------------------------------------
        // 1️⃣ Load the HTML document with JavaScript execution enabled
        // --------------------------------------------------------------------
        // The `true` argument tells Aspose to enable script execution.
        HTMLLoadOptions loadOptions = new HTMLLoadOptions(true);

        // Adjust the path to point at your own dynamic.html file.
        HTMLDocument htmlDocument = new HTMLDocument(
                Paths.get("YOUR_DIRECTORY/dynamic.html").toUri(),
                loadOptions);
```

> **팁:** HTML 파일을 소스 폴더와 같은 위치에 두거나 절대 경로를 사용하세요. 그렇지 않으면 `toUri()` 호출이 `FileNotFoundException`을 발생시킵니다.

---

## 2단계: JavaScript 실행 대기

Aspose는 가벼운 엔진에서 스크립트를 실행하지만 페이지가 완전히 로드될 때까지 잠시 기다려야 합니다. 실제 서비스에서는 DOM‑ready 이벤트에 연결하는 것이 좋지만, 간단한 데모에서는 짧은 대기만으로 충분합니다.

```java
        // --------------------------------------------------------------------
        // 2️⃣ Wait for JavaScript execution (simple pause for demo purposes)
        // --------------------------------------------------------------------
        // 2 seconds is usually enough for tiny demos; increase if your script
        // does heavy work or makes async network calls.
        Thread.sleep(2000);
```

> **왜 대기가 필요한가요?** 라이브러리는 스크립트를 동기적으로 실행하지만 `setTimeout` 같은 작업은 큐에 쌓입니다. `Thread.sleep`을 사용해 큐가 비워질 때까지 기다려야 DOM을 정확히 검사할 수 있습니다.

---

## 3단계: 쿼리 셀렉터로 테이블 행 확인 – DOM 검증

스크립트가 실행된 뒤에는 브라우저 콘솔에서처럼 CSS 셀렉터를 사용해 요소를 가져올 수 있습니다. 여기서는 `id="report"`인 테이블을 찾아 행 수를 출력합니다.

```java
        // --------------------------------------------------------------------
        // 3️⃣ Inspect the DOM after script execution
        // --------------------------------------------------------------------
        // Using a CSS selector to find the table generated by JavaScript.
        Element reportTable = htmlDocument.querySelector("table#report");

        // Guard against a missing table – helpful when the script fails.
        if (reportTable == null) {
            System.out.println("⚠️ No table with id='report' found.");
            return;
        }

        // The Element API gives us direct access to rows collection.
        int rowCount = reportTable.getRows().getLength();
        System.out.println("Rows after script execution: " + rowCount);
```

샘플 `dynamic.html`에 대한 일반적인 출력은 다음과 같습니다:

```
Rows after script execution: 5
```

다른 숫자가 표시된다면 HTML에 포함된 스크립트를 다시 확인하세요 – 조건에 따라 행을 생성하고 있을 수 있습니다.

---

## 4단계: 최종 DOM 상태를 PDF로 변환

DOM 검증이 끝났다면 마지막 단계는 한 줄 코드입니다: `HTMLDocument`를 `Converter.convert`에 전달하면 됩니다. 결과 PDF는 스크립트 실행 후 브라우저가 렌더링하는 모습과 정확히 일치합니다.

```java
        // --------------------------------------------------------------------
        // 4️⃣ Convert the final DOM state to a PDF file
        // --------------------------------------------------------------------
        Converter.convert(htmlDocument,
                Paths.get("YOUR_DIRECTORY/dynamic.pdf").toUri());

        System.out.println("✅ PDF generated at YOUR_DIRECTORY/dynamic.pdf");
    }
}
```

`dynamic.pdf`를 아무 뷰어로 열어보면, 완전히 채워진 테이블과 스타일, 스크립트가 삽입한 이미지가 표시됩니다.

## 전체 작업 예제 (모든 단계 결합)

아래는 `src/main/java/RunJsBeforeConversion.java`에 복사해 넣을 수 있는 전체 소스 파일입니다. `YOUR_DIRECTORY`를 `dynamic.html`이 위치한 실제 폴더 경로로 바꾸세요.

```java
import com.aspose.html.*;
import com.aspose.html.dom.*;
import java.nio.file.Paths;

/**
 * End‑to‑end demo: load a dynamic HTML page, let its JavaScript run,
 * query the generated table, and convert the result to PDF.
 *
 * Requires Aspose.HTML for Java 23.10+ on the classpath.
 */
public class RunJsBeforeConversion {
    public static void main(String[] args) throws Exception {

        // 1️⃣ Load HTML with scripts enabled
        HTMLLoadOptions loadOptions = new HTMLLoadOptions(true);
        HTMLDocument htmlDocument = new HTMLDocument(
                Paths.get("YOUR_DIRECTORY/dynamic.html").toUri(),
                loadOptions);

        // 2️⃣ Wait for JavaScript execution (2 seconds)
        Thread.sleep(2000);

        // 3️⃣ Query selector table rows – verify script output
        Element reportTable = htmlDocument.querySelector("table#report");
        if (reportTable == null) {
            System.out.println("⚠️ No table with id='report' found.");
            return;
        }
        int rowCount = reportTable.getRows().getLength();
        System.out.println("Rows after script execution: " + rowCount);

        // 4️⃣ Convert the final DOM to PDF
        Converter.convert(htmlDocument,
                Paths.get("YOUR_DIRECTORY/dynamic.pdf").toUri());

        System.out.println("✅ PDF generated at YOUR_DIRECTORY/dynamic.pdf");
    }
}
```

실행 방법:

```bash
mvn compile exec:java -Dexec.mainClass=RunJsBeforeConversion
```

또는 동일한 Gradle 명령을 사용합니다.

## 흔히 발생하는 문제와 해결 방법

| Issue | Why it Happens | Fix |
|-------|----------------|-----|
| **Blank PDF** | 스크립트가 비활성화(`HTMLLoadOptions(false)`) 되었거나 대기 시간이 너무 짧음 | 스크립트를 `true`로 유지하고, 페이지가 비동기 호출을 사용한다면 `Thread.sleep` 시간을 늘리세요. |
| **`reportTable` is null** | 셀렉터가 잘못됐거나 스크립트가 요소를 생성하지 않음 | HTML의 `<script>`가 실제로 `<table id="report">`를 추가하는지 확인하세요. Chrome DevTools로 최종 DOM을 검사합니다. |
| **Missing fonts or CSS** | 외부 스타일시트가 접근 불가능 | 절대 URL을 제공하거나 CSS 파일을 로컬에 복사해 `file://` 경로로 참조하세요. |
| **Large pages cause memory spikes** | Aspose가 전체 DOM을 메모리에 로드 | `HTMLLoadOptions.setMaxMemoryUsage`로 사용량을 제한하거나 변환을 여러 청크로 나누세요. |

## 솔루션 확장하기

- **다중 페이지** – 동적 페이지가 페이지네이션을 사용한다면 URL 해시(`#page=2` 등)를 바꾼 뒤 `Converter.convert`를 루프 안에서 호출합니다.  
- **헤드리스 브라우저 대안** – WebGL처럼 매우 복잡한 스크립트는 **Playwright** 같은 실제 헤드리스 브라우저와 연동해 렌더링된 HTML을 Aspose에 전달하는 방법을 고려하세요.  
- **맞춤 렌더링 옵션** – `PdfSaveOptions`를 사용해 페이지 크기, 여백, 이미지 품질 등을 설정할 수 있습니다. 예: `new PdfSaveOptions(PdfPageSize.A4, PdfPageOrientation.Portrait)`.
