- **Ο κώδικας είναι thread‑safe;** Το αντικείμενο `Configuration` μπορεί να μοιραστεί, αλλά κάθε νήμα πρέπει να χρησιμοποιεί τη δική του παρουσία `HTMLDocument`.

## Εισαγωγή Πακέτων

import com.aspose.html.Configuration;

## Τι είναι το “handle credentials aspose html”;
Η φράση περιγράφει τη διαδικασία προσάρτησης ενός **CredentialHandler** (ή οποιουδήποτε προσαρμοσμένου `MessageHandler`) στην εσωτερική υπηρεσία δικτύου του Aspose.HTML. Αυτός ο χειριστής παρεμβαίνει στα εξερχόμενα HTTP αιτήματα, προσθέτει τις απαιτούμενες κεφαλίδες αυθεντικοποίησης και στη συνέχεια επιτρέπει στο αίτημα να προχωρήσει με ασφάλεια. Σκεφτείτε το ως έναν φρουρό ασφαλείας που ελέγχει κάθε επισκέπτη πριν εισέλθει στο κτίριο.

## Γιατί να χρησιμοποιήσετε το σωλήνα πιστοποιήσεων του Aspose.HTML;
- **Ενσωματωμένη ασφάλεια** – Δεν χρειάζεται να δημιουργείτε χειροκίνητα κεφαλίδες `Authorization` για κάθε αίτημα.  
- **Επαναχρησιμοποίηση** – Μόλις καταχωρηθεί ο χειριστής, κάθε `HTMLDocument` που δημιουργείται με την ίδια `Configuration` κληρονομεί αυτόματα τα διαπιστευτήρια.  
- **Επεκτασιμότητα** – Μπορείτε να αλυσίδωσετε πολλούς χειριστές (logging, caching, proxy) χωρίς να αλλάξετε τη λογική της επιχείρησής σας.  
- **Απόδοση** – Η βιβλιοθήκη επαναχρησιμοποιεί συνδέσεις όπου είναι δυνατόν, μειώνοντας την καθυστέρηση.

## Οδηγός Βήμα‑Βήμα

### Βήμα 1: Δημιουργία Αντικειμένου Configuration
Πρώτα, δημιουργούμε ένα αντικείμενο `Configuration`. Αυτό το αντικείμενο είναι το κεντρικό σημείο όπου το Aspose.HTML αποθηκεύει υπηρεσίες, χειριστές και άλλες επιλογές.

```java
Configuration configuration = new Configuration();
```

### Βήμα 2: Εισαγωγή του CredentialHandler στην Αλυσίδα Message Handler
Στη συνέχεια, παίρνουμε την υπηρεσία δικτύου από τη διαμόρφωση και προσθέτουμε τον προσαρμοσμένο `CredentialHandler` στην αρχή της συλλογής χειριστών. Η τοποθέτησή του στο index 0 εξασφαλίζει ότι εκτελείται πριν από οποιονδήποτε άλλο χειριστή.

```java
INetworkService service = configuration.getService(INetworkService.class);
MessageHandlerCollection handlers = service.getMessageHandlers();
handlers.insertItem(0, new CredentialHandler());
```

> **Pro tip:** Αν χρειάζεται να υποστηρίξετε πολλαπλά σχήματα αυθεντικοποίησης, μπορείτε να προσθέσετε επιπλέον χειριστές μετά τον `CredentialHandler` χωρίς να επηρεάσετε την προτεραιότητά του.

### Βήμα 3: Φόρτωση HTML Εγγράφου με τις Διαμορφωμένες Πιστοποιήσεις
Τώρα δημιουργούμε ένα `HTMLDocument` χρησιμοποιώντας τη διαμόρφωση που ετοιμάσαμε. Η URL στο παράδειγμα δείχνει σε μια δημόσια υπηρεσία δοκιμών (`httpbin.org`) που απαιτεί basic authentication.

```java
HTMLDocument document = new HTMLDocument("https://httpbin.org/basic-auth/username/securelystoredpassword", configuration);
```

### Βήμα 4: (Προαιρετικό) Ανάκτηση Περιεχομένου Εγγράφου
Αν θέλετε να εξετάσετε το ανακτημένο HTML, απλώς μετατρέψτε το έγγραφο σε συμβολοσειρά και εκτυπώστε το. Αυτό το βήμα είναι χρήσιμο για αποσφαλμάτωση ή περαιτέρω επεξεργασία με DOM APIs.

```java
String content = document.toString();
System.out.println(content);
```

### Βήμα 5: Καθαρισμός Πόρων
Πάντα απελευθερώνετε το `HTMLDocument` όταν τελειώσετε. Αυτό απελευθερώνει τους εγγενείς πόρους και αποτρέπει διαρροές μνήμης—ιδιαίτερα σημαντικό σε υπηρεσίες που τρέχουν για μεγάλο χρονικό διάστημα.

```java
document.dispose();
```

## Συχνές Ερωτήσεις

**Ε: Ποιος είναι ο σκοπός του `MessageHandlerCollection`;**  
Α: Αποθηκεύει μια αλυσίδα χειριστών που μπορούν να τροποποιούν, να καταγράφουν ή να μπλοκάρουν τα δικτυακά αιτήματα που κάνει το Aspose.HTML. Η προσθήκη ενός `CredentialHandler` σε αυτή τη συλλογή ενεργοποιεί την αυτόματη αυθεντικοποίηση.

**Ε: Μπορώ να χρησιμοποιήσω OAuth tokens αντί για basic auth;**  
Ν: Απόλυτα. Υλοποιήστε έναν προσαρμοσμένο χειριστή που προσθέτει την κεφαλίδα `Authorization: Bearer <token>` και εισάγετέ τον στη συλλογή όπως κάνετε με τον `CredentialHandler`.

**Ε: Τα διαπιστευτήρια αποθηκεύονται ως απλό κείμενο;**  
Α: Το παράδειγμα χρησιμοποιεί έναν απλό χειριστή για λόγους επεξήγησης. Σε παραγωγικό περιβάλλον, αποθηκεύετε τα μυστικά με ασφαλή τρόπο (π.χ. Java Keystore, Azure Key Vault) και τα ανακτάτε κατά την εκτέλεση.

**Ε: Υποστηρίζει το Aspose.HTML αυθεντικοποίηση proxy;**  
Α: Ναι. Μπορείτε να προσθέσετε έναν ξεχωριστό `ProxyHandler` στην ίδια `MessageHandlerCollection` και να τον διαμορφώσετε με τα διαπιστευτήρια του proxy.

**Ε: Πώς μπορώ να εντοπίσω την κίνηση δικτύου;**  
Α: Προσθέστε έναν logging handler (π.χ. `new LoggingHandler()`) μετά τον credential handler για να καταγράψετε λεπτομέρειες αιτήματος/απάντησης.
