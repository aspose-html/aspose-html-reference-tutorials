# Πώς να ορίσετε DPI κατά τη μετατροπή SVG σε PNG με Java

Έχετε αναρωτηθεί ποτέ **πώς να ορίσετε DPI** για μια μετατροπή SVG‑σε‑PNG ώστε το αποτέλεσμα να φαίνεται καθαρό σε ένα αφίσα έτοιμη για εκτύπωση; Δεν είστε ο μόνος. Σε πολλά έργα—σκεφτείτε φυλλάδια μάρκετινγκ, προεπισκοπήσεις UI ή τεχνικά διαγράμματα—η λήψη ενός PNG υψηλής ανάλυσης από ένα SVG είναι απαραίτητη.  

Σε αυτό το tutorial θα περάσουμε βήμα προς βήμα τις ακριβείς ενέργειες για **μετατροπή SVG σε PNG**, **εξαγωγή PNG υψηλής ανάλυσης**, και, το πιο σημαντικό, έλεγχο του DPI χρησιμοποιώντας τη βιβλιοθήκη Aspose.HTML for Java. Στο τέλος θα έχετε ένα επαναχρησιμοποιήσιμο απόσπασμα κώδικα που μπορείτε να ενσωματώσετε σε οποιαδήποτε εφαρμογή Java, ανεξάρτητα αν δημιουργείτε ένα εργαλείο επιφάνειας εργασίας ή μια υπηρεσία backend.

## Τι Θα Χρειαστεί

- **Java 8+** (ο κώδικας μεταγλωττίζεται με οποιοδήποτε πρόσφατο JDK)
- **Aspose.HTML for Java** JARs (διαθέσιμα από το Maven Central ή την ιστοσελίδα Aspose)
- Ένα αρχείο SVG που θέλετε να rasterize  
- Μια μικρή δόση περιέργειας—τίποτα άλλο.

Αν είστε άνετοι με το Maven, απλώς προσθέστε την εξάρτηση που φαίνεται στην επόμενη ενότητα· διαφορετικά, κατεβάστε τα JAR χειροκίνητα και προσθέστε τα στο classpath σας.

## Βήμα 1: Προσθέστε τη Βιβλιοθήκη Java Image Conversion

Πρώτα απ' όλα—το έργο σας χρειάζεται τη **java image conversion library** που πραγματικά ξέρει πώς να διαβάσει SVG και να γράψει PNG. Η Aspose.HTML είναι μια αξιόπιστη επιλογή επειδή διαχειρίζεται CSS, γραμματοσειρές και σύνθετα διανυσματικά χαρακτηριστικά αμέσως.

**Χρήστες Maven** μπορούν να επικολλήσουν αυτό στο `pom.xml` σας:

```xml
<dependency>
    <groupId>com.aspose</groupId>
    <artifactId>aspose-html</artifactId>
    <version>23.12</version> <!-- use the latest stable version -->
</dependency>
```

**Χρήστες Gradle** μπορούν να προσθέσουν:

```gradle
implementation 'com.aspose:aspose-html:23.12'
```

Αν προτιμάτε τη χειροκίνητη προσέγγιση, κατεβάστε το JAR από τη σελίδα λήψης του Aspose και τοποθετήστε το στο `libs/`, έπειτα προσθέστε το στη διαδρομή κατασκευής του IDE σας.

> **Pro tip:** Παρακολουθείτε τον αριθμό έκδοσης· οι νεότερες εκδόσεις συχνά βελτιώνουν τη διαχείριση DPI και διορθώνουν σφάλματα σε ειδικές περιπτώσεις.

## Βήμα 2: Δημιουργήστε τις Επιλογές Μετατροπής και Ορίστε το Επιθυμητό DPI

Τώρα που η βιβλιοθήκη είναι ενσωματωμένη, πρέπει να της πούμε *πώς* θέλουμε να φαίνεται το PNG εξόδου. Εδώ μπαίνει σε παιχνίδι η κύρια λέξη‑κλειδί—**πώς να ορίσετε DPI**. Η κλάση `ImageConversionOptions` σας παρέχει λεπτομερή έλεγχο τόσο της οριζόντιας (`dpiX`) όσο και της κάθετης (`dpiY`) ανάλυσης.

```java
import com.aspose.html.converters.ImageConversionOptions;

// Step 2: Build the options object and set DPI
ImageConversionOptions conversionOptions = new ImageConversionOptions();
conversionOptions.setDpiX(300); // 300 DPI horizontally
conversionOptions.setDpiY(300); // 300 DPI vertically
```

Γιατί 300 DPI; Για τις περισσότερες διαδικασίες εκτύπωσης, 300 σημεία‑ανά‑ίντσα θεωρούνται «ποιότητα εκτύπωσης». Αν στοχεύετε μόνο σε web πόρους, μπορείτε να μειώσετε σε 72 ή 96 DPI για να διατηρήσετε μικρά τα μεγέθη αρχείων.

> **Τι γίνεται αν χρειάζομαι διαφορετικό DPI για το πλάτος και το ύψος;**  
> Απλώς αλλάξτε τα `setDpiX` και `setDpiY` ανεξάρτητα. Η βιβλιοθήκη σέβεται τις μη‑ομοιόμορφες τιμές, κάτι που είναι χρήσιμο για αναμορφωτικές εικόνες.

## Βήμα 3: Εκτελέστε τη Μετατροπή SVG‑σε‑PNG

Με τις επιλογές έτοιμες, η πραγματική μετατροπή είναι μια ενιαία στατική κλήση. Θα χρησιμοποιήσουμε το `java.nio.file.Paths` για να διατηρήσουμε την ανεξαρτησία πλατφόρμας.

```java
import com.aspose.html.converters.Converter;
import java.nio.file.Paths;

public class SvgToPng {
    public static void main(String[] args) throws Exception {
        // Paths to input SVG and output PNG
        var inputSvg  = Paths.get("YOUR_DIRECTORY/input.svg").toUri();
        var outputPng = Paths.get("YOUR_DIRECTORY/output.png").toUri();

        // Convert using the DPI settings we defined earlier
        Converter.convert(inputSvg, outputPng, conversionOptions);
    }
}
```

Αυτό είναι—εκτελέστε τη μέθοδο `main` και θα βρείτε ένα καθαρό PNG 300 DPI στο `YOUR_DIRECTORY`. Η βιβλιοθήκη αυτόματα κλιμακώνει τα διανυσματικά γραφικά ώστε να ταιριάζουν με το DPI που ορίσατε, διατηρώντας την αναλογία διαστάσεων και την ευκρίνεια του κειμένου.

## Βήμα 4: Επαληθεύστε το Αποτέλεσμα (Προαιρετικό αλλά Συνιστώμενο)

Μια γρήγορη έλεγχος λογικής μπορεί να σας σώσει από προβλήματα αργότερα. Ανοίξτε το παραγόμενο PNG σε οποιονδήποτε προβολέα εικόνας που εμφανίζει μεταδεδομένα DPI (π.χ., Photoshop, GIMP ή ακόμη και την καρτέλα «Λεπτομέρειες» του Windows Explorer). Θα πρέπει να δείτε **300 DPI** καταχωρημένο κάτω από την οριζόντια και κάθετη ανάλυση.

Αν το αρχείο φαίνεται θολό, ελέγξτε ξανά:

1. Το αρχικό SVG δεν είναι χαμηλής ανάλυσης από την αρχή (τα διανυσματικά αρχεία είναι ανεξάρτητα από ανάλυση, αλλά οι ενσωματωμένες bitmap εικόνες μέσα σε αυτά μπορούν να περιορίσουν την ποιότητα).  
2. Έχετε πράγματι αποθηκεύσει το αρχείο μετά τον ορισμό του DPI—μερικές φορές τα IDE αποθηκεύουν στην cache παλιές εκδόσεις.  
3. Οι επιλογές μετατροπής δεν έχουν αντικατασταθεί αλλού στον κώδικά σας.

## Γιατί το DPI είναι Σημαντικό για την Εξαγωγή PNG Υψηλής Ανάλυσης

Μπορεί να αναρωτιέστε, «Γιατί να ασχοληθώ με το DPI; Δεν είναι το PNG απλώς εικονοστοιχεία;» Η αλήθεια είναι ότι το DPI είναι μια ετικέτα *metadata* που λέει στις επόμενες εφαρμογές (ειδικά στις εκτυπώσεις) πόσα εικονοστοιχεία αντιστοιχούν σε μια ίντσα φυσικού χώρου. Αν δώσετε ένα PNG 1200 × 800 σε έναν εκτυπωτή χωρίς σωστά metadata DPI, ο εκτυπωτής μπορεί να υποθέσει προεπιλογή (συχνά 72 DPI) και να μεγεθύνει την εικόνα, με αποτέλεσμα θολό αποτέλεσμα.

Η σωστή ρύθμιση του DPI είναι επίσης κέρδος από άποψη απόδοσης: αποφεύγετε την ανάγκη επανακλιμάκωσης raster εικόνων αργότερα, κάτι που μπορεί να είναι υπολογιστικά δαπανηρό και να μειώσει την ποιότητα.

## Περιπτώσεις Ορίων & Συνηθισμένα Πιθανά Σφάλματα

| Κατάσταση | Τι να Προσέξετε | Πώς να Διορθώσετε |
|-----------|-------------------|------------|
| **Το SVG περιέχει ενσωματωμένες bitmap εικόνες** | Η ενσωματωμένη bitmap μπορεί να είναι χαμηλής ανάλυσης, προκαλώντας το τελικό PNG να φαίνεται ψήγματο ακόμη και σε υψηλό DPI. | Αντικαταστήστε τη bitmap με μια έκδοση υψηλότερης ανάλυσης ή επεξεργαστείτε το SVG ώστε να αναφέρεται σε εξωτερική εικόνα. |
| **Μεγάλες τιμές DPI (π.χ., 1200 DPI)** | Η κατανάλωση μνήμης αυξάνεται απότομα· μπορεί να προκύψει `OutOfMemoryError`. | Αυξήστε το μέγεθος heap της JVM (`-Xmx2g`) ή περιορίστε το DPI σε ένα λογικό μέγιστο για τη χρήση σας. |
| **Μη‑τετράγωνα pixel** | Κάποιες οθόνες ερμηνεύουν το DPI διαφορετικά, οδηγώντας σε τεντωμένες εικόνες. | Βεβαιωθείτε ότι το `setDpiX` ισούται με το `setDpiY` εκτός αν έχετε συγκεκριμένο λόγο για διαφορετικές τιμές. |
| **Απουσία γραμματοσειρών** | Το κείμενο στο SVG μπορεί να επιστρέψει σε προεπιλεγμένη γραμματοσειρά, αλλάζοντας τη διάταξη. | Ενσωματώστε τις γραμματοσειρές στο SVG ή εγκαταστήστε τις απαιτούμενες γραμματοσειρές στο μηχάνημα εκτέλεσης. |

## Σύντομη Ανακεφαλαίωση (σε Μία Πρόταση)

Για **πώς να ορίσετε DPI** σε μια μετατροπή SVG‑σε‑PNG, δημιουργήστε ένα αντικείμενο `ImageConversionOptions`, ορίστε `dpiX`/`dpiY` και καλέστε το `Converter.convert` από τη βιβλιοθήκη Aspose.HTML Java.

## Επόμενα Βήματα & Σχετικά Θέματα

- **Batch processing:** Επανάληψη σε έναν φάκελο SVG αρχείων και εφαρμογή των ίδιων ρυθμίσεων DPI.  
- **Dynamic DPI:** Ανάγνωση αρχείου ρυθμίσεων ή παραμέτρου γραμμής εντολών για ορισμό DPI κατά την εκτέλεση.  
- **Alternative libraries:** Αν δεν μπορείτε να χρησιμοποιήσετε την Aspose, σκεφτείτε **Batik** (Apache) ή **SVG Salamander**, αν και μπορεί να απαιτούν επιπλέον κώδικα για διαχείριση DPI.  
- **Export high resolution PNG** for Android assets: συνδυάστε αυτήν την τεχνική με τους φακέλους Android `drawable‑hdpi`, `xhdpi`, κ.λπ.

Μη διστάσετε να πειραματιστείτε—δοκιμάστε 72 DPI για μικρογραφίες web, 600 DPI για εκτυπώσεις μεγάλου φορμάτ, ή ακόμη 150 DPI ως ενδιάμεσο. Ο κώδικας παραμένει ίδιος· μόνο οι αριθμοί αλλάζουν.
