# Εξαγωγή κειμένου από HTML – Πλήρης Οδηγός Βήμα‑βήμα

Έχετε ποτέ χρειαστεί να **εξάγετε κείμενο από HTML** αλλά δεν ήξερες ποιο API να επιλέξεις; Δεν είστε μόνοι. Πολλοί προγραμματιστές Java κοιτάζουν ένα τεράστιο `<div>` και αναρωτιούνται πώς να πάρουν μόνο τις αναγνώσιμες λέξεις, ειδικά όταν το έγγραφο είναι σελίδωση.  

Σε αυτό το σεμινάριο θα σας δείξουμε ακριβώς **πώς να λάβετε το κείμενο μιας σελίδας** από ένα σελιδοποιημένο αρχείο HTML χρησιμοποιώντας το Aspose.HTML for Java. Στο τέλος θα μπορείτε να **εξάγετε το περιεχόμενο σελίδας HTML**, να φορτώσετε το έγγραφο και να εκτυπώσετε το κείμενο οποιασδήποτε σελίδας επιλέγετε — χωρίς επιπλέον τεχνάσματα ανάλυσης.

Θα καλύψουμε όλα όσα χρειάζεστε: την απαιτούμενη εξάρτηση Maven, τη φόρτωση του αρχείου, τη ρύθμιση του extractor, τη διαχείριση ειδικών περιπτώσεων όπως η απουσία σελίδων, και τον καθαρισμό πόρων. Αν έχετε ήδη ένα έργο Java και ένα τοπικό αρχείο HTML, μπορείτε να ακολουθήσετε αμέσως.  

**Προαπαιτούμενα** – JDK 8 ή νεότερο, Maven (ή Gradle) και ένα αντίγραφο του Aspose.HTML for Java JAR. Δεν χρειάζονται άλλες βιβλιοθήκες.

## Τι Θα Επιτύχετε

- Φορτώστε ένα έγγραφο HTML σε Java (`load html document java`).
- Επιλέξτε έναν συγκεκριμένο αριθμό σελίδας.
- Εξάγετε το αποδοθέν κείμενο ακριβώς όπως θα το εμφανίσει ένας φυλλομετρητής.
- Εκτυπώστε το αποτέλεσμα στην κονσόλα.
- Κατανοήστε πώς να προσαρμόσετε τον extractor για διαφορετικά σενάρια.

## 📦 Προσθήκη Aspose.HTML στο Έργο σας

Αν χρησιμοποιείτε Maven, προσθέστε την παρακάτω εξάρτηση στο `pom.xml` σας. Για Gradle, οι ίδιες συντεταγμένες λειτουργούν με τη ρύθμιση `implementation`.

```xml
<!-- Maven dependency for Aspose.HTML -->
<dependency>
    <groupId>com.aspose</groupId>
    <artifactId>aspose-html</artifactId>
    <version>23.12</version> <!-- latest as of Feb 2026 -->
</dependency>
```

> **Συμβουλή:** Ελέγχετε πάντα τις επίσημες σημειώσεις έκδοσης του Aspose.HTML για διορθώσεις σφαλμάτων που επηρεάζουν την εξαγωγή κειμένου.

## Βήμα 1 – Φόρτωση του Εγγράφου HTML

Το πρώτο πράγμα που κάνετε όταν θέλετε να **εξάγετε κείμενο από HTML** είναι να φορτώσετε το αρχείο σε ένα αντικείμενο `HtmlDocument`. Αυτή η κλάση αναλύει τη σήμανση, δημιουργεί ένα DOM και προετοιμάζει τη μηχανή διάταξης.

```java
import com.aspose.html.*;

public class PageTextExample {
    public static void main(String[] args) throws Exception {
        // Load the HTML file from your local folder
        // Replace YOUR_DIRECTORY with the actual path on your machine
        HtmlDocument htmlDoc = new HtmlDocument(
            "YOUR_DIRECTORY/paginated.html",
            new LoadOptions()
        );
```

Γιατί να χρησιμοποιήσετε `LoadOptions`; Σας επιτρέπει να ελέγχετε στοιχεία όπως η κωδικοποίηση χαρακτήρων και η φόρτωση πόρων, κάτι που μπορεί να είναι κρίσιμο όταν το HTML αναφέρεται σε εξωτερικά CSS ή εικόνες.

## Βήμα 2 – Δημιουργία TextExtractor

`TextExtractor` είναι ο κύριος μηχανισμός που διασχίζει τη αποδοθέν διάταξη και εξάγει τους ορατούς χαρακτήρες. Σκεφτείτε το ως την εντολή “copy‑text” που θα χρησιμοποιούσατε σε έναν φυλλομετρητή.

```java
        // Initialise the extractor for the loaded document
        TextExtractor textExtractor = new TextExtractor(htmlDoc);
```

Μπορείτε επίσης να επαναχρησιμοποιήσετε τον ίδιο extractor για πολλές σελίδες, αλλά η δημιουργία ενός νέου κάθε φορά κρατά τη διαχείριση της κατάστασης απλή.

## Βήμα 3 – Ρύθμιση του Extractor (Επιλογή Σελίδας & Αποδοθέν Κείμενο)

Τώρα λέμε στον extractor **πώς να λάβει το κείμενο μιας σελίδας**. Οι αριθμοί σελίδων ξεκινούν από 1, έτσι το `2` σημαίνει “η δεύτερη εκτυπωμένη σελίδα”.

```java
        // Choose which page to extract (1‑based index)
        textExtractor.setPageNumber(2);

        // Ask for computed (rendered) text, not just raw DOM strings
        textExtractor.setExtractComputed(true);
```

Η ρύθμιση `setExtractComputed(true)` είναι απαραίτητη όταν η σελίδα περιέχει περιεχόμενο που δημιουργείται από CSS ή placeholders γεμάτα από JavaScript — χωρίς αυτήν θα βλέπατε μόνο τη γυμνή σήμανση.

## Βήμα 4 – Εκτέλεση της Εξαγωγής

Με όλα ρυθμισμένα, η πραγματική εξαγωγή είναι μια εντολή μίας γραμμής.

```java
        // Extract the text for the selected page
        String pageTextContent = textExtractor.extract();
```

Αν η ζητούμενη σελίδα δεν υπάρχει, το `extract()` επιστρέφει μια κενή συμβολοσειρά. Μπορείτε να το προστατέψετε ελέγχοντας πρώτα το `htmlDoc.getPageCount()`.

```java
        if (textExtractor.getPageNumber() > htmlDoc.getPageCount()) {
            System.out.println("Requested page is out of range.");
        } else {
            System.out.println("Page 2 text:");
            System.out.println(pageTextContent);
        }
```

**Αναμενόμενη έξοδος κονσόλας** (περιορισμένη για συντομία):

```
Page 2 text:
Welcome to the second chapter…
Lorem ipsum dolor sit amet, consectetur…
```

Θα παρατηρήσετε ότι οι αλλαγές γραμμής ταιριάζουν με τη οπτική διάταξη, όχι με τον αρχικό κώδικα πηγής.

## Βήμα 5 – Καθαρισμός Πόρων

Το Aspose.HTML χρησιμοποιεί εγγενείς πόρους, έτσι πρέπει πάντα να τα απελευθερώνετε όταν τελειώσετε. Η παράλειψη αυτού του βήματος μπορεί να οδηγήσει σε διαρροές μνήμης, ειδικά σε υπηρεσίες που τρέχουν για μεγάλο χρονικό διάστημα.

```java
        // Release native resources
        textExtractor.dispose();
        htmlDoc.dispose();
    }
}
```

## Διαχείριση Συνηθισμένων Ειδικών Περιπτώσεων

| Κατάσταση | Τι να κάνετε |
|-----------|------------|
| **HTML περιέχει εξωτερικό CSS** | Περνάτε ένα `LoadOptions` με `setBaseUri` που δείχνει στο φάκελο που περιέχει τα αρχεία CSS. |
| **Ο αριθμός σελίδας είναι δυναμικός** | Ελέγξτε πρώτα το `htmlDoc.getPageCount()` και περιορίστε τον ζητούμενο αριθμό. |
| **Χρειάζεστε απλό κείμενο από ολόκληρο το έγγραφο** | Παραλείψτε το `setPageNumber` ή ορίστε το σε `1` και επαναλάβετε μέχρι το `getPageCount()`. |
| **Μεγάλα αρχεία προκαλούν OutOfMemoryError** | Επεξεργαστείτε τις σελίδες μία τη φορά και απελευθερώστε τον extractor μετά από κάθε επανάληψη. |

## Εναλλακτική: Εξαγωγή Ολόκληρου του Εγγράφου Μιας Στιγμής

Αν δεν σας ενδιαφέρει η σελιδοποίηση, απλώς παραλείψτε την κλήση `setPageNumber`:

```java
TextExtractor extractor = new TextExtractor(htmlDoc);
extractor.setExtractComputed(true);
String allText = extractor.extract();
System.out.println(allText);
```

Αυτό σας δίνει το πλήρες **extract html page content** σε μία ενέργεια.

Το ίδιο μοτίβο λειτουργεί για εξαγωγή ολόκληρου του εγγράφου, και τώρα ξέρετε πώς να διαχειριστείτε ελλιπείς σελίδες, εξωτερικούς πόρους και μεγάλα αρχεία.

## Τι Ακολουθεί;

- Ομαδική εξαγωγή: Επανάληψη σε όλες τις σελίδες και εγγραφή της κάθε μιας σε ξεχωριστό αρχείο `.txt`.
- Δείκτης αναζήτησης: Εισάγετε τις εξαγόμενες συμβολοσειρές στο Lucene ή Elasticsearch για αναζήτηση πλήρους κειμένου.
- Μετατροπή σε PDF: Συνδυάστε το `TextExtractor` με το Aspose.PDF για να δημιουργήσετε PDF με δυνατότητα αναζήτησης απευθείας από HTML.

Μη διστάσετε να πειραματιστείτε με το `setExtractComputed(false)` για να δείτε το ακατέργαστο κείμενο DOM, ή να ρυθμίσετε το `LoadOptions` για προσαρμοσμένες συμβολοσειρές user‑agent όταν φορτώνετε απομακρυσμένα URLs.

### Συχνές Ερωτήσεις

**Ε: Λειτουργεί αυτό με HTML που φορτώνεται από URL;**  
Α: Ναι. Αντικαταστήστε τη διαδρομή αρχείου με τη συμβολοσειρά URL και προαιρετικά ορίστε `LoadOptions.setResourceLoading(true)`.

**Ε: Μπορώ να εξάγω κείμενο από ένα συγκεκριμένο στοιχείο αντί για ολόκληρη τη σελίδα;**  
Α: Χρησιμοποιήστε `HtmlDocument.getElementById` για να εντοπίσετε το στοιχείο, μετά δημιουργήστε έναν `TextExtractor` για αυτό το υπο‑έγγραφο.

**Ε: Υπάρχει όριο στο μέγεθος της σελίδας;**  
Α: Δεν υπάρχει πραγματικά, αλλά εξαιρετικά μεγάλες σελίδες μπορεί να αυξήσουν τη χρήση μνήμης. Επεξεργαστείτε τις σταδιακά αν αντιμετωπίσετε προβλήματα απόδοσης.
