# Εκτέλεση JavaScript σε Java – Πλήρης Οδηγός για Προσθήκη Στοιχείων στο Body

Έχετε χρειαστεί ποτέ να **εκτελέσετε JavaScript σε Java** αλλά δεν ήξερατε πώς να γεφυρώσετε τα δύο περιβάλλοντα; Δεν είστε μόνοι. Πολλοί προγραμματιστές Java συναντούν εμπόδια όταν θέλουν να κάνουν **JavaScript DOM manipulation** χωρίς να ανοίξουν έναν πλήρη περιηγητή. Τα καλά νέα; Με το Aspose.HTML μπορείτε να τρέξετε ένα script, **να προσθέσετε στοιχείο στο body**, και να αποθηκεύσετε το τροποποιημένο markup—όλα από καθαρό κώδικα Java.

Σε αυτό το tutorial θα καλύψουμε όλα όσα χρειάζεστε: φόρτωση αρχείου HTML, συγγραφή ενός μικρού script που **τροποποιεί HTML με JavaScript**, εκτέλεση του script από Java, και τέλος αποθήκευση των αλλαγών. Στο τέλος θα έχετε ένα αυτόνομο, εκτελέσιμο παράδειγμα που μπορείτε να ενσωματώσετε σε οποιοδήποτε έργο.

## Τι Θα Χρειαστείτε

- **Java 8** ή νεότερη (το API λειτουργεί με Java 8+)
- **Aspose.HTML for Java** βιβλιοθήκη (έκδοση 22.12 ή νεότερη)
- Ένα απλό αρχείο προτύπου HTML (θα δημιουργήσουμε ένα εν κινήσει)
- Ένα IDE ή κειμενογράφο—δεν χρειάζεται τίποτα περίπλοκο, ακόμη και ένας απλός επεξεργαστής κειμένου αρκεί

> Συμβουλή επαγγελματία: Αν χρησιμοποιείτε Maven, προσθέστε την εξάρτηση Aspose.HTML στο `pom.xml`. Διαφορετικά, απλώς τοποθετήστε τα JARs στην classpath σας.

## Βήμα 1 – Φόρτωση του HTML Εγγράφου (Προετοιμασία για Εκτέλεση JavaScript σε Java)

Το πρώτο που κάνουμε είναι να δημιουργήσουμε μια παρουσία `HTMLDocument` που δείχνει στο αρχείο προέλευσης. Σκεφτείτε αυτό το αντικείμενο ως τη γέφυρα μεταξύ Java και του δέντρου DOM που θα επηρεάσει το script.

```java
import com.aspose.html.HTMLDocument;

// Step 1: Load the HTML document you want to manipulate
HTMLDocument htmlDoc = new HTMLDocument("YOUR_DIRECTORY/template.html");
```

> **Γιατί είναι σημαντικό:** Η κλάση `HTMLDocument` αναλύει το αρχείο σε μια αναπαράσταση DOM που η μηχανή σκριπτ του Aspose μπορεί να αλληλεπιδράσει. Χωρίς τη φόρτωση του εγγράφου πρώτα, δεν υπάρχει τίποτα για το script να τροποποιήσει.

## Βήμα 2 – Γράψτε JavaScript που **Προσθέτει Στοιχείο στο Body**

Τώρα δημιουργούμε ένα μικρό απόσπασμα που δημιουργεί ένα στοιχείο `<p>` και το προσθέτει στο `<body>`. Πρόκειται για κλασική **JavaScript DOM manipulation**, αλλά θα το τρέξουμε από Java.

```java
// Step 2: JavaScript that adds a new paragraph to the body
String jsCode = ""
    + "var p = document.createElement('p');"
    + "p.textContent = 'Generated by JavaScript';"
    + "document.body.appendChild(p);";
```

> **Γιατί λειτουργεί:** Η `document.createElement` δημιουργεί έναν νέο κόμβο, το `textContent` ορίζει το εσωτερικό του κείμενο, και η `appendChild` τον συνδέει με το υπάρχον στοιχείο `<body>`. Το script είναι καθαρό client‑side JavaScript· το Aspose.HTML το εκτελεί σε περιβάλλον χωρίς γραφικό περιβάλλον (headless).

## Βήμα 3 – Δημιουργία της Μηχανής Σκριπτ (Η Μηχανή που **Τρέχει JavaScript από Java**)

Το Aspose.HTML περιλαμβάνει μια `ScriptEngine` που μπορεί να αξιολογήσει JavaScript έναντι του φορτωμένου DOM. Είναι το κομμάτι που πραγματικά **τρέχει JavaScript από Java**.

```java
import com.aspose.html.scripting.ScriptEngine;

// Step 3: Bind the engine to the document we just loaded
ScriptEngine scriptEngine = new ScriptEngine(htmlDoc);
```

> **Γιατί το χρειάζεστε:** Η μηχανή απομονώνει την εκτέλεση του script, αποτρέποντας την επίδρασή του στη κύρια διαδικασία Java. Επίσης επιστρέφει ένα `ScriptExecutionResult` ώστε να μπορείτε να επαληθεύσετε την επιτυχία.

## Βήμα 4 – Εκτέλεση του Script και Έλεγχος του Αποτελέσματος

Με όλα έτοιμα, τελικά ενεργοποιούμε το script. Η μέθοδος `execute` επιστρέφει ένα αντικείμενο αποτελέσματος που μας λέει αν το script εκτελέστηκε χωρίς σφάλματα.

```java
import com.aspose.html.scripting.ScriptExecutionResult;

// Step 4: Run the JavaScript and verify it succeeded
ScriptExecutionResult execResult = scriptEngine.execute(jsCode);
if (execResult.isSuccess()) {
    System.out.println("Script executed successfully.");
} else {
    System.err.println("Script failed: " + execResult.getErrorMessage());
}
```

> **Ακραία περίπτωση:** Αν το script πετάξει εξαίρεση (π.χ. σφάλμα σύνταξης), το `isSuccess()` θα είναι `false` και το `getErrorMessage()` παρέχει τις λεπτομέρειες. Πάντα ελέγχετε αυτόν τον δείκτη σε κώδικα παραγωγής.

## Βήμα 5 – Αποθήκευση του Τροποποιημένου Εγγράφου (Ολοκλήρωση **Τροποποίησης HTML με JavaScript**)

Αφού το script μεταβλήθηκε το DOM, αποθηκεύουμε τις αλλαγές στο δίσκο. Το αρχείο που προκύπτει περιέχει τώρα την καινούρια παράγραφο.

```java
// Step 5: Write the updated DOM back to an HTML file
htmlDoc.save("YOUR_DIRECTORY/modified.html");
System.out.println("Document saved.");
```

> **Επαλήθευση αποτελέσματος:** Ανοίξτε το `modified.html` σε οποιονδήποτε περιηγητή. Θα πρέπει να δείτε το αρχικό περιεχόμενο συν μια νέα γραμμή που γράφει “Generated by JavaScript”.

## Βήμα 6 – Πλήρες Παράδειγμα Εργασίας (Όλα τα Βήματα Συνδυασμένα)

Παρακάτω βρίσκεται το πλήρες πρόγραμμα που μπορείτε να αντιγράψετε‑και‑επικολλήσετε σε ένα αρχείο `JsDomManipulation.java`. Περιλαμβάνει όλες τις εισαγωγές, τη διαχείριση σφαλμάτων, και σχόλια για σαφήνεια.

```java
import com.aspose.html.HTMLDocument;
import com.aspose.html.scripting.ScriptEngine;
import com.aspose.html.scripting.ScriptExecutionResult;

public class JsDomManipulation {
    public static void main(String[] args) throws Exception {

        // 1️⃣ Load the HTML document to be manipulated
        HTMLDocument htmlDoc = new HTMLDocument("YOUR_DIRECTORY/template.html");

        // 2️⃣ Define JavaScript that adds a new paragraph to the <body>
        String jsCode = "var p = document.createElement('p');"
                      + "p.textContent = 'Generated by JavaScript';"
                      + "document.body.appendChild(p);";

        // 3️⃣ Create a script engine bound to the loaded document
        ScriptEngine scriptEngine = new ScriptEngine(htmlDoc);

        // 4️⃣ Execute the JavaScript and report success
        ScriptExecutionResult execResult = scriptEngine.execute(jsCode);
        if (execResult.isSuccess()) {
            System.out.println("Script executed successfully.");
        } else {
            System.err.println("Script failed: " + execResult.getErrorMessage());
            return; // Stop if the script didn't run
        }

        // 5️⃣ Save the modified document back to HTML
        htmlDoc.save("YOUR_DIRECTORY/modified.html");
        System.out.println("Document saved.");
    }
}
```

### Αναμενόμενη Έξοδος

Η εκτέλεση του προγράμματος εμφανίζει:

```
Script executed successfully.
Document saved.
```

Και το `modified.html` περιέχει κάτι σαν:

```html
<!DOCTYPE html>
<html>
<head>
    <title>Template</title>
</head>
<body>
    <h1>Original Content</h1>
    <p>Generated by JavaScript</p>
</body>
</html>
```

## Συχνές Ερωτήσεις & Παγίδες

| Ερώτηση | Απάντηση |
|----------|--------|
| **Μπορώ να τρέξω πολλαπλά scripts διαδοχικά;** | Ναι. Απλώς καλέστε ξανά `scriptEngine.execute()` με ένα νέο string script. Η ίδια παρουσία `HTMLDocument` διατηρεί την κατάσταση. |
| **Τι γίνεται αν το αρχείο HTML λείπει το στοιχείο `<body>`;** | Το Aspose.HTML θα δημιουργήσει αυτόματα ένα στοιχείο `<body>` κατά την ανάλυση, οπότε το script θα λειτουργήσει. Ωστόσο, για προβλέψιμα αποτελέσματα, ξεκινήστε με ένα καλά δομημένο πρότυπο. |
| **Είναι αυτή η προσέγγιση συμβατή με Java 17;** | Απόλυτα. Η βιβλιοθήκη στοχεύει σε Java 8+, οπότε οι νεότερες εκδόσεις λειτουργούν κανονικά. |
| **Χρειάζομαι πλήρες περιβάλλον περιηγητή;** | Όχι. Το Aspose.HTML παρέχει μια ελαφριά, headless μηχανή που ερμηνεύει τα τυπικά APIs DOM χωρίς να εκκινεί Chrome ή Firefox. |
| **Πώς διαχειρίζομαι μεγάλα αρχεία HTML;** | Φορτώστε το έγγραφο ως ροή (`new HTMLDocument(InputStream)`) για να αποφύγετε τη φόρτωση ολόκληρου του αρχείου στη μνήμη ταυτόχρονα. |

## Επόμενα Βήματα

- **Διαπέραση του DOM** για τροποποίηση υπαρχόντων κόμβων βάσει χαρακτηριστικών.
- **Εκτέλεση εξωτερικών scripts** που λαμβάνονται από URL (χρησιμοποιήστε `scriptEngine.executeFromUrl()` αν χρειάζεται).
- Συνδυάστε αυτήν την τεχνική με **μετατροπή σε PDF**—το Aspose.HTML μπορεί να αποδώσει το τελικό DOM σε PDF με μία κλήση.

Αν σας ενδιαφέρει το **run JavaScript from Java** για δοκιμές ή server‑side rendering, σκεφτείτε να τυλίξετε τη λογική σε μια βοηθητική κλάση και να την επαναχρησιμοποιήσετε σε μικρο‑υπηρεσίες.
