# Jalankan JavaScript di Java – Panduan Lengkap Menambahkan Elemen ke Body

Pernahkah Anda perlu **execute JavaScript in Java** tetapi tidak yakin cara menjembatani kedua dunia? Anda bukan satu-satunya. Banyak pengembang Java mengalami kebuntuan ketika ingin melakukan **JavaScript DOM manipulation** tanpa membuka browser penuh. Kabar baik? Dengan Aspose.HTML Anda dapat menjalankan skrip, **append element to body**, dan menyimpan markup yang diubah—semua dari kode Java murni.

Dalam tutorial ini kami akan membahas semua yang perlu Anda ketahui: memuat file HTML, menulis skrip kecil yang **modifies HTML with JavaScript**, mengeksekusi skrip tersebut dari Java, dan akhirnya menyimpan perubahan. Pada akhir Anda akan memiliki contoh yang mandiri dan dapat dijalankan yang dapat Anda masukkan ke dalam proyek apa pun.

## Apa yang Anda Butuhkan

- **Java 8** atau lebih baru (API bekerja dengan Java 8+)
- **Aspose.HTML for Java** library (versi 22.12 atau lebih baru)
- File templat HTML sederhana (kami akan membuatnya secara langsung)
- IDE atau editor teks—tidak perlu yang mewah, bahkan editor teks biasa sudah cukup

> Pro tip: Jika Anda menggunakan Maven, tambahkan dependensi Aspose.HTML ke `pom.xml` Anda. Jika tidak, cukup letakkan JAR ke dalam classpath Anda.

## Langkah 1 – Memuat Dokumen HTML (Persiapkan untuk Execute JavaScript in Java)

Hal pertama yang kami lakukan adalah membuat instance `HTMLDocument` yang menunjuk ke file sumber. Anggap objek ini sebagai jembatan antara Java dan pohon DOM yang nanti akan disentuh oleh skrip.

```java
import com.aspose.html.HTMLDocument;

// Step 1: Load the HTML document you want to manipulate
HTMLDocument htmlDoc = new HTMLDocument("YOUR_DIRECTORY/template.html");
```

> **Why this matters:** Kelas `HTMLDocument` mengurai file menjadi representasi DOM yang dapat berinteraksi dengan mesin skrip Aspose. Tanpa memuat dokumen terlebih dahulu, tidak ada yang dapat diubah oleh skrip.

## Langkah 2 – Tulis JavaScript yang **Append Element to Body**

Sekarang kami membuat potongan kode kecil yang membuat elemen `<p>` dan menambahkannya ke `<body>`. Ini adalah **JavaScript DOM manipulation** klasik, tetapi kami akan menjalankannya dari Java.

```java
// Step 2: JavaScript that adds a new paragraph to the body
String jsCode = ""
    + "var p = document.createElement('p');"
    + "p.textContent = 'Generated by JavaScript';"
    + "document.body.appendChild(p);";
```

> **Why this works:** `document.createElement` membuat node baru, `textContent` mengatur teks di dalamnya, dan `appendChild` menempelkannya ke elemen `<body>` yang sudah ada. Skrip ini adalah JavaScript client‑side murni; Aspose.HTML mengeksekusinya dalam lingkungan headless.

## Langkah 3 – Buat Script Engine (Mesin yang **Run JavaScript from Java**)

Aspose.HTML dilengkapi dengan `ScriptEngine` yang dapat mengevaluasi JavaScript terhadap DOM yang dimuat. Ini adalah komponen yang sebenarnya **run JavaScript from Java**.

```java
import com.aspose.html.scripting.ScriptEngine;

// Step 3: Bind the engine to the document we just loaded
ScriptEngine scriptEngine = new ScriptEngine(htmlDoc);
```

> **Why you need this:** Mesin ini mengisolasi eksekusi skrip, mencegahnya memengaruhi proses Java utama Anda. Ia juga mengembalikan `ScriptExecutionResult` sehingga Anda dapat memverifikasi keberhasilan.

## Langkah 4 – Eksekusi Skrip dan Periksa Hasil

Setelah semua disiapkan, kami akhirnya menjalankan skrip. Metode `execute` mengembalikan objek hasil yang memberi tahu apakah skrip berjalan tanpa error.

```java
import com.aspose.html.scripting.ScriptExecutionResult;

// Step 4: Run the JavaScript and verify it succeeded
ScriptExecutionResult execResult = scriptEngine.execute(jsCode);
if (execResult.isSuccess()) {
    System.out.println("Script executed successfully.");
} else {
    System.err.println("Script failed: " + execResult.getErrorMessage());
}
```

> **Edge case:** Jika skrip melemparkan pengecualian (misalnya, kesalahan sintaks), `isSuccess()` akan menjadi `false` dan `getErrorMessage()` memberikan detailnya. Selalu periksa ini dalam kode produksi.

## Langkah 5 – Simpan Dokumen yang Dimodifikasi (Selesaikan **Modify HTML with JavaScript**)

Setelah skrip mengubah DOM, kami menyimpan perubahan kembali ke disk. File yang dihasilkan kini berisi paragraf baru.

```java
// Step 5: Write the updated DOM back to an HTML file
htmlDoc.save("YOUR_DIRECTORY/modified.html");
System.out.println("Document saved.");
```

> **Result verification:** Buka `modified.html` di browser apa pun. Anda akan melihat konten asli ditambah baris baru yang berisi “Generated by JavaScript”.

## Langkah 6 – Contoh Kerja Lengkap (Semua Langkah Digabungkan)

Berikut adalah program lengkap yang dapat Anda salin‑tempel ke file `JsDomManipulation.java`. Program ini mencakup semua import, penanganan error, dan komentar untuk kejelasan.

```java
import com.aspose.html.HTMLDocument;
import com.aspose.html.scripting.ScriptEngine;
import com.aspose.html.scripting.ScriptExecutionResult;

public class JsDomManipulation {
    public static void main(String[] args) throws Exception {

        // 1️⃣ Load the HTML document to be manipulated
        HTMLDocument htmlDoc = new HTMLDocument("YOUR_DIRECTORY/template.html");

        // 2️⃣ Define JavaScript that adds a new paragraph to the <body>
        String jsCode = "var p = document.createElement('p');"
                      + "p.textContent = 'Generated by JavaScript';"
                      + "document.body.appendChild(p);";

        // 3️⃣ Create a script engine bound to the loaded document
        ScriptEngine scriptEngine = new ScriptEngine(htmlDoc);

        // 4️⃣ Execute the JavaScript and report success
        ScriptExecutionResult execResult = scriptEngine.execute(jsCode);
        if (execResult.isSuccess()) {
            System.out.println("Script executed successfully.");
        } else {
            System.err.println("Script failed: " + execResult.getErrorMessage());
            return; // Stop if the script didn't run
        }

        // 5️⃣ Save the modified document back to HTML
        htmlDoc.save("YOUR_DIRECTORY/modified.html");
        System.out.println("Document saved.");
    }
}
```

### Output yang Diharapkan

Running the program prints:

```
Script executed successfully.
Document saved.
```

And `modified.html` now contains something like:

```html
<!DOCTYPE html>
<html>
<head>
    <title>Template</title>
</head>
<body>
    <h1>Original Content</h1>
    <p>Generated by JavaScript</p>
</body>
</html>
```

## Pertanyaan Umum & Hal-hal yang Perlu Diwaspadai

| Question | Answer |
|----------|--------|
| **Bisakah saya menjalankan beberapa skrip secara berurutan?** | Ya. Cukup panggil `scriptEngine.execute()` lagi dengan string skrip baru. Instance `HTMLDocument` yang sama mempertahankan keadaannya. |
| **Bagaimana jika file HTML tidak memiliki tag `<body>`?** | Aspose.HTML akan secara otomatis membuat elemen `<body>` selama parsing, sehingga skrip tetap berfungsi. Namun, untuk hasil yang dapat diprediksi, mulailah dengan templat yang terstruktur dengan baik. |
| **Apakah pendekatan ini kompatibel dengan Java 17?** | Tentu saja. Perpustakaan ini menargetkan Java 8+, jadi runtime yang lebih baru tidak masalah. |
| **Apakah saya memerlukan lingkungan browser penuh?** | Tidak. Aspose.HTML menyediakan mesin ringan berbasis headless yang menafsirkan API DOM standar tanpa meluncurkan Chrome atau Firefox. |
| **Bagaimana cara menangani file HTML besar?** | Muat dokumen sebagai stream (`new HTMLDocument(InputStream)`) untuk menghindari memuat seluruh file ke memori sekaligus. |

- **Inject CSS** untuk menata elemen yang baru ditambahkan.

- **Run external scripts** yang diambil dari URL (gunakan `scriptEngine.executeFromUrl()` jika diperlukan).
- Gabungkan teknik ini dengan **PDF conversion**—Aspose.HTML dapat merender DOM akhir ke PDF dalam satu panggilan.

Jika Anda tertarik pada **run JavaScript from Java** untuk pengujian atau rendering sisi server, pertimbangkan membungkus logika dalam kelas utilitas dan menggunakannya kembali di seluruh micro‑service.
