# กำหนดค่าแบบอักษรสำหรับ HTML‑to‑PDF Java ด้วย Aspose.HTML

## บทนำ
ในบทแนะนำนี้คุณจะได้เรียนรู้วิธี **สร้าง PDF จาก HTML** ด้วย Aspose.HTML และกำหนดค่าแบบอักษรสำหรับการแปลง HTML‑to‑PDF ใน Java เมื่อทำงานกับเอกสาร HTML การตั้งค่าแบบอักษรที่ถูกต้องจะทำให้ PDF ที่สร้างขึ้นดูเหมือนหน้าเว็บต้นฉบับอย่างแม่นยำ—รักษาสีแบรนด์, การพิมพ์, และการจัดวาง ไม่ว่าคุณจะสร้างรายงาน, ใบแจ้งหนี้, หรือกระบวนการสร้างเอกสารใด ๆ การกำหนดค่าแบบอักษรที่เหมาะสมเป็นกุญแจสำคัญสำหรับ PDF ที่ดูเป็นมืออาชีพ มาดำเนินการผ่านกระบวนการทั้งหมด ตั้งแต่การเตรียมสภาพแวดล้อมจนถึงการแปลง HTML เป็น PDF ด้วยแบบอักษรและ CSS ที่กำหนดเอง

## คำตอบอย่างรวดเร็ว
- **วัตถุประสงค์หลักของบทแนะนำนี้คืออะไร?** กำหนดค่าแบบอักษรสำหรับการแปลง HTML‑to‑PDF ใน Java ด้วย Aspose.HTML.  
- **ไลบรารีใดที่จัดการการแปลง?** Aspose.HTML for Java (คลาส `Converter`).  
- **ฉันต้องการใบอนุญาตหรือไม่?** ใบอนุญาตชั่วคราวของ Aspose จะลบข้อจำกัดการประเมิน; จำเป็นต้องมีใบอนุญาตเต็มสำหรับการใช้งานจริง.  
- **ควรวางแบบอักษรที่กำหนดเองไว้ที่ไหน?** ในโฟลเดอร์ที่อ้างอิงโดย `FontsLookupFolder`, เช่น โฟลเดอร์ `fonts` ที่อยู่ข้างๆ โปรเจคของคุณ.  
- **ฉันสามารถปรับแต่งผลลัพธ์ PDF ได้หรือไม่?** ได้—ใช้ `PdfSaveOptions` เพื่อปรับขนาดหน้า, ระยะขอบ, และอื่น ๆ.

## อะไรคือ **generate PDF from HTML** และทำไมการกำหนดค่าแบบอักษรจึงสำคัญ?
กระบวนการ **generate PDF from HTML** จะเรนเดอร์เอกสาร HTML ให้เป็นหน้ากระดาษ PDF แบบเต็มแบบ. แบบอักษรเป็นส่วนสำคัญของการเรนเดอร์เนื่องจากมีผลต่อการจัดวาง, ระยะห่างระหว่างบรรทัด, และความแม่นยำของภาพ. โดยการชี้ Aspose.HTML ไปยังโฟลเดอร์แบบอักษรที่กำหนดเอง คุณจะทำให้ PDF ใช้แบบอักษรที่คุณออกแบบสำหรับหน้าเว็บอย่างแม่นยำ, กำจัดแบบอักษรสำรองและรักษาความสอดคล้องของแบรนด์.

## ทำไมต้องใช้ Aspose.HTML สำหรับการกำหนดค่าแบบอักษร?
- **การเรนเดอร์ที่แม่นยำ:** รองรับ CSS2.1 และหลายฟีเจอร์ของ CSS3 ทำให้ HTML ของคุณดูเหมือนเดิมใน PDF.  
- **ข้ามแพลตฟอร์ม:** ทำงานบนระบบปฏิบัติการใดก็ได้ที่รัน Java 1.8+.  
- **ความยืดหยุ่นของใบอนุญาต:** ทดสอบด้วยใบอนุญาตชั่วคราว, แล้วเปลี่ยนเป็นใบอนุญาตเต็มสำหรับการใช้งานจริง.  
- **ประสิทธิภาพ:** การแปลงที่เร็วแม้กับหน้าเว็บที่ซับซ้อน.

## คู่มือขั้นตอนต่อขั้นตอน

### ขั้นตอนที่ 1: สร้างเนื้อหา HTML
เราจะเริ่มด้วยการสร้างไฟล์ HTML ง่าย ๆ ที่เราจะเปลี่ยนเป็น PDF ต่อไป.

#### 1.1 เขียนโค้ด HTML
```java
String code = "<h1>FontsSettings property</h1>\r\n" +
    "<p>The FontsSettings property is used for configuration of fonts handling.</p>\r\n";
```

โค้ดส่วนนี้กำหนดหัวเรื่องและย่อหน้า คุณสามารถขยาย HTML ด้วยองค์ประกอบเพิ่มเติมได้หากต้องการทดสอบสไตล์อื่น ๆ.

#### 1.2 บันทึก HTML ลงไฟล์
```java
try (java.io.FileWriter fileWriter = new java.io.FileWriter("user-agent-fontsetting.html")) {
    fileWriter.write(code);
}
```

`FileWriter` จะเขียนสตริงไปยังไฟล์ `user-agent-fontsetting.html` ในโฟลเดอร์โปรเจคของคุณ หลังจากขั้นตอนนี้คุณจะมีไฟล์ HTML จริงพร้อมสำหรับการประมวลผล.

### ขั้นตอนที่ 2: กำหนดค่าสภาพแวดล้อม Aspose.HTML
ตอนนี้เราจะตั้งค่าอ็อบเจ็กต์ `Configuration` ของ Aspose.HTML ซึ่งให้เราควบคุมวิธีการเรนเดอร์ HTML.

#### 2.1 สร้างอินสแตนซ์ Configuration
```java
com.aspose.html.Configuration configuration = new com.aspose.html.Configuration();
```

อ็อบเจ็กต์ `Configuration` เป็นจุดเริ่มต้นสำหรับการปรับแต่งตัวเลือกการเรนเดอร์ เช่น การจัดการแบบอักษรและพฤติกรรมของ user‑agent.

#### 2.2 เข้าถึง User Agent Service
```java
com.aspose.html.services.IUserAgentService userAgent = configuration.getService(com.aspose.html.services.IUserAgentService.class);
```

`IUserAgentService` จัดการสไตล์ชีต, แบบอักษร, และรายละเอียดการเรนเดอร์อื่น ๆ เราจะใช้มันเพื่อแทรก CSS ที่กำหนดเองและชี้ไปยังโฟลเดอร์แบบอักษรของเรา.

### ขั้นตอนที่ 3: ใช้สไตล์และแบบอักษรที่กำหนดเอง
เมื่อสภาพแวดล้อมพร้อม เราสามารถเพิ่มกฎ CSS และบอก Aspose.HTML ว่าจะหาแบบอักษรของเราได้ที่ไหน.

#### 3.1 ตั้งค่า CSS ที่กำหนดเอง
```java
userAgent.setUserStyleSheet("h1 { color:#a52a2a; }\r\n" +
    "p { color:grey; }\r\n");
```

CSS นี้ทำให้หัวเรื่องเป็นสีสีน้ำตาลและย่อหน้าเป็นสีเทา คุณสามารถเพิ่มกฎ CSS ที่ถูกต้องใด ๆ ที่นี่—Aspose.HTML รองรับสเปค CSS2.1 เต็มรูปแบบและหลายฟีเจอร์ของ CSS3 *(นี่คือตัวอย่างของ **apply custom css**.)*

#### 3.2 ชี้ไปยังโฟลเดอร์แบบอักษรที่กำหนดเอง
```java
userAgent.getFontsSettings().setFontsLookupFolder("fonts");
```

วางไฟล์ `.ttf` หรือ `.otf` ที่คุณต้องการใช้ไว้ในโฟลเดอร์ชื่อ `fonts` ที่อยู่ที่รากของโปรเจคของคุณ Aspose.HTML จะโหลดแบบอักษรเหล่านี้โดยอัตโนมัติระหว่างการเรนเดอร์.

> **เคล็ดลับ:** หากคุณมีหลายตระกูลแบบอักษร, ให้จัดระเบียบในโฟลเดอร์ย่อยและเพิ่มโฟลเดอร์หลักแต่ละโฟลเดอร์เข้าไปใน `FontsLookupFolder` โดยใช้รายการคั่นด้วยเซมิโคลอน.

### ขั้นตอนที่ 4: โหลดเอกสาร HTML ด้วยการกำหนดค่า
ตอนนี้เราจะโหลดไฟล์ HTML ที่สร้างไว้ก่อนหน้านี้ โดยใช้การกำหนดค่าที่กำหนดเองที่เราสร้างขึ้น.

```java
com.aspose.html.HTMLDocument document = new com.aspose.html.HTMLDocument("user-agent-fontsetting.html", configuration);
```

อ็อบเจ็กต์ `HTMLDocument` ตอนนี้เป็นตัวแทนของ HTML ที่มีสไตล์พร้อมสำหรับการแปลง.

### ขั้นตอนที่ 5: แปลง HTML เป็น PDF
สุดท้าย เราจะทำการ **aspose html pdf conversion** เพื่อสร้างไฟล์ PDF ที่เคารพแบบอักษรและสไตล์ที่กำหนดเองของเรา.

```java
com.aspose.html.converters.Converter.convertHTML(
    document,
    new com.aspose.html.saving.PdfSaveOptions(),
    "user-agent-fontsetting_out.pdf"
);
```

อ็อบเจ็กต์ `PdfSaveOptions` ให้คุณปรับแต่งการตั้งค่าผลลัพธ์ เช่น ขนาดหน้า, การบีบอัด, และเมตาดาต้า สำหรับการแปลงพื้นฐาน ตัวเลือกเริ่มต้นทำงานได้อย่างสมบูรณ์.

### ขั้นตอนที่ 6: ทำความสะอาดทรัพยากร
การทำลายอย่างเหมาะสมช่วยป้องกันการรั่วไหลของหน่วยความจำ, โดยเฉพาะเมื่อประมวลผลเอกสารจำนวนมากในแอปพลิเคชันที่ทำงานต่อเนื่อง.

#### 6.1 ปล่อย HTMLDocument
```java
if (document != null) {
    document.dispose();
}
```

#### 6.2 ปล่อย Configuration
```java
if (configuration != null) {
    configuration.dispose();
}
```

การเรียกเหล่านี้จะปล่อยทรัพยากรเนทีฟที่ Aspose.HTML จัดสรร.

## คำถามที่พบบ่อย

**Q: ฉันสามารถใช้แบบอักษรใดก็ได้กับ Aspose.HTML สำหรับ Java หรือไม่?**  
A: ได้, แบบอักษร TrueType (`.ttf`) หรือ OpenType (`.otf`) ใด ๆ ที่ระบบปฏิบัติการของคุณรองรับสามารถใช้ได้ เพียงวางไฟล์เหล่านั้นในโฟลเดอร์ที่คุณตั้งค่าโดย `FontsLookupFolder`.

**Q: ฉันต้องการใบอนุญาตเพื่อใช้ Aspose.HTML สำหรับ Java หรือไม่?**  
A: แม้ว่าคุณจะสามารถประเมินไลบรารีโดยไม่ใช้ใบอนุญาต, แต่ [temporary Aspose license](https://purchase.aspose.com/temporary-license/) จะลบข้อจำกัดการประเมิน. สำหรับการใช้งานจริง จำเป็นต้องมีใบอนุญาตเต็ม.

**Q: ฉันสามารถปรับแต่งผลลัพธ์ PDF ได้อย่างไร?**  
A: ส่งอินสแตนซ์ `PdfSaveOptions` ที่กำหนดค่าแล้วไปยัง `convertHTML`. คุณสามารถตั้งค่าขนาดหน้า, ระยะขอบ, ระดับการบีบอัด, และอื่น ๆ.

**Q: สามารถใช้สไตล์ CSS ที่ซับซ้อนมากขึ้นได้หรือไม่?**  
A: ได้, Aspose.HTML รองรับ CSS ช่วงกว้าง ตัวเลือกซับซ้อน, media queries, และ pseudo‑classes ทำงานเช่นเดียวกับในเบราว์เซอร์ แม้ว่าบางฟีเจอร์ CSS3/4 ที่ใหม่มากอาจยังไม่รองรับเต็มที่.

**Q: ฉันจะหา ตัวอย่างและเอกสารเพิ่มเติมได้จากที่ไหน?**  
A: เยี่ยมชมหน้าเอกสารอย่างเป็นทางการของ [Aspose.HTML for Java](https://reference.aspose.com/html/java/) เพื่อดูอ้างอิง API รายละเอียดและตัวอย่างโค้ดเพิ่มเติม.

**Q: ใบอนุญาตชั่วคราวของ Aspose มีผลต่อการแปลงอย่างไร?**  
A: ใบอนุญาตชั่วคราวยกเลิกข้อจำกัด 10 หน้าและลายน้ำที่ปรากฏในโหมดประเมิน ทำให้คุณสามารถทดสอบเวิร์กโฟลว์ **aspose html pdf conversion** ได้อย่างเต็มที่.
