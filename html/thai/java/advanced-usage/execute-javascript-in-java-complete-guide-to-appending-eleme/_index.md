# Execute JavaScript in Java – Complete Guide to Appending Elements to the Body

เคยต้องการ **execute JavaScript in Java** แต่ไม่แน่ใจว่าจะเชื่อมต่อสองโลกนี้อย่างไรหรือไม่? คุณไม่ได้เป็นคนเดียวที่เจอปัญหา นักพัฒนา Java หลายคนเจออุปสรรคเมื่อพวกเขาต้องการทำ **JavaScript DOM manipulation** โดยไม่ต้องเปิดเบราว์เซอร์เต็มรูปแบบ ข่าวดีคือ? ด้วย Aspose.HTML คุณสามารถรันสคริปต์, **append element to body**, และบันทึกมาร์กอัปที่เปลี่ยนแปลงแล้ว—ทั้งหมดจากโค้ด Java ธรรมดา

ในบทแนะนำนี้เราจะพาคุณผ่านทุกอย่างที่ต้องรู้: การโหลดไฟล์ HTML, การเขียนสคริปต์เล็ก ๆ ที่ **modifies HTML with JavaScript**, การรันสคริปต์นั้นจาก Java, และสุดท้ายการบันทึกการเปลี่ยนแปลง เมื่อจบคุณจะได้ตัวอย่างที่ทำงานได้เองและสามารถนำไปใช้ในโปรเจคใดก็ได้

## What You’ll Need
## สิ่งที่คุณต้องเตรียม

- **Java 8** หรือใหม่กว่า (API ทำงานกับ Java 8+)
- **Aspose.HTML for Java** library (เวอร์ชัน 22.12 หรือใหม่กว่า)
- ไฟล์เทมเพลต HTML ง่าย ๆ (เราจะสร้างขึ้นในขณะทำงาน)
- IDE หรือโปรแกรมแก้ไขข้อความ—ไม่ต้องหรูหรา แม้แต่โปรแกรมแก้ไขข้อความธรรมดาก็ใช้ได้

> เคล็ดลับ: หากคุณใช้ Maven ให้เพิ่ม dependency ของ Aspose.HTML ลงใน `pom.xml` ของคุณ มิฉะนั้น แค่วางไฟล์ JAR ลงใน classpath ก็พอ

## Step 1 – Load the HTML Document (Prepare to Execute JavaScript in Java)
## ขั้นตอนที่ 1 – โหลดเอกสาร HTML (เตรียมพร้อมเพื่อ Execute JavaScript in Java)

สิ่งแรกที่เราทำคือสร้างอินสแตนซ์ `HTMLDocument` ที่ชี้ไปยังไฟล์ต้นฉบับ คิดว่าอ็อบเจกต์นี้เป็นสะพานระหว่าง Java กับต้นไม้ DOM ที่สคริปต์จะไปสัมผัสในภายหลัง

```java
import com.aspose.html.HTMLDocument;

// Step 1: Load the HTML document you want to manipulate
HTMLDocument htmlDoc = new HTMLDocument("YOUR_DIRECTORY/template.html");
```

> **Why this matters:** คลาส `HTMLDocument` จะทำการพาร์สไฟล์เป็นโครงสร้าง DOM ที่เอนจินสคริปต์ของ Aspose สามารถโต้ตอบได้ หากไม่โหลดเอกสารก่อน จะไม่มีอะไรให้สคริปต์แก้ไข

## Step 2 – Write JavaScript that **Append Element to Body**
## ขั้นตอนที่ 2 – เขียน JavaScript ที่ **Append Element to Body**

ตอนนี้เราจะสร้างสคริปต์สั้น ๆ ที่สร้างองค์ประกอบ `<p>` แล้วเพิ่มลงใน `<body>` นี่คือการทำ **JavaScript DOM manipulation** แบบคลาสสิก แต่เราจะรันมันจาก Java

```java
// Step 2: JavaScript that adds a new paragraph to the body
String jsCode = ""
    + "var p = document.createElement('p');"
    + "p.textContent = 'Generated by JavaScript';"
    + "document.body.appendChild(p);";
```

> **Why this works:** `document.createElement` สร้างโหนดใหม่, `textContent` ตั้งค่าข้อความภายใน, และ `appendChild` เชื่อมต่อโหนดนั้นกับองค์ประกอบ `<body>` ที่มีอยู่ สคริปต์นี้เป็น JavaScript ฝั่งไคลเอนต์แบบบริสุทธิ์; Aspose.HTML จะรันมันในสภาพแวดล้อม headless

## Step 3 – Create the Script Engine (The Engine That **Run JavaScript from Java**)
## ขั้นตอนที่ 3 – สร้าง Script Engine (Engine ที่ **Run JavaScript from Java**)

Aspose.HTML มาพร้อมกับ `ScriptEngine` ที่สามารถประเมิน JavaScript กับ DOM ที่โหลดไว้แล้ว มันคือส่วนที่ทำให้ **run JavaScript from Java** จริง ๆ

```java
import com.aspose.html.scripting.ScriptEngine;

// Step 3: Bind the engine to the document we just loaded
ScriptEngine scriptEngine = new ScriptEngine(htmlDoc);
```

> **Why you need this:** เอนจินจะแยกการรันสคริปต์ออกจากกระบวนการหลักของ Java เพื่อป้องกันไม่ให้ส่งผลกระทบต่อโปรเซสหลักของคุณ นอกจากนี้ยังคืนค่า `ScriptExecutionResult` เพื่อให้คุณตรวจสอบความสำเร็จได้

## Step 4 – Execute the Script and Check the Result
## ขั้นตอนที่ 4 – Execute Script และตรวจสอบผลลัพธ์

เมื่อทุกอย่างพร้อม เราจะทำการรันสคริปต์จริง ๆ เมธอด `execute` จะคืนออบเจกต์ผลลัพธ์ที่บอกว่าสคริปต์ทำงานสำเร็จโดยไม่มีข้อผิดพลาดหรือไม่

```java
import com.aspose.html.scripting.ScriptExecutionResult;

// Step 4: Run the JavaScript and verify it succeeded
ScriptExecutionResult execResult = scriptEngine.execute(jsCode);
if (execResult.isSuccess()) {
    System.out.println("Script executed successfully.");
} else {
    System.err.println("Script failed: " + execResult.getErrorMessage());
}
```

> **Edge case:** หากสคริปต์โยนข้อยกเว้น (เช่น syntax error) `isSuccess()` จะเป็น `false` และ `getErrorMessage()` จะให้รายละเอียดเสมอ ควรตรวจสอบส่วนนี้เสมอในโค้ด production

## Step 5 – Save the Modified Document (Finish **Modify HTML with JavaScript**)
## ขั้นตอนที่ 5 – บันทึกเอกสารที่แก้ไข (เสร็จสิ้น **Modify HTML with JavaScript**)

หลังจากสคริปต์ทำการเปลี่ยนแปลง DOM แล้ว เราจะบันทึกการเปลี่ยนแปลงกลับไปยังดิสก์ ไฟล์ที่ได้จะมีย่อหน้าที่เพิ่มใหม่อยู่

```java
// Step 5: Write the updated DOM back to an HTML file
htmlDoc.save("YOUR_DIRECTORY/modified.html");
System.out.println("Document saved.");
```

> **Result verification:** เปิด `modified.html` ด้วยเบราว์เซอร์ใดก็ได้ คุณควรเห็นเนื้อหาเดิมพร้อมบรรทัดใหม่ที่เขียนว่า “Generated by JavaScript”

## Step 6 – Full Working Example (All Steps Combined)
## ขั้นตอนที่ 6 – ตัวอย่างทำงานเต็ม (รวมทุกขั้นตอน)

ด้านล่างเป็นโปรแกรมเต็มที่คุณสามารถคัดลอก‑วางลงในไฟล์ `JsDomManipulation.java` มันรวมการ import ทั้งหมด, การจัดการข้อผิดพลาด, และคอมเมนต์เพื่อความชัดเจน

```java
import com.aspose.html.HTMLDocument;
import com.aspose.html.scripting.ScriptEngine;
import com.aspose.html.scripting.ScriptExecutionResult;

public class JsDomManipulation {
    public static void main(String[] args) throws Exception {

        // 1️⃣ Load the HTML document to be manipulated
        HTMLDocument htmlDoc = new HTMLDocument("YOUR_DIRECTORY/template.html");

        // 2️⃣ Define JavaScript that adds a new paragraph to the <body>
        String jsCode = "var p = document.createElement('p');"
                      + "p.textContent = 'Generated by JavaScript';"
                      + "document.body.appendChild(p);";

        // 3️⃣ Create a script engine bound to the loaded document
        ScriptEngine scriptEngine = new ScriptEngine(htmlDoc);

        // 4️⃣ Execute the JavaScript and report success
        ScriptExecutionResult execResult = scriptEngine.execute(jsCode);
        if (execResult.isSuccess()) {
            System.out.println("Script executed successfully.");
        } else {
            System.err.println("Script failed: " + execResult.getErrorMessage());
            return; // Stop if the script didn't run
        }

        // 5️⃣ Save the modified document back to HTML
        htmlDoc.save("YOUR_DIRECTORY/modified.html");
        System.out.println("Document saved.");
    }
}
```

## Common Questions & Gotchas
## คำถามทั่วไป & ข้อควรระวัง

| Question | Answer |
|----------|--------|
| **Can I run multiple scripts sequentially?** | Yes. Just call `scriptEngine.execute()` again with a new script string. The same `HTMLDocument` instance retains its state. |
| **Can I run multiple scripts sequentially?** | ใช่ เพียงเรียก `scriptEngine.execute()` อีกครั้งพร้อมสคริปต์ใหม่ อินสแตนซ์ `HTMLDocument` เดิมจะคงสภาพไว้ |
| **What if the HTML file is missing a `<body>` tag?** | Aspose.HTML will auto‑create a `<body>` element during parsing, so the script still works. However, for predictable results, start with a well‑formed template. |
| **What if the HTML file is missing a `<body>` tag?** | Aspose.HTML จะสร้างองค์ประกอบ `<body>` อัตโนมัติระหว่างการพาร์ส ทำให้สคริปต์ยังทำงานได้ อย่างไรก็ตามเพื่อผลลัพธ์ที่คาดเดาได้ ควรเริ่มจากเทมเพลตที่สมบูรณ์ |
| **Is this approach compatible with Java 17?** | Absolutely. The library targets Java 8+, so newer runtimes are fine. |
| **Is this approach compatible with Java 17?** | แน่นอน ไลบรารีรองรับ Java 8+ ดังนั้น runtime ที่ใหม่กว่าก็ใช้ได้ไม่มีปัญหา |
| **Do I need a full browser environment?** | No. Aspose.HTML provides a lightweight, headless engine that interprets standard DOM APIs without launching Chrome or Firefox. |
| **Do I need a full browser environment?** | ไม่จำเป็น Aspose.HTML มีเอนจินแบบ headless ที่เบาและรองรับ API ของ DOM มาตรฐานโดยไม่ต้องเปิด Chrome หรือ Firefox |
| **How do I handle large HTML files?** | Load the document as a stream (`new HTMLDocument(InputStream)`) to avoid loading the entire file into memory at once. |
| **How do I handle large HTML files?** | โหลดเอกสารเป็นสตรีม (`new HTMLDocument(InputStream)`) เพื่อหลีกเลี่ยงการโหลดไฟล์ทั้งหมดเข้าสู่หน่วยความจำพร้อมกัน |
