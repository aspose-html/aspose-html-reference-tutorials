# Java’da JavaScript Çalıştırma – Gövdeye Eleman Ekleme İçin Tam Kılavuz

Hiç **Java’da JavaScript çalıştırma** ihtiyacı duydunuz mu ama iki dünyayı nasıl birleştireceğinizi bilemediniz mi? Tek başınıza değilsiniz. Birçok Java geliştiricisi, tam bir tarayıcı açmadan **JavaScript DOM manipülasyonu** yapmak istediğinde bir duvara çarpar. İyi haber? Aspose.HTML ile bir betik çalıştırabilir, **gövdede eleman ekleyebilir** ve değiştirilen işaretlemeyi kaydedebilirsiniz—hepsi saf Java kodundan.

Bu öğreticide bilmeniz gereken her şeyi adım adım inceleyeceğiz: bir HTML dosyasını yükleme, **JavaScript ile HTML'i değiştiren** küçük bir betik yazma, bu betiği Java’dan çalıştırma ve sonunda değişiklikleri kalıcı hâle getirme. Sonunda, herhangi bir projeye ekleyebileceğiniz, bağımsız ve çalıştırılabilir bir örnek elde edeceksiniz.

## İhtiyacınız Olanlar

- **Java 8** veya daha yeni (API Java 8+ ile çalışır)
- **Aspose.HTML for Java** kütüphanesi (sürüm 22.12 veya sonrası)
- Basit bir HTML şablon dosyası (kod içinde oluşturacağız)
- Bir IDE veya metin editörü—fancy bir şeye gerek yok, düz metin editörü de yeterli

> Pro ipucu: Maven kullanıyorsanız, Aspose.HTML bağımlılığını `pom.xml` dosyanıza ekleyin. Aksi takdirde, JAR dosyalarını sınıf yolunuza (classpath) bırakmanız yeterli.

## 1. Adım – HTML Belgesini Yükleme (Java’da JavaScript Çalıştırmaya Hazırlık)

İlk olarak, kaynak dosyaya işaret eden bir `HTMLDocument` örneği oluştururuz. Bu nesneyi, Java ile betiğin daha sonra dokunacağı DOM ağacı arasındaki köprü olarak düşünebilirsiniz.

```java
import com.aspose.html.HTMLDocument;

// Step 1: Load the HTML document you want to manipulate
HTMLDocument htmlDoc = new HTMLDocument("YOUR_DIRECTORY/template.html");
```

> **Neden önemli:** `HTMLDocument` sınıfı dosyayı, Aspose betik motorunun etkileşime girebileceği bir DOM temsiline dönüştürür. Belgeyi yüklemeden önce, betiğin değiştirebileceği bir şey olmaz.

## 2. Adım – **Gövdede Eleman Ekle** JavaScript'i Yazma

Şimdi, bir `<p>` elemanı oluşturup `<body>` içine ekleyen küçük bir kod parçası hazırlıyoruz. Bu klasik **JavaScript DOM manipülasyonu** örneği, ancak Java’dan çalıştıracağız.

```java
// Step 2: JavaScript that adds a new paragraph to the body
String jsCode = ""
    + "var p = document.createElement('p');"
    + "p.textContent = 'Generated by JavaScript';"
    + "document.body.appendChild(p);";
```

> **Neden çalışıyor:** `document.createElement` yeni bir düğüm oluşturur, `textContent` iç metnini ayarlar ve `appendChild` mevcut `<body>` elemanına ekler. Betik saf istemci‑tarafı JavaScript’tir; Aspose.HTML bunu başsız (headless) bir ortamda yürütür.

## 3. Adım – Script Motorunu Oluşturma (**Java’dan JavaScript Çalıştıran** Motor)

Aspose.HTML, yüklü DOM’a karşı JavaScript değerlendirebilen bir `ScriptEngine` ile birlikte gelir. İşte **Java’dan JavaScript çalıştıran** parça bu motor.

```java
import com.aspose.html.scripting.ScriptEngine;

// Step 3: Bind the engine to the document we just loaded
ScriptEngine scriptEngine = new ScriptEngine(htmlDoc);
```

> **Neden buna ihtiyacınız var:** Motor, betik yürütmesini izole eder, ana Java sürecinizi etkilemesini önler. Ayrıca bir `ScriptExecutionResult` döndürür, böylece başarıyı doğrulayabilirsiniz.

## 4. Adım – Betiği Çalıştırma ve Sonucu Kontrol Etme

Her şey hazır olduğunda, betiği nihayet çalıştırıyoruz. `execute` metodu, betiğin hatasız çalışıp çalışmadığını belirten bir sonuç nesnesi döndürür.

```java
import com.aspose.html.scripting.ScriptExecutionResult;

// Step 4: Run the JavaScript and verify it succeeded
ScriptExecutionResult execResult = scriptEngine.execute(jsCode);
if (execResult.isSuccess()) {
    System.out.println("Script executed successfully.");
} else {
    System.err.println("Script failed: " + execResult.getErrorMessage());
}
```

> **Köşe durumu:** Betik bir istisna fırlatırsa (ör. sözdizimi hatası), `isSuccess()` **false** döner ve `getErrorMessage()` detayları sağlar. Üretim kodunda her zaman kontrol edin.

## 5. Adım – Değiştirilen Belgeyi Kaydetme (**JavaScript ile HTML Değiştirme** Tamamlanıyor)

Betik DOM’u değiştirdikten sonra, değişiklikleri diske kaydediyoruz. Oluşan dosya artık yeni paragrafı içeriyor.

```java
// Step 5: Write the updated DOM back to an HTML file
htmlDoc.save("YOUR_DIRECTORY/modified.html");
System.out.println("Document saved.");
```

> **Sonuç doğrulama:** `modified.html` dosyasını herhangi bir tarayıcıda açın. Orijinal içeriğin yanında “Generated by JavaScript” yazan yeni bir satır görmelisiniz.

## 6. Adım – Tam Çalışan Örnek (Tüm Adımlar Birleştirildi)

Aşağıda, `JsDomManipulation.java` dosyasına kopyalayıp yapıştırabileceğiniz tam program yer alıyor. Tüm importları, hata yönetimini ve açıklamaları içerir.

```java
import com.aspose.html.HTMLDocument;
import com.aspose.html.scripting.ScriptEngine;
import com.aspose.html.scripting.ScriptExecutionResult;

public class JsDomManipulation {
    public static void main(String[] args) throws Exception {

        // 1️⃣ Load the HTML document to be manipulated
        HTMLDocument htmlDoc = new HTMLDocument("YOUR_DIRECTORY/template.html");

        // 2️⃣ Define JavaScript that adds a new paragraph to the <body>
        String jsCode = "var p = document.createElement('p');"
                      + "p.textContent = 'Generated by JavaScript';"
                      + "document.body.appendChild(p);";

        // 3️⃣ Create a script engine bound to the loaded document
        ScriptEngine scriptEngine = new ScriptEngine(htmlDoc);

        // 4️⃣ Execute the JavaScript and report success
        ScriptExecutionResult execResult = scriptEngine.execute(jsCode);
        if (execResult.isSuccess()) {
            System.out.println("Script executed successfully.");
        } else {
            System.err.println("Script failed: " + execResult.getErrorMessage());
            return; // Stop if the script didn't run
        }

        // 5️⃣ Save the modified document back to HTML
        htmlDoc.save("YOUR_DIRECTORY/modified.html");
        System.out.println("Document saved.");
    }
}
```

### Beklenen Çıktı

Programı çalıştırdığınızda şu çıktı verir:

```
Script executed successfully.
Document saved.
```

Ve `modified.html` dosyası şu şekilde bir içerik taşır:

```html
<!DOCTYPE html>
<html>
<head>
    <title>Template</title>
</head>
<body>
    <h1>Original Content</h1>
    <p>Generated by JavaScript</p>
</body>
</html>
```

## Yaygın Sorular & Dikkat Edilmesi Gerekenler

| Soru | Cevap |
|------|-------|
| **Birden fazla betiği sıralı çalıştırabilir miyim?** | Evet. Yeni bir betik dizesiyle `scriptEngine.execute()` metodunu tekrar çağırın. Aynı `HTMLDocument` örneği durumunu korur. |
| **HTML dosyasında `<body>` etiketi yoksa ne olur?** | Aspose.HTML, ayrıştırma sırasında otomatik olarak bir `<body>` elementi oluşturur, böylece betik hâlâ çalışır. Ancak tutarlı sonuçlar için iyi biçimlendirilmiş bir şablonla başlamanız önerilir. |
| **Bu yaklaşım Java 17 ile uyumlu mu?** | Kesinlikle. Kütüphane Java 8+ hedeflediği için daha yeni çalışma zamanlarıyla da sorunsuz çalışır. |
| **Tam bir tarayıcı ortamına ihtiyacım var mı?** | Hayır. Aspose.HTML, Chrome veya Firefox başlatmadan standart DOM API’lerini yorumlayan hafif, başsız bir motor sağlar. |
| **Büyük HTML dosyalarını nasıl yönetirim?** | Belgeyi bir akış olarak yükleyin (`new HTMLDocument(InputStream)`) böylece tüm dosyayı belleğe aynı anda almaktan kaçınırsınız. |

## Sonraki Adımlar

Artık **Java’da JavaScript çalıştırma** konusunu bildiğinize göre, daha ileri **JavaScript DOM manipülasyonu** tekniklerini keşfedebilirsiniz:

- Yeni eklenen elemanları stillendirmek için **CSS enjekte edin**.
- Özelliklere göre mevcut düğümleri değiştirmek için **DOM’u dolaşın**.
- Bir URL’den alınan **harici betikleri çalıştırın** (gerekirse `scriptEngine.executeFromUrl()` kullanın).
- Bu tekniği **PDF dönüşümü** ile birleştirin—Aspose.HTML, son DOM’u tek bir çağrıyla PDF’e render edebilir.

**Java’dan JavaScript çalıştırma** ile test veya sunucu‑tarafı render işlemleri için ilgileniyorsanız, mantığı bir yardımcı sınıfa (utility class) paketleyip mikro‑servisler arasında yeniden kullanmayı düşünün.
