# Ejecutar JavaScript en Java – Guía completa para añadir elementos al body

¿Alguna vez necesitaste **ejecutar JavaScript en Java** pero no sabías cómo conectar ambos mundos? No eres el único. Muchos desarrolladores Java se topan con un obstáculo cuando quieren realizar **manipulación del DOM con JavaScript** sin lanzar un navegador completo. ¿La buena noticia? Con Aspose.HTML puedes ejecutar un script, **añadir un elemento al body**, y guardar el marcado modificado, todo desde código Java puro.

En este tutorial recorreremos todo lo que necesitas saber: cargar un archivo HTML, escribir un pequeño script que **modifique HTML con JavaScript**, ejecutar ese script desde Java y, finalmente, persistir los cambios. Al final tendrás un ejemplo autocontenido y ejecutable que podrás incorporar a cualquier proyecto.

## Lo que necesitarás

- **Java 8** o superior (la API funciona con Java 8+)
- Biblioteca **Aspose.HTML for Java** (versión 22.12 o posterior)
- Un archivo de plantilla HTML sencillo (crearemos uno al vuelo)
- Un IDE o editor de texto—no hace falta nada sofisticado, incluso un editor de texto plano sirve

> Consejo profesional: Si usas Maven, agrega la dependencia de Aspose.HTML a tu `pom.xml`. De lo contrario, simplemente coloca los JARs en tu classpath.

## Paso 1 – Cargar el documento HTML (Preparar la ejecución de JavaScript en Java)

Lo primero que hacemos es crear una instancia de `HTMLDocument` que apunte al archivo fuente. Piensa en este objeto como el puente entre Java y el árbol DOM que el script tocará más adelante.

```java
import com.aspose.html.HTMLDocument;

// Step 1: Load the HTML document you want to manipulate
HTMLDocument htmlDoc = new HTMLDocument("YOUR_DIRECTORY/template.html");
```

> **Por qué es importante:** La clase `HTMLDocument` analiza el archivo y lo convierte en una representación DOM con la que el motor de scripting de Aspose puede interactuar. Sin cargar el documento primero, no hay nada que el script pueda modificar.

## Paso 2 – Escribir JavaScript que **añada un elemento al body**

Ahora redactamos un pequeño fragmento que crea un elemento `<p>` y lo agrega al `<body>`. Esto es la clásica **manipulación del DOM con JavaScript**, pero lo ejecutaremos desde Java.

```java
// Step 2: JavaScript that adds a new paragraph to the body
String jsCode = ""
    + "var p = document.createElement('p');"
    + "p.textContent = 'Generated by JavaScript';"
    + "document.body.appendChild(p);";
```

> **Por qué funciona:** `document.createElement` crea un nuevo nodo, `textContent` establece su texto interno y `appendChild` lo adjunta al elemento `<body>` existente. El script es JavaScript puro del lado del cliente; Aspose.HTML lo ejecuta en un entorno sin cabeza.

## Paso 3 – Crear el motor de scripts (El motor que **ejecuta JavaScript desde Java**)

Aspose.HTML incluye un `ScriptEngine` que puede evaluar JavaScript contra el DOM cargado. Es la pieza que realmente **ejecuta JavaScript desde Java**.

```java
import com.aspose.html.scripting.ScriptEngine;

// Step 3: Bind the engine to the document we just loaded
ScriptEngine scriptEngine = new ScriptEngine(htmlDoc);
```

> **Por qué lo necesitas:** El motor aísla la ejecución del script, evitando que afecte a tu proceso principal de Java. Además, devuelve un `ScriptExecutionResult` para que puedas verificar el éxito.

## Paso 4 – Ejecutar el script y comprobar el resultado

Con todo configurado, finalmente lanzamos el script. El método `execute` devuelve un objeto de resultado que indica si el script se ejecutó sin errores.

```java
import com.aspose.html.scripting.ScriptExecutionResult;

// Step 4: Run the JavaScript and verify it succeeded
ScriptExecutionResult execResult = scriptEngine.execute(jsCode);
if (execResult.isSuccess()) {
    System.out.println("Script executed successfully.");
} else {
    System.err.println("Script failed: " + execResult.getErrorMessage());
}
```

> **Caso límite:** Si el script lanza una excepción (por ejemplo, un error de sintaxis), `isSuccess()` será `false` y `getErrorMessage()` proporcionará los detalles. Siempre verifica esto en código de producción.

## Paso 5 – Guardar el documento modificado (Finalizar **modificar HTML con JavaScript**)

Después de que el script haya mutado el DOM, persistimos los cambios en disco. El archivo resultante ahora contiene el nuevo párrafo.

```java
// Step 5: Write the updated DOM back to an HTML file
htmlDoc.save("YOUR_DIRECTORY/modified.html");
System.out.println("Document saved.");
```

> **Verificación del resultado:** Abre `modified.html` en cualquier navegador. Deberías ver el contenido original más una nueva línea que dice “Generated by JavaScript”.

## Paso 6 – Ejemplo completo (Todos los pasos combinados)

A continuación tienes el programa completo que puedes copiar y pegar en un archivo `JsDomManipulation.java`. Incluye todas las importaciones, manejo de errores y comentarios para mayor claridad.

```java
import com.aspose.html.HTMLDocument;
import com.aspose.html.scripting.ScriptEngine;
import com.aspose.html.scripting.ScriptExecutionResult;

public class JsDomManipulation {
    public static void main(String[] args) throws Exception {

        // 1️⃣ Load the HTML document to be manipulated
        HTMLDocument htmlDoc = new HTMLDocument("YOUR_DIRECTORY/template.html");

        // 2️⃣ Define JavaScript that adds a new paragraph to the <body>
        String jsCode = "var p = document.createElement('p');"
                      + "p.textContent = 'Generated by JavaScript';"
                      + "document.body.appendChild(p);";

        // 3️⃣ Create a script engine bound to the loaded document
        ScriptEngine scriptEngine = new ScriptEngine(htmlDoc);

        // 4️⃣ Execute the JavaScript and report success
        ScriptExecutionResult execResult = scriptEngine.execute(jsCode);
        if (execResult.isSuccess()) {
            System.out.println("Script executed successfully.");
        } else {
            System.err.println("Script failed: " + execResult.getErrorMessage());
            return; // Stop if the script didn't run
        }

        // 5️⃣ Save the modified document back to HTML
        htmlDoc.save("YOUR_DIRECTORY/modified.html");
        System.out.println("Document saved.");
    }
}
```

### Salida esperada

Al ejecutar el programa se imprime:

```
Script executed successfully.
Document saved.
```

Y `modified.html` ahora contiene algo como:

```html
<!DOCTYPE html>
<html>
<head>
    <title>Template</title>
</head>
<body>
    <h1>Original Content</h1>
    <p>Generated by JavaScript</p>
</body>
</html>
```

## Preguntas frecuentes y trampas comunes

| Pregunta | Respuesta |
|----------|-----------|
| **¿Puedo ejecutar varios scripts secuencialmente?** | Sí. Simplemente llama a `scriptEngine.execute()` nuevamente con una nueva cadena de script. La misma instancia de `HTMLDocument` conserva su estado. |
| **¿Qué pasa si el archivo HTML no tiene una etiqueta `<body>`?** | Aspose.HTML creará automáticamente un elemento `<body>` durante el análisis, por lo que el script seguirá funcionando. Sin embargo, para obtener resultados predecibles, comienza con una plantilla bien formada. |
| **¿Este enfoque es compatible con Java 17?** | Absolutamente. La biblioteca está dirigida a Java 8+, así que los entornos más recientes funcionan sin problemas. |
| **¿Necesito un entorno de navegador completo?** | No. Aspose.HTML proporciona un motor ligero y sin cabeza que interpreta las APIs estándar del DOM sin lanzar Chrome o Firefox. |
| **¿Cómo manejo archivos HTML muy grandes?** | Carga el documento como un flujo (`new HTMLDocument(InputStream)`) para evitar cargar todo el archivo en memoria de una sola vez. |
