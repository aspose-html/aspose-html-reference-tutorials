## Problemas comunes y soluciones
- **Fuentes faltantes en PDF** – Asegúrese de que las fuentes requeridas estén instaladas en el servidor o incrústelas usando `PdfSaveOptions`.  
- **Los archivos EPUB grandes generan presión de memoria** – Use procesamiento basado en flujos (`MemoryStream` → `FileStream`) para reducir el uso del heap.  
- **El renderizado del canvas aparece en blanco** – Verifique que el canvas se haya dibujado completamente antes de invocar la API de conversión.

**P: ¿Puedo convertir HTML a PDF sin una licencia?**  
R: Hay una prueba gratuita disponible para evaluación, pero se requiere una licencia comercial para implementaciones en producción.

**P: ¿Aspose.HTML admite características de CSS3?**  
R: Sí, el motor de renderizado admite la mayoría de las propiedades CSS3, incluidas flexbox y diseños de cuadrícula.

**P: ¿Cómo automatizo el llenado de múltiples formularios HTML?**  
R: Use la API `Form` para cargar un documento, establecer los valores de los campos programáticamente y luego guardar el resultado.

**P: ¿Es posible convertir una página HTML directamente a SVG?**  
R: Absolutamente – la clase `HtmlToSvgConverter` maneja esta conversión con alta fidelidad.

**P: ¿Cuál es la mejor manera de convertir un canvas HTML grande a PDF?**  
R: Renderice el canvas a un mapa de bits primero, luego use `PdfSaveOptions` para incrustar la imagen, o use el método incorporado canvas‑to‑PDF para salida vectorial.
