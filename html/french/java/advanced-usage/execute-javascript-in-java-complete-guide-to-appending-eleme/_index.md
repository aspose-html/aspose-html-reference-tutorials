# Exécuter du JavaScript en Java – Guide complet pour ajouter des éléments au corps du document

Vous avez déjà eu besoin d'**exécuter du JavaScript en Java** sans savoir comment relier les deux mondes ? Vous n'êtes pas seul. De nombreux développeurs Java se heurtent à un mur lorsqu'ils souhaitent réaliser une **manipulation du DOM en JavaScript** sans lancer un navigateur complet. La bonne nouvelle ? Avec Aspose.HTML, vous pouvez exécuter un script, **ajouter un élément au body**, et enregistrer le balisage modifié—tout cela depuis du code Java pur.

Dans ce tutoriel, nous passerons en revue tout ce que vous devez savoir : charger un fichier HTML, écrire un petit script qui **modifie le HTML avec JavaScript**, exécuter ce script depuis Java, puis persister les changements. À la fin, vous disposerez d'un exemple autonome et exécutable que vous pourrez intégrer à n'importe quel projet.

## Ce dont vous aurez besoin

- **Java 8** ou version ultérieure (l'API fonctionne avec Java 8+)
- Bibliothèque **Aspose.HTML for Java** (version 22.12 ou plus récente)
- Un fichier modèle HTML simple (nous le créerons à la volée)
- Un IDE ou un éditeur de texte—rien de spécial, même un éditeur texte suffit

> Pro tip : Si vous utilisez Maven, ajoutez la dépendance Aspose.HTML à votre `pom.xml`. Sinon, déposez simplement les JARs dans votre classpath.

## Étape 1 – Charger le document HTML (Préparer l'exécution de JavaScript en Java)

La première chose que nous faisons est de créer une instance `HTMLDocument` qui pointe vers le fichier source. Considérez cet objet comme le pont entre Java et l'arbre DOM que le script touchera plus tard.

```java
import com.aspose.html.HTMLDocument;

// Step 1: Load the HTML document you want to manipulate
HTMLDocument htmlDoc = new HTMLDocument("YOUR_DIRECTORY/template.html");
```

> **Pourquoi c’est important :** La classe `HTMLDocument` analyse le fichier en une représentation DOM avec laquelle le moteur de script Aspose peut interagir. Sans charger le document au préalable, il n’y a rien à modifier pour le script.

## Étape 2 – Écrire le JavaScript qui **ajoute un élément au body**

Nous rédigeons maintenant un petit extrait qui crée un élément `<p>` et l’ajoute au `<body>`. C’est de la **manipulation du DOM en JavaScript** classique, mais nous l’exécuterons depuis Java.

```java
// Step 2: JavaScript that adds a new paragraph to the body
String jsCode = ""
    + "var p = document.createElement('p');"
    + "p.textContent = 'Generated by JavaScript';"
    + "document.body.appendChild(p);";
```

> **Pourquoi cela fonctionne :** `document.createElement` crée un nouveau nœud, `textContent` définit son texte interne, et `appendChild` le rattache à l’élément `<body>` existant. Le script est du JavaScript pur côté client ; Aspose.HTML l’exécute dans un environnement sans tête.

## Étape 3 – Créer le moteur de script (Le moteur qui **exécute du JavaScript depuis Java**)

Aspose.HTML fournit un `ScriptEngine` capable d’évaluer du JavaScript sur le DOM chargé. C’est le composant qui **exécute du JavaScript depuis Java**.

```java
import com.aspose.html.scripting.ScriptEngine;

// Step 3: Bind the engine to the document we just loaded
ScriptEngine scriptEngine = new ScriptEngine(htmlDoc);
```

> **Pourquoi vous en avez besoin :** Le moteur isole l’exécution du script, empêchant qu’il n’affecte votre processus Java principal. Il renvoie également un `ScriptExecutionResult` afin que vous puissiez vérifier le succès.

## Étape 4 – Exécuter le script et vérifier le résultat

Une fois tout configuré, nous lançons enfin le script. La méthode `execute` renvoie un objet résultat qui indique si le script s’est exécuté sans erreurs.

```java
import com.aspose.html.scripting.ScriptExecutionResult;

// Step 4: Run the JavaScript and verify it succeeded
ScriptExecutionResult execResult = scriptEngine.execute(jsCode);
if (execResult.isSuccess()) {
    System.out.println("Script executed successfully.");
} else {
    System.err.println("Script failed: " + execResult.getErrorMessage());
}
```

> **Cas limite :** Si le script lève une exception (par ex., une erreur de syntaxe), `isSuccess()` sera `false` et `getErrorMessage()` fournira les détails. Vérifiez toujours cela dans le code de production.

## Étape 5 – Enregistrer le document modifié (Terminer **la modification du HTML avec JavaScript**)

Après que le script ait muté le DOM, nous persistons les changements sur le disque. Le fichier résultant contient maintenant le nouveau paragraphe.

```java
// Step 5: Write the updated DOM back to an HTML file
htmlDoc.save("YOUR_DIRECTORY/modified.html");
System.out.println("Document saved.");
```

> **Vérification du résultat :** Ouvrez `modified.html` dans n’importe quel navigateur. Vous devriez voir le contenu original plus une nouvelle ligne affichant « Generated by JavaScript ».

## Étape 6 – Exemple complet fonctionnel (Toutes les étapes combinées)

Voici le programme complet que vous pouvez copier‑coller dans un fichier `JsDomManipulation.java`. Il inclut tous les imports, la gestion des erreurs et des commentaires pour plus de clarté.

```java
import com.aspose.html.HTMLDocument;
import com.aspose.html.scripting.ScriptEngine;
import com.aspose.html.scripting.ScriptExecutionResult;

public class JsDomManipulation {
    public static void main(String[] args) throws Exception {

        // 1️⃣ Load the HTML document to be manipulated
        HTMLDocument htmlDoc = new HTMLDocument("YOUR_DIRECTORY/template.html");

        // 2️⃣ Define JavaScript that adds a new paragraph to the <body>
        String jsCode = "var p = document.createElement('p');"
                      + "p.textContent = 'Generated by JavaScript';"
                      + "document.body.appendChild(p);";

        // 3️⃣ Create a script engine bound to the loaded document
        ScriptEngine scriptEngine = new ScriptEngine(htmlDoc);

        // 4️⃣ Execute the JavaScript and report success
        ScriptExecutionResult execResult = scriptEngine.execute(jsCode);
        if (execResult.isSuccess()) {
            System.out.println("Script executed successfully.");
        } else {
            System.err.println("Script failed: " + execResult.getErrorMessage());
            return; // Stop if the script didn't run
        }

        // 5️⃣ Save the modified document back to HTML
        htmlDoc.save("YOUR_DIRECTORY/modified.html");
        System.out.println("Document saved.");
    }
}
```

### Résultat attendu

L’exécution du programme affiche :

```
Script executed successfully.
Document saved.
```

Et `modified.html` contient désormais quelque chose comme :

```html
<!DOCTYPE html>
<html>
<head>
    <title>Template</title>
</head>
<body>
    <h1>Original Content</h1>
    <p>Generated by JavaScript</p>
</body>
</html>
```

## Questions fréquentes & Pièges

| Question | Réponse |
|----------|--------|
| **Puis‑je exécuter plusieurs scripts séquentiellement ?** | Oui. Appelez simplement `scriptEngine.execute()` à nouveau avec une nouvelle chaîne de script. La même instance `HTMLDocument` conserve son état. |
| **Que se passe‑t‑il si le fichier HTML ne contient pas de balise `<body>` ?** | Aspose.HTML créera automatiquement un élément `<body>` lors de l’analyse, de sorte que le script fonctionne toujours. Cependant, pour des résultats prévisibles, commencez avec un modèle bien formé. |
| **Cette approche est‑elle compatible avec Java 17 ?** | Absolument. La bibliothèque cible Java 8+, donc les runtimes plus récents fonctionnent sans problème. |
| **Ai‑je besoin d’un environnement de navigateur complet ?** | Non. Aspose.HTML fournit un moteur léger et sans tête qui interprète les API DOM standards sans lancer Chrome ou Firefox. |
| **Comment gérer de gros fichiers HTML ?** | Chargez le document sous forme de flux (`new HTMLDocument(InputStream)`) afin d’éviter de charger tout le fichier en mémoire d’un coup. |
