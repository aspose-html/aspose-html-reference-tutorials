# Извлечение заголовка из HTML в Java – Полное руководство по песочнице

Когда‑нибудь вам нужно было **извлечь заголовок из HTML**, но вы не знали, как безопасно выполнить страницу?  
Возможно, вы пытались загрузить удалённый файл и получили `NullPointerException`, потому что скрипт никогда не завершился.  

В этом руководстве мы покажем вам надёжный способ **извлечь заголовок из HTML** с помощью Aspose HTML for Java, при этом удерживая скрипт в песочнице, настраивая тайм‑аут скрипта и загружая HTML‑документ в Java. К концу вы получите готовый к запуску фрагмент кода, чёткое объяснение каждой настройки и советы по обработке граничных случаев.

> **Что вы узнаете**
> - Как **запускать HTML в песочнице** с пользовательским размером экрана.  
> - Точные шаги для **загрузки HTML документа в Java** с использованием Aspose HTML.  
> - Как **настроить тайм‑аут скрипта**, чтобы вредоносные скрипты не зависали в вашем приложении.  
> - Как прочитать полученный тег `<title>` после завершения скрипта (или его тайм‑аута).

## Обзор: Почему песочница важна при извлечении заголовка из HTML

Подумайте о песочнице как о небольшом, огороженном игровом поле для HTML‑страницы.  
Если страница содержит JavaScript, который пытается получать внешние ресурсы, открывать новые окна или попасть в бесконечный цикл, песочница останавливает его мгновенно.  
Эта система безопасности особенно ценна, когда единственное, что вам действительно нужно, — элемент `<title>`; нет необходимости раскрывать всю вашу JVM потенциально вредоносному коду.

Ниже приведён полный, исполняемый пример. Не стесняйтесь скопировать‑вставить его в новый Maven‑проект с зависимостью Aspose HTML for Java.

```java
import com.aspose.html.dom.Document;
import com.aspose.html.environment.Sandbox;
import com.aspose.html.environment.SandboxOptions;
import com.aspose.html.environment.ScriptExecutionOptions;

public class ExtractTitleDemo {
    public static void main(String[] args) {
        // 1️⃣ Configure sandbox display size (optional but helpful for layout‑dependent scripts)
        SandboxOptions sandboxOptions = new SandboxOptions();
        sandboxOptions.setScreenWidth(800);
        sandboxOptions.setScreenHeight(600);

        // 2️⃣ Set a maximum script execution time – 2 seconds in this case
        ScriptExecutionOptions scriptOptions = new ScriptExecutionOptions();
        scriptOptions.setTimeout(2000); // milliseconds

        // 3️⃣ Create the sandbox, load the HTML file, and let the script run
        try (Sandbox sandbox = new Sandbox(sandboxOptions, scriptOptions);
             Document document = sandbox.openDocument("YOUR_DIRECTORY/scripted.html")) {

            // 4️⃣ The script has already run (or timed‑out). Grab the title.
            System.out.println("Title after script: " + document.getTitle());
        } catch (Exception e) {
            System.err.println("Failed to extract title: " + e.getMessage());
        }
    }
}
```

**Ожидаемый вывод (когда скрипт завершается в течение 2 секунд):**

```
Title after script: My Awesome Page
```

Если скрипт превышает лимит, песочница прерывает его, и вы всё равно получаете тот заголовок, который был до тайм‑аута.

## Шаг 1 – Настройка тайм‑аута скрипта (и почему это важно)

Когда вы **настраиваете тайм‑аут скрипта**, вы указываете песочнице, как долго скрипт может работать, прежде чем его принудительно остановить.  
Лимит в 2 секунды — хорошая отправная точка; он достаточно длителен для большинства скриптов, манипулирующих DOM, но достаточно короток, чтобы ваш сервер оставался отзывчивым.

```java
ScriptExecutionOptions scriptOptions = new ScriptExecutionOptions();
scriptOptions.setTimeout(2000); // 2000 ms = 2 seconds
```

> **Pro tip:** Если вы замечаете, что легитимные страницы обрезаются, увеличьте тайм‑аут до 5000 мс.  
> С другой стороны, если вы обрабатываете недоверенный контент, держите его низким, чтобы избежать атак отказа в обслуживании.

## Шаг 2 – Загрузка HTML‑документа в Java (с использованием Aspose HTML)

Строка `sandbox.openDocument("YOUR_DIRECTORY/scripted.html")` выполняет основную работу для шага **загрузки HTML документа в Java**.  
Aspose HTML берёт на себя разбор, выполнение встроенных скриптов и построение DOM, который вы можете запросить.

```java
Document document = sandbox.openDocument("YOUR_DIRECTORY/scripted.html");
```

Убедитесь, что путь указывает на реальный файл на сервере, или используйте объект `File`/`URL`, если предпочитаете более динамический подход.  
Песочница автоматически будет учитывать размеры экрана, которые вы задали ранее, что может влиять на скрипты, запрашивающие `window.innerWidth`.

## Шаг 3 – Запуск HTML в песочнице: позвольте движку выполнить свою работу

Вам не нужно вызывать какой‑либо дополнительный метод «run» — песочница выполняет страницу сразу после её открытия.  
Это магия **запуска HTML в песочнице**: движок разбирает разметку, генерирует событие `DOMContentLoaded` и исполняет любые теги `<script>` — всё внутри изолированной среды.

Если страница содержит `setTimeout` или длительные циклы, тайм‑аут, настроенный в Шаге 1, вмешается.  
Никакой дополнительный код не требуется — просто расслабьтесь и позвольте песочнице справиться с этим.

## Шаг 4 – Получение заголовка после выполнения скрипта

После завершения (или прерывания) работы песочницы вы можете извлечь заголовок непосредственно из DOM:

```java
String title = document.getTitle();
System.out.println("Title after script: " + title);
```

Даже если исходный HTML имел пустой `<title>`, а скрипт позже установил его, вы всё равно увидите окончательное значение — именно то, что нужно при **извлечении заголовка из HTML**.

## Бонус: Обработка тайм‑аутов и ошибок корректно

Реальная реализация должна предусматривать два распространённых сбоя:

1. **Тайм‑ауты** — скрипт не завершился вовремя.  
   *Решение:* Перехватить исключение тайм‑аута (Aspose бросает специфичный подкласс) и вернуть оригинальный заголовок или заполнитель.

2. **Некорректный HTML** — файл не может быть разобран.  
   *Решение:* Обернуть создание песочницы в блок `try‑with‑resources` (как показано) и записать ошибку в журнал для последующего анализа.

```java
catch (com.aspose.html.environment.ScriptTimeoutException toe) {
    System.err.println("Script timed out – using original title.");
    System.out.println("Title after script: " + document.getTitle());
}
```

## Часто задаваемые вопросы и граничные случаи

**Что если страница использует внешние скрипты?**  
Песочница по умолчанию блокирует сетевые запросы, поэтому такие скрипты просто не выполняются. Если вам *нужны* они, включите пользовательский `NetworkHandler` — но это противоречит цели безопасной песочницы.

**Можно ли изменить размер области просмотра после создания песочницы?**  
Нет. `SandboxOptions` должны быть заданы до создания экземпляра песочницы. Создайте новую песочницу, если нужен другой размер.

**Сохраняется ли регистр заголовка?**  
Да. Aspose HTML возвращает точную строку, хранящуюся в элементе `<title>` после выполнения скрипта, сохраняя регистр и пробелы.
