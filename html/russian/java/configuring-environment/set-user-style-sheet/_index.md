# Создать PDF из HTML – Установить пользовательскую таблицу стилей в Aspose.HTML для Java

## Введение
В этом руководстве вы узнаете, как **создать PDF из HTML** с помощью Aspose.HTML для Java, применяя пользовательскую таблицу стилей. Хотели ли вы когда‑нибудь изменить внешний вид ваших HTML‑документов своим уникальным стилем? Представьте, что вы создаёте веб‑страницу и хотите, чтобы заголовки выделялись определённым цветом, а абзацы выглядели одинаково на всех устройствах. Здесь на помощь приходит *пользовательская таблица стилей* и **User Agent Service**. Мы пройдём каждый шаг — от написания простого HTML‑файла, настройки пользовательского агента, до **конвертации HTML в PDF** — чтобы вы могли сразу увидеть результат.

## Быстрые ответы
- **Что означает «создать PDF из HTML»?** Это рендеринг HTML‑документа (с CSS, изображениями, шрифтами и т.д.) и сохранение визуального результата в виде PDF‑файла.  
- **Какой компонент Aspose требуется?** Библиотека Aspose.HTML для Java предоставляет движок конвертации и User Agent Service.  
- **Нужна ли лицензия для тестирования?** Бесплатная пробная версия подходит для разработки; для продакшна требуется коммерческая лицензия.  
- **Можно ли использовать внешний CSS‑файл?** Да — вы можете подключать внешние таблицы стилей так же, как в обычном браузере.  
- **Сколько времени занимает конвертация?** Для простого документа, как в этом руководстве, конвертация завершается менее чем за секунду.  
- **Зачем настраивать User Agent Service?** Он позволяет внедрить пользовательскую таблицу стилей, задать наборы символов по умолчанию и управлять параметрами рендеринга, обеспечивая единообразный вывод PDF.  

## Что такое «создать PDF из HTML»?
Создание PDF из HTML — это процесс преобразования веб‑ориентированного документа (HTML + CSS) в фиксированный PDF‑файл. Это полезно для генерации отчётов, счетов‑фактур или любого печатного материала напрямую из веб‑контента.

## Почему использовать User Agent Service в Aspose.HTML?
**User Agent Service** предоставляет низкоуровневый контроль над параметрами рендеринга, такими как набор символов по умолчанию, язык, шрифты и — что самое важное для этого руководства — пользовательская таблица стилей. Применяя стили на этом уровне, вы гарантируете одинаковый визуальный результат, даже если исходный HTML не содержит собственного CSS.

## Предварительные требования
Прежде чем перейти к коду, убедитесь, что у вас есть следующее:

1. **Aspose.HTML для Java** — скачайте последнюю JAR‑файл со [страницы релизов Aspose](https://releases.aspose.com/html/java/).  
2. **Java Development Kit (JDK) 8+** — убедитесь, что `java -version` выводит 8 или выше.  
3. **IDE** — IntelliJ IDEA, Eclipse или NetBeans подойдут.  
4. **Базовые знания HTML/CSS** — необязательно, но будет полезно.

## Импорт пакетов
Для начала импортируйте необходимые классы Java. Единственный явный импорт, который нужен в этом примере, — `java.io.IOException`; классы Aspose будут использоваться через полностью квалифицированные имена позже.

```java
import java.io.IOException;
```

## Шаг 1: Создать простой HTML‑документ
Сначала создадим минимальный HTML‑файл (`document.html`), который будет источником для конвертации в PDF.

```java
String code = "<h1>User Agent Service</h1>\r\n" +
        "<p>The User Agent Service allows you to specify a custom user stylesheet, a primary character set for the document, language, and fonts settings.</p>\r\n";
try (java.io.FileWriter fileWriter = new java.io.FileWriter("document.html")) {
    fileWriter.write(code);
} catch (IOException e) {
    e.printStackTrace();
}
```

> **Совет:** Держите HTML‑файл в той же директории, что и ваш Java‑исходник, чтобы избежать проблем с путями.

## Шаг 2: Настроить конфигурацию Aspose.HTML
Создайте объект `Configuration`. Этот объект служит контейнером для всех сервисов (включая User Agent Service), которые вы будете использовать дальше.

```java
com.aspose.html.Configuration configuration = new com.aspose.html.Configuration();
```

## Шаг 3: Доступ к User Agent Service  
**User Agent Service** позволяет внедрять пользовательскую таблицу стилей, задавать набор символов по умолчанию и управлять другими параметрами рендеринга.

```java
com.aspose.html.services.IUserAgentService userAgent = configuration.getService(com.aspose.html.services.IUserAgentService.class);
```

## Шаг 4: Определить и применить пользовательскую таблицу стилей  
Теперь задаём правила CSS, которые будут применяться к HTML при рендеринге. Здесь мы **используем сервис пользовательского агента** для установки таблицы стилей.

```java
userAgent.setUserStyleSheet("h1 { color:#a52a2a; font-size:2em; }\r\n" +
        "p { background-color:GhostWhite; color:SlateGrey; font-size:1.2em; }\r\n");
```

> **Почему это важно:** Применяя таблицу стилей на уровне пользовательского агента, вы гарантируете, что стили будут учтены даже если оригинальный HTML не ссылается на CSS‑файл.

## Шаг 5: Загрузить HTML‑документ с пользовательской конфигурацией  
Передайте как путь к файлу, так и экземпляр `Configuration` в конструктор `HTMLDocument`. Это привяжет пользовательскую таблицу стилей к документу.

```java
com.aspose.html.HTMLDocument document = new com.aspose.html.HTMLDocument("document.html", configuration);
```

## Шаг 6: Конвертировать HTML в PDF  
Когда документ полностью стилизован, вызовите статический метод `convertHTML` для **конвертации HTML в PDF**. Объект `PdfSaveOptions` позволяет тонко настроить вывод (например, размер страницы, сжатие).

```java
com.aspose.html.converters.Converter.convertHTML(
        document,
        new com.aspose.html.saving.PdfSaveOptions(),
        "user-agent-stylesheet_out.pdf"
);
```

> **Результат:** `user-agent-stylesheet_out.pdf` будет содержать заголовок коричневого цвета и абзац с фоном GhostWhite, точно как указано в таблице стилей.

## Шаг 7: Очистить ресурсы  
Всегда освобождайте объекты Aspose, чтобы освободить нативную память.

```java
if (document != null) {
    document.dispose();
}
if (configuration != null) {
    configuration.dispose();
}
```

## Распространённые проблемы и решения
| Проблема | Причина | Решение |
|----------|---------|---------|
| **Пустой PDF‑файл** | Таблица стилей не применена или документ загружен без конфигурации. | Убедитесь, что `configuration` передаётся в `HTMLDocument` и что `setUserStyleSheet` вызывается до загрузки. |
| **Предупреждение о неподдерживаемом свойстве CSS** | Aspose.HTML не поддерживает некоторые продвинутые свойства CSS. | Используйте только свойства, перечисленные в документации Aspose.HTML, или замените их более простыми стилями. |
| **FileNotFoundException** | Неправильный путь к `document.html`. | Укажите абсолютный путь или разместите HTML‑файл в корне проекта. |

## Часто задаваемые вопросы

**Q:** Можно ли применять разные стили к разным элементам HTML?  
**A:** Конечно! Вы можете определить любое количество правил CSS в пользовательской таблице стилей.

**Q:** Что делать, если нужно менять таблицу стилей динамически?  
**A:** Вызовите `setUserStyleSheet` снова перед созданием нового экземпляра `HTMLDocument`; новые стили будут применены при следующей конвертации.

**Q:** Можно ли использовать внешние CSS‑файлы с Aspose.HTML для Java?  
**A:** Да, вы можете либо подключить внешний файл стилей в HTML, либо загрузить его содержимое и передать в `setUserStyleSheet`.

**Q:** Как Aspose.HTML обрабатывает неподдерживаемые свойства CSS?  
**A:** Неподдерживаемые свойства игнорируются, позволяя остальной части таблицы стилей отрисоваться без ошибок.

**Q:** Можно ли конвертировать HTML в форматы, отличные от PDF?  
**A:** Да, Aspose.HTML поддерживает конвертацию в XPS, TIFF, PNG, JPEG и другие форматы с помощью соответствующего класса `SaveOptions`.
