# Отображение HTML в PNG на Java – Полное руководство

Когда‑нибудь вам нужно было **отобразить HTML в PNG** на Java, работая с защищённым эндпоинтом? С Aspose.HTML вы можете загрузить защищённую страницу, **добавить токен доступа**, и **сохранить HTML как PNG** — всё это в нескольких строках.  

В этом руководстве мы пройдём каждый шаг: от подготовки пользовательских заголовков до преобразования загруженного документа в чёткий PNG‑файл. К концу вы получите готовую к запуску программу, способную отобразить любую аутентифицированную HTML‑страницу в виде изображения на диске.

## Что вы узнаете

* Как **добавить токен доступа** к HTTP‑запросу, используя `Map` заголовков.  
* Точный синтаксис **как передать значения заголовков** в `HTMLDocument`.  
* Самый простой способ **сохранить HTML как PNG** с помощью `Converter` из Aspose.HTML.  
* Советы по устранению распространённых проблем (например, ошибки сертификатов, отсутствие ресурсов).  

Никаких внешних инструментов, никакой магии — только чистый Java, несколько импортов и библиотека Aspose.HTML (версия 23.10 или новее).  

### Предварительные требования

* Установлен Java 8 или новее.  
* JAR‑файл Aspose.HTML for Java в вашем classpath.  
* Действительный токен доступа для целевого сайта (можно получить через OAuth2, API‑key и т.п.).  

Если вы знакомы с базовым синтаксисом Java, вы готовы приступить.  

## Отображение HTML в PNG – Обзор

Суть проста: создаём `HTMLDocument`, который умеет получать страницу **с пользовательскими заголовками**, затем передаём этот документ в `Converter.convertHtmlToPng`. Конвертер берёт на себя всю тяжёлую работу — разметку, CSS, изображения, шрифты — и вы получаете пиксель‑точный снимок.

```java
import com.aspose.html.*;
import java.util.*;

public class AuthenticatedLoad {
    public static void main(String[] args) throws Exception {

        // Step 1: Prepare the HTTP headers with the bearer token
        Map<String, String> authHeaders = new HashMap<>();
        authHeaders.put("Authorization", "Bearer my-secure-token");

        // Step 2: Load the protected HTML page using the custom headers
        String protectedUrl = "https://secure.example.com/report.html";
        HTMLDocument document = new HTMLDocument(protectedUrl, authHeaders);

        // Step 3: Render the loaded page to a PNG file to verify the result
        String outputImagePath = "YOUR_DIRECTORY/secure.png";
        Converter.convertHtmlToPng(document, outputImagePath);

        System.out.println("Page rendered and saved to: " + outputImagePath);
    }
}
```

Этот фрагмент кода — полное решение, но разберём, почему каждая строка важна.

## Добавление токена доступа к HTTP‑запросу

Когда сайт защищает свои ресурсы, он ожидает заголовок `Authorization` вида `Bearer <token>`. Поместив этот заголовок в `Map<String,String>` и передав карту в конструктор `HTMLDocument`, Aspose.HTML автоматически вставляет заголовок во все свои запросы (HTML, CSS, изображения, AJAX‑вызовы).

```java
Map<String, String> authHeaders = new HashMap<>();
authHeaders.put("Authorization", "Bearer my-secure-token");
```

**Почему использовать карту?**  
* Она типобезопасна и позволяет добавить *любое* количество пользовательских заголовков — идеально для API, которым нужен `X‑API‑KEY` или `Accept-Language`.  
* Карта переиспользуется для всех последующих запросов ресурсов, обеспечивая единообразную аутентификацию.

> **Pro tip:** Если ваш токен истекает через короткий промежуток времени, обновите его перед созданием `HTMLDocument`. Иначе вы получите ошибку 401, и PNG будет пустым.

## Как передать заголовок с помощью пользовательских заголовков

Перегрузка `HTMLDocument` в Aspose.HTML, принимающая `Map`, — самый чистый способ **использовать пользовательские заголовки**. Можно также вручную задействовать `HttpClient`, но это добавит лишней сложности.

```java
HTMLDocument document = new HTMLDocument(protectedUrl, authHeaders);
```

Внутри библиотека формирует внутренний `HttpWebRequest` и копирует каждую запись из `authHeaders`. Это значит, что вы также можете передать такие заголовки, как:

```java
authHeaders.put("User-Agent", "MyApp/1.0");
authHeaders.put("Accept-Language", "en-US");
```

Если нужно **добавить токен доступа** условно (например, только для определённых доменов), просто проверьте URL перед заполнением карты.

## Сохранение HTML в файл PNG

Последний шаг — где происходит магия: преобразование полностью загруженного DOM в растровое изображение. `Converter.convertHtmlToPng` автоматически обрабатывает разметку, DPI и глубину цвета.

```java
String outputImagePath = "YOUR_DIRECTORY/secure.png";
Converter.convertHtmlToPng(document, outputImagePath);
```

Вы можете настроить качество вывода, передав `PngDevice` с пользовательскими параметрами, но значение по умолчанию подходит для большинства сценариев.

**Ожидаемый результат:** PNG‑файл, расположенный по пути `YOUR_DIRECTORY/secure.png`, который выглядит точно так же, как страница в браузере (за исключением интерактивного JavaScript).

## Проверка отрендеренного изображения

После завершения программы откройте PNG в любом просмотрщике изображений. Если на странице отображается экран входа или сообщение об ошибке 401, проверьте:

1. Токен доступа всё ещё действителен.  
2. Правильное написание заголовка `Authorization` (`Bearer` + пробел).  
3. Целевой URL доступен с вашего компьютера (фаервол, VPN и т.п.).  

Если всё согласовано, вы увидите защищённый отчёт, отрендеренный пиксель‑в‑пиксель.

## Пограничные случаи и распространённые подводные камни

| Ситуация | Что проверить | Предлагаемое решение |
|-----------|---------------|----------------------|
| **Самоподписанный SSL‑сертификат** | `SSLHandshakeException` | Добавьте пользовательский `TrustManager` или запустите Java с параметром `-Djavax.net.ssl.trustStore`, указывающим на сертификат. |
| **Большая страница (более 10 МБ)** | Переполнение памяти | Увеличьте размер кучи JVM (`-Xmx2g`) или рендерите только конкретный элемент, используя `document.getElementById`. |
| **Динамический контент, загруженный через AJAX** | Отсутствие контента в PNG | Вызовите `document.waitForLoad()` перед конвертацией или включите `HtmlLoadOptions.setEnableJavaScript(true)`. |
| **Отсутствующие шрифты** | Текст отображается шрифтом‑заменой | Установите необходимые шрифты на хосте или внедрите их через CSS `@font-face`. |

Раннее решение этих вопросов экономит часы отладки позже.
