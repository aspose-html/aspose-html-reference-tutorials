# Как установить DPI при конвертации SVG в PNG с помощью Java

Когда‑нибудь задумывались **how to set DPI** при конвертации SVG‑в‑PNG, чтобы результат выглядел чётко на готовом к печати плакате? Вы не одиноки. Во многих проектах — подумайте о рекламных листовках, макетах UI или технических схемах — получение PNG высокого разрешения из SVG является обязательным.  

В этом руководстве мы пройдём точные шаги, чтобы **convert SVG to PNG**, **export high resolution PNG**, а главное — управлять DPI с помощью библиотеки Aspose.HTML for Java. К концу вы получите переиспользуемый фрагмент кода, который можно вставить в любое Java‑приложение, будь то настольный инструмент или серверный сервис.

## Что вам понадобится

- **Java 8+** (код компилируется любой современной JDK)
- **Aspose.HTML for Java** JAR‑ы (доступны в Maven Central или на сайте Aspose)
- SVG‑файл, который вы хотите растрировать  
- Капля любопытства — больше ничего.

Если вы знакомы с Maven, просто добавьте зависимость, показанную в следующем разделе; иначе скачайте JAR‑ы вручную и добавьте их в classpath.

## Шаг 1: Добавьте библиотеку Java для конвертации изображений

Прежде всего — вашему проекту нужна **java image conversion library**, которая умеет читать SVG и записывать PNG. Aspose.HTML — надёжный выбор, потому что он поддерживает CSS, шрифты и сложные векторные возможности «из коробки».

**Maven‑пользователи** могут вставить это в ваш `pom.xml`:

```xml
<dependency>
    <groupId>com.aspose</groupId>
    <artifactId>aspose-html</artifactId>
    <version>23.12</version> <!-- use the latest stable version -->
</dependency>
```

**Пользователи Gradle** могут добавить:

```gradle
implementation 'com.aspose:aspose-html:23.12'
```

Если вы предпочитаете ручной способ, скачайте JAR со страницы загрузки Aspose и поместите его в `libs/`, затем добавьте в путь сборки вашей IDE.

> **Pro tip:** Следите за номером версии; более новые релизы часто улучшают обработку DPI и исправляют редкие баги.

## Шаг 2: Создайте параметры конвертации и задайте нужный DPI

Теперь, когда библиотека подключена, нам нужно указать, *как* должен выглядеть итоговый PNG. Здесь в игру вступает основной запрос — **how to set DPI**. Класс `ImageConversionOptions` даёт детальный контроль над горизонтальным (`dpiX`) и вертикальным (`dpiY`) разрешением.

```java
import com.aspose.html.converters.ImageConversionOptions;

// Step 2: Build the options object and set DPI
ImageConversionOptions conversionOptions = new ImageConversionOptions();
conversionOptions.setDpiX(300); // 300 DPI horizontally
conversionOptions.setDpiY(300); // 300 DPI vertically
```

Почему 300 DPI? Для большинства печатных процессов 300 точек‑на‑дюйм считается «печатным качеством». Если вы создаёте только веб‑активы, можете снизить значение до 72 или 96 DPI, чтобы уменьшить размер файлов.

> **What if I need a different DPI for width and height?**  
> Просто измените `setDpiX` и `setDpiY` независимо друг от друга. Библиотека поддерживает неоднородные значения, что удобно для анаморфных изображений.

## Шаг 3: Выполните конвертацию SVG‑в‑PNG

С готовыми параметрами сама конвертация — это один статический вызов. Мы будем использовать `java.nio.file.Paths`, чтобы оставаться кроссплатформенными.

```java
import com.aspose.html.converters.Converter;
import java.nio.file.Paths;

public class SvgToPng {
    public static void main(String[] args) throws Exception {
        // Paths to input SVG and output PNG
        var inputSvg  = Paths.get("YOUR_DIRECTORY/input.svg").toUri();
        var outputPng = Paths.get("YOUR_DIRECTORY/output.png").toUri();

        // Convert using the DPI settings we defined earlier
        Converter.convert(inputSvg, outputPng, conversionOptions);
    }
}
```

И всё — запустите метод `main`, и в `YOUR_DIRECTORY` появится чёткий PNG с 300 DPI. Библиотека автоматически масштабирует векторную графику под указанный DPI, сохраняя соотношение сторон и чёткость текста.

## Шаг 4: Проверьте результат (необязательно, но рекомендуется)

Быстрая проверка может избавить от головной боли позже. Откройте полученный PNG в любой программе‑просмотрщике, показывающей DPI‑метаданные (например, Photoshop, GIMP или даже вкладка «Details» в Проводнике Windows). Вы должны увидеть **300 DPI** в полях горизонтального и вертикального разрешения.

Если файл выглядит размытым, проверьте:

1. Оригинальный SVG не является низкокачественным изначально (векторные файлы независимы от разрешения, но встроенные растровые изображения могут ограничивать качество).  
2. Вы действительно сохранили файл после установки DPI — иногда IDE кэшируют старые сборки.  
3. Параметры конвертации не были переопределены где‑то ещё в коде.

## Почему DPI важно для Export High Resolution PNG

Вы можете спросить: «Зачем вообще DPI? Разве PNG не состоит только из пикселей?» На самом деле DPI — это *метаданные*, которые сообщают downstream‑приложениям (особенно печатным) сколько пикселей соответствует одному дюйму физического пространства. Если передать принтеру PNG 1200 × 800 без правильного DPI, принтер может предположить значение по умолчанию (часто 72 DPI) и масштабировать изображение, что приводит к размытому выводу.

Корректная установка DPI также экономит ресурсы: вам не придётся позже увеличивать растровые изображения, что дорого по вычислениям и ухудшает качество.

## Edge Cases & Common Pitfalls

| Ситуация | На что обратить внимание | Как исправить |
|-----------|-------------------|------------|
| **SVG содержит встроенные растровые изображения** | Встроенный растровый файл может быть низкого разрешения, из‑за чего итоговый PNG будет пикселизирован даже при высоком DPI. | Замените растровый файл на версию более высокого разрешения или отредактируйте SVG, чтобы ссылаться на внешнее изображение. |
| **Большие значения DPI (например, 1200 DPI)** | Потребление памяти резко возрастает; может возникнуть `OutOfMemoryError`. | Увеличьте размер кучи JVM (`-Xmx2g`) или ограничьте DPI разумным максимумом для вашего случая. |
| **Неквадратные пиксели** | Некоторые дисплеи интерпретируют DPI иначе, что приводит к растянутым изображениям. | Убедитесь, что `setDpiX` равно `setDpiY`, если только у вас нет особой причины их различать. |
| **Отсутствие шрифтов** | Текст в SVG может перейти к шрифту по умолчанию, изменив макет. | Встроите шрифты в SVG или установите необходимые шрифты на машине выполнения. |

## Краткое резюме (в одном предложении)

Чтобы **how to set DPI** для конвертации SVG‑в‑PNG, создайте объект `ImageConversionOptions`, задайте `dpiX`/`dpiY` и вызовите `Converter.convert` из библиотеки Aspose.HTML Java.

## Следующие шаги и связанные темы

- **Пакетная обработка:** Пройдитесь по каталогу SVG‑файлов и примените одинаковые настройки DPI.  
- **Динамический DPI:** Считайте значение из конфигурационного файла или аргумента командной строки, чтобы задавать DPI во время выполнения.  
- **Альтернативные библиотеки:** Если вы не можете использовать Aspose, рассмотрите **Batik** (Apache) или **SVG Salamander**, хотя им может потребоваться дополнительный код для работы с DPI.  
- **Export high resolution PNG** для Android‑ресурсов: комбинируйте эту технику с папками Android `drawable‑hdpi`, `xhdpi` и т.д.

Экспериментируйте — попробуйте 72 DPI для веб‑миниатюр, 600 DPI для крупноформатных печатных материалов или даже 150 DPI как компромисс. Код остаётся тем же; меняются только цифры.
