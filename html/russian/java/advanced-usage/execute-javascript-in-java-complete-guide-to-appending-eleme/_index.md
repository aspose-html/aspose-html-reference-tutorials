# Выполнение JavaScript в Java – Полное руководство по добавлению элементов в body

Когда‑нибудь вам нужно было **execute JavaScript in Java**, но вы не знали, как соединить эти два мира? Вы не одиноки. Многие разработчики Java сталкиваются с проблемой, когда хотят выполнить **JavaScript DOM manipulation** без запуска полноценного браузера. Хорошая новость? С Aspose.HTML вы можете выполнить скрипт, **append element to body**, и сохранить изменённую разметку — всё из чистого кода Java.

В этом руководстве мы пройдём всё, что вам нужно знать: загрузка HTML‑файла, написание небольшого скрипта, который **modifies HTML with JavaScript**, выполнение этого скрипта из Java и, наконец, сохранение изменений. К концу вы получите автономный, готовый к запуску пример, который можно добавить в любой проект.

## Что понадобится

- **Java 8** или новее (API работает с Java 8+)
- **Aspose.HTML for Java** library (version 22.12 or later)
- Простой файл шаблона HTML (мы создадим его на лету)
- IDE или текстовый редактор — ничего сложного, даже обычный текстовый редактор подойдёт

> **Подсказка:** если вы используете Maven, добавьте зависимость Aspose.HTML в ваш `pom.xml`. В противном случае просто поместите JAR‑файлы в ваш classpath.

## Шаг 1 – Загрузка HTML‑документа (Подготовка к выполнению JavaScript в Java)

Первое, что мы делаем, — создаём экземпляр `HTMLDocument`, указывающий на исходный файл. Считайте этот объект мостом между Java и деревом DOM, к которому позже будет обращаться скрипт.

```java
import com.aspose.html.HTMLDocument;

// Step 1: Load the HTML document you want to manipulate
HTMLDocument htmlDoc = new HTMLDocument("YOUR_DIRECTORY/template.html");
```

> **Почему это важно:** Класс `HTMLDocument` парсит файл в представление DOM, с которым может взаимодействовать движок скриптов Aspose. Без предварительной загрузки документа скрипт не будет иметь чего изменять.

## Шаг 2 – Написание JavaScript, который **Append Element to Body**

Теперь мы создаём небольшой фрагмент, который создаёт элемент `<p>` и добавляет его в `<body>`. Это классическая **JavaScript DOM manipulation**, но мы запустим её из Java.

```java
// Step 2: JavaScript that adds a new paragraph to the body
String jsCode = ""
    + "var p = document.createElement('p');"
    + "p.textContent = 'Generated by JavaScript';"
    + "document.body.appendChild(p);";
```

> **Почему это работает:** `document.createElement` создаёт новый узел, `textContent` задаёт его внутренний текст, а `appendChild` присоединяет его к существующему элементу `<body>`. Скрипт — чистый клиент‑сайд JavaScript; Aspose.HTML исполняет его в безголовом окружении.

## Шаг 3 – Создание Script Engine (Движок, который **Run JavaScript from Java**)

Aspose.HTML поставляется с `ScriptEngine`, который может оценивать JavaScript относительно загруженного DOM. Это часть, которая действительно **run JavaScript from Java**.

```java
import com.aspose.html.scripting.ScriptEngine;

// Step 3: Bind the engine to the document we just loaded
ScriptEngine scriptEngine = new ScriptEngine(htmlDoc);
```

> **Зачем это нужно:** Движок изолирует выполнение скрипта, не позволяя ему влиять на основной процесс Java. Он также возвращает `ScriptExecutionResult`, чтобы вы могли проверить успешность.

## Шаг 4 – Выполнение скрипта и проверка результата

Когда всё настроено, мы наконец запускаем скрипт. Метод `execute` возвращает объект результата, который сообщает, прошёл ли скрипт без ошибок.

```java
import com.aspose.html.scripting.ScriptExecutionResult;

// Step 4: Run the JavaScript and verify it succeeded
ScriptExecutionResult execResult = scriptEngine.execute(jsCode);
if (execResult.isSuccess()) {
    System.out.println("Script executed successfully.");
} else {
    System.err.println("Script failed: " + execResult.getErrorMessage());
}
```

> **Пограничный случай:** Если скрипт бросит исключение (например, синтаксическую ошибку), `isSuccess()` будет `false`, а `getErrorMessage()` предоставит детали. Всегда проверяйте это в продакшн‑коде.

## Шаг 5 – Сохранение изменённого документа (Завершение **Modify HTML with JavaScript**)

После того как скрипт изменил DOM, мы сохраняем изменения на диск. Получившийся файл теперь содержит новый абзац.

```java
// Step 5: Write the updated DOM back to an HTML file
htmlDoc.save("YOUR_DIRECTORY/modified.html");
System.out.println("Document saved.");
```

> **Проверка результата:** Откройте `modified.html` в любом браузере. Вы должны увидеть оригинальное содержимое плюс новую строку «Generated by JavaScript».

## Шаг 6 – Полный рабочий пример (Все шаги вместе)

Ниже приведена полная программа, которую можно скопировать в файл `JsDomManipulation.java`. В ней включены все импорты, обработка ошибок и комментарии для ясности.

```java
import com.aspose.html.HTMLDocument;
import com.aspose.html.scripting.ScriptEngine;
import com.aspose.html.scripting.ScriptExecutionResult;

public class JsDomManipulation {
    public static void main(String[] args) throws Exception {

        // 1️⃣ Load the HTML document to be manipulated
        HTMLDocument htmlDoc = new HTMLDocument("YOUR_DIRECTORY/template.html");

        // 2️⃣ Define JavaScript that adds a new paragraph to the <body>
        String jsCode = "var p = document.createElement('p');"
                      + "p.textContent = 'Generated by JavaScript';"
                      + "document.body.appendChild(p);";

        // 3️⃣ Create a script engine bound to the loaded document
        ScriptEngine scriptEngine = new ScriptEngine(htmlDoc);

        // 4️⃣ Execute the JavaScript and report success
        ScriptExecutionResult execResult = scriptEngine.execute(jsCode);
        if (execResult.isSuccess()) {
            System.out.println("Script executed successfully.");
        } else {
            System.err.println("Script failed: " + execResult.getErrorMessage());
            return; // Stop if the script didn't run
        }

        // 5️⃣ Save the modified document back to HTML
        htmlDoc.save("YOUR_DIRECTORY/modified.html");
        System.out.println("Document saved.");
    }
}
```

### Ожидаемый вывод

Запуск программы выводит:

```
Script executed successfully.
Document saved.
```

А `modified.html` теперь содержит примерно следующее:

```html
<!DOCTYPE html>
<html>
<head>
    <title>Template</title>
</head>
<body>
    <h1>Original Content</h1>
    <p>Generated by JavaScript</p>
</body>
</html>
```

## Часто задаваемые вопросы и подводные камни

| Question | Answer |
|----------|--------|
| **Can I run multiple scripts sequentially?** | Да. Просто вызовите `scriptEngine.execute()` снова с новой строкой скрипта. Один и тот же экземпляр `HTMLDocument` сохраняет своё состояние. |
| **What if the HTML file is missing a `<body>` tag?** | Aspose.HTML автоматически создаст элемент `<body>` во время парсинга, так что скрипт всё равно будет работать. Однако для предсказуемых результатов лучше начинать с корректного шаблона. |
| **Is this approach compatible with Java 17?** | Абсолютно. Библиотека нацелена на Java 8+, поэтому более новые среды выполнения подходят. |
| **Do I need a full browser environment?** | Нет. Aspose.HTML предоставляет лёгкий безголовый движок, который интерпретирует стандартные API DOM без запуска Chrome или Firefox. |
| **How do I handle large HTML files?** | Загружайте документ как поток (`new HTMLDocument(InputStream)`) чтобы избежать загрузки всего файла в память сразу. |
