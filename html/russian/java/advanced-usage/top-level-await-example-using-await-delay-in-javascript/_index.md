# пример top level await – использование await delay в JavaScript

Когда‑нибудь задумывались, как приостановить выполнение модуля без обёртывания всего в async IIFE? Именно это и позволяет **пример top level await**. В этом руководстве мы пройдёмся по небольшому веб‑страничному примеру, который использует `await delay javascript` для откладывания работы, а затем создаёт `increment counter class`, использующий **public class fields javascript**. К концу вы получите полностью готовый фрагмент кода, который можно скопировать и вставить в любой современный браузер.

Мы рассмотрим всё: от определения класса с публичным полем до подключения простого помощника задержки на основе промисов. Никаких внешних библиотек, без сборки — только чистый HTML, `<script type="module">` и новейшие возможности ECMAScript. Если вы уже знакомы с async‑функциями, то увидите, почему top‑level await выглядит как естественное продолжение. Если вы только знакомы с синтаксисом **javascript class public field**, не переживайте — мы объясним причину каждой строки.

## Что вы узнаете

- Как работает **пример top level await** внутри модульного скрипта.  
- Паттерн для помощника `await delay javascript`, имитирующего `setTimeout` с помощью промисов.  
- Как написать **increment counter class**, использующий публичное поле (`count`) и статический блок инициализации.  
- Ожидаемый вывод в консоль и как проверить, что статический блок отрабатывает до остального скрипта.  
- Советы по устранению распространённых проблем, например, в старых браузерах, не поддерживающих top‑level await.

> **Pro tip:** Современные браузеры (Chrome 106+, Firefox 102+, Edge 106+) уже поддерживают top‑level await. Если нужно поддерживать более старые среды, рассмотрите возможность сборки с помощью Vite или Babel.

## Шаг 1 – Определить класс с публичным полем и статическим блоком

Первая часть нашего демо‑примерa — класс, хранящий числовой счётчик. Обратите внимание на строку `count = 0;` — это синтаксис **public class fields javascript**, введённый в ES2022. Он создаёт свойство экземпляра без конструктора.

```html
<!DOCTYPE html>
<html>
<head>
    <title>ECMAScript 2022 Demo</title>
</head>
<body>
<img src="placeholder.png" alt="top level await example screenshot" title="top level await example">

<script type="module">
    // Step 1: Class definition
    class Counter {
        // Public field holds the current count
        count = 0;

        // Static block runs once when the class is first evaluated
        static {
            console.log('Counter class initialized');
        }

        // Simple method that increments the public field
        increment() {
            this.count++;
        }
    }
```

**Зачем нужен статический блок?** Он позволяет выполнить одноразовую инициализацию (например, логирование) в момент парсинга модуля, *до* выполнения любого top‑level await. Это гарантирует, что сообщение появится первым в консоли, подтверждая раннюю оценку класса.

## Шаг 2 – Создать помощник `await delay javascript`

Далее нам нужен небольшой помощник задержки на основе промисов. По сути это обёртка над `setTimeout`, но возвращающая промис, что делает её await‑able.

```javascript
    // Step 2: Promise‑based delay helper
    const delay = ms => new Promise(resolve => setTimeout(resolve, ms));
```

**Крайний случай:** Если `ms` отрицательно или не является числом, `setTimeout` трактует его как `0`. Можно добавить проверку, но для демо‑примерa однострочник остаётся более читаемым.

## Шаг 3 – Использовать Top‑Level Await для паузы выполнения

Теперь звезда шоу: top‑level await. Поскольку наш скрипт является модулем (`type="module"`), мы можем разместить `await` непосредственно в верхнем уровне. Это приостанавливает модуль, пока промис не разрешится, позволяя контролировать порядок побочных эффектов.

```javascript
    // Step 3: Top‑level await – delay half a second
    await delay(500);   // Wait 500 ms so the static block logs first
```

**Распространённый вопрос:** «Можно ли использовать top‑level await в обычном `<script>`?» Нет — поддержка есть только у модульных скриптов. Если забыть `type="module"`, получим синтаксическую ошибку.

## Шаг 4 – Создать экземпляр класса, увеличить счёт и вывести результат

Наконец, собираем всё вместе: создаём экземпляр, вызываем `increment()`, выводим окончательное значение счётчика. Это демонстрирует **increment counter class** в действии.

```javascript
    // Step 4: Work with the Counter instance
    const counterInstance = new Counter();
    counterInstance.increment();               // count becomes 1
    console.log('Final count:', counterInstance.count);
</script>
</body>
</html>
```

### Ожидаемый вывод в консоль

```
Counter class initialized
Final count: 1
```

Если открыть страницу в DevTools, вы увидите сообщение из статического блока **до** завершения задержки, подтверждая, что класс был оценён первым.

## Шаг 5 – Почему использовать публичные поля класса вместо конструктора?

Возможно, вы задаётесь вопросом, почему мы не написали так:

```javascript
class Counter {
    constructor() {
        this.count = 0;
    }
}
```

Оба подхода работают, но **public class fields javascript** дают более чистый, декларативный синтаксис. Поле объявляется один раз, а не внутри каждого вызова конструктора, что упрощает чтение при росте класса. Кроме того, статические блоки нельзя разместить внутри конструктора, поэтому вынесение инициализационной логики становится более естественным.

## Шаг 6 – Адаптация примера для реальных приложений

В продакшене часто требуется больше одного счётчика. Вот несколько быстрых идей:

- **Несколько счётчиков:** Хранить их в `Map` с ключом‑идентификатором.  
- **Сохранённое состояние:** Заменить `console.log` на запись в `localStorage`.  
- **Асинхронная инициализация:** Использовать статический блок для получения конфигурации с сервера до создания любых экземпляров.

Все эти паттерны продолжают выигрывать от top‑level await, потому что вы можете один раз загрузить данные при загрузке модуля и гарантировать их готовность для всех потребителей.

## Часто задаваемые вопросы

**В опрос:** Блокирует ли top‑level await другие модули?  
**О ответ:** Нет. Каждый модуль работает независимо. Приостановлен только тот модуль, где находится await; остальные продолжают загружаться параллельно.

**В опрос:** Что если промис задержки отклонится?  
**О ответ:** Необработанное отклонение прервет оценку модуля и отобразится как ошибка в консоли. Оберните `await` в `try…catch`, если нужен плавный откат.

**В опрос:** Требуется ли ключевое слово `static`?  
**О ответ:** Не для самого счётчика, но статический блок удобен для демонстрации кода, который выполняется *один раз* при загрузке — отлично подходит для логирования или обнаружения возможностей.

Полный, исполняемый фрагмент кода находится выше, а вывод в консоль подтверждает, что статический блок отрабатывает до кода с задержкой.
