## Как создать HTML — пошаговое руководство

### Шаг 1: Инициализировать пустой HTML‑документ
Первое, что нужно сделать, — создать новый экземпляр `HTMLDocument`. Этот класс служит холстом, на котором будет размещён весь ваш HTML‑контент.

```java
com.aspose.html.HTMLDocument document = new com.aspose.html.HTMLDocument();
```

Здесь мы создаём объект HTML‑документа с именем `document`. Представьте, что это чистый лист бумаги, на котором вы будете писать.

### Шаг 2: Подготовить путь для сохранения документа
Далее необходимо указать, куда будет сохраняться наш новый HTML‑документ. Важно задать чёткий путь к файлу.

```java
String documentPath = "create-new-document.html";
```

Здесь мы сохраняем имя файла и его расположение в переменной `documentPath`. При желании можете изменить имя файла, но не забудьте оставить расширение `.html`, чтобы файл распознавался как HTML‑документ.

### Шаг 3: Создать текстовый элемент и добавить его в документ
Теперь самая интересная часть! Добавим содержимое в наш только‑что созданный HTML‑документ. Мы создадим текстовый узел с сообщением «Hello World!» и присоединим его к телу документа.

```java
com.aspose.html.dom.Text text = document.createTextNode("Hello World!");
document.getBody().appendChild(text);
```

На этом этапе мы используем метод `createTextNode` для создания нового текстового элемента. Этот метод аналогичен написанию первой строки текста на чистом листе бумаги. Затем методом `appendChild` присоединяем этот текст к `body` нашего HTML‑документа, фактически размещая его на холсте.

### Шаг 4: Сохранить документ на диск
Контент готов — пора сохранять работу! Этот шаг важен, иначе все усилия пропадут.

```java
document.save(documentPath);
```

Вызов метода `save` инструктирует программу записать текущее состояние документа по указанному пути, тем самым создавая реальный файл `.html` на диске. Сохранённый документ — как готовая картина: он постоянен и готов к показу миру!

### Шаг 5: Освободить ресурсы документа
В Java рекомендуется освобождать ресурсы, когда они больше не нужны, чтобы избежать утечек памяти. Поэтому мы освободим объект документа.

```java
finally {
    if (document != null) {
        document.dispose();
    }
}
```

С помощью `dispose()` мы гарантируем, что память, выделенная под `document`, будет возвращена системе. Этот завершающий шаг важен для стабильной работы вашего приложения в долгосрочной перспективе.

## Распространённые проблемы и их решения
| Проблема | Причина | Решение |
|----------|---------|----------|
| **`NullPointerException` при `document.dispose()`** | `document` не был создан из‑за предыдущей ошибки. | Оберните инициализацию в `try‑catch` и убедитесь, что `document` создан до выполнения блока `finally`. |
| **Сохранённый файл пустой** | Не было добавлено ни одного узла перед вызовом `save()`. | Добавьте хотя бы один элемент (например, текстовый узел или `<div>`) перед сохранением. |
| **Файл не найден при последующем чтении** | Неправильный относительный путь. | Используйте абсолютный путь или `Paths.get(System.getProperty("user.dir"), "create-new-document.html")`. |
