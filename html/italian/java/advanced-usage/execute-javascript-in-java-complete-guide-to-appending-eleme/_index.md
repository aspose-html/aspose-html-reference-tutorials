# Eseguire JavaScript in Java – Guida completa per aggiungere elementi al body

Hai mai avuto bisogno di **eseguire JavaScript in Java** ma non sapevi come collegare i due mondi? Non sei l'unico. Molti sviluppatori Java si trovano in difficoltà quando vogliono effettuare **manipolazione del DOM con JavaScript** senza avviare un browser completo. La buona notizia? Con Aspose.HTML puoi eseguire uno script, **aggiungere un elemento al body**, e salvare il markup modificato—tutto da puro codice Java.

In questo tutorial ti guideremo passo passo su tutto ciò che devi sapere: caricare un file HTML, scrivere un piccolo script che **modifica HTML con JavaScript**, eseguire quello script da Java e, infine, persistere le modifiche. Alla fine avrai un esempio autonomo e eseguibile che potrai inserire in qualsiasi progetto.

## Cosa ti servirà

- **Java 8** o versioni successive (l'API funziona con Java 8+)
- **Aspose.HTML for Java** library (versione 22.12 o successiva)
- Un semplice file modello HTML (ne creeremo uno al volo)
- Un IDE o un editor di testo—nulla di speciale, anche un semplice editor di testo andrà bene

> Pro tip: Se stai usando Maven, aggiungi la dipendenza Aspose.HTML al tuo `pom.xml`. Altrimenti, basta inserire i JAR nel tuo classpath.

## Passo 1 – Caricare il documento HTML (Prepararsi a eseguire JavaScript in Java)

La prima cosa che facciamo è creare un'istanza di `HTMLDocument` che punti al file di origine. Pensa a questo oggetto come al ponte tra Java e l'albero DOM che lo script toccherà in seguito.

```java
import com.aspose.html.HTMLDocument;

// Step 1: Load the HTML document you want to manipulate
HTMLDocument htmlDoc = new HTMLDocument("YOUR_DIRECTORY/template.html");
```

> **Perché è importante:** La classe `HTMLDocument` analizza il file in una rappresentazione DOM con cui il motore di scripting Aspose può interagire. Senza caricare prima il documento, lo script non ha nulla da modificare.

## Passo 2 – Scrivere JavaScript che **Aggiunge un elemento al body**

Ora creiamo un piccolo snippet che crea un elemento `<p>` e lo aggiunge al `<body>`. Si tratta di una classica **manipolazione del DOM con JavaScript**, ma lo eseguiremo da Java.

```java
// Step 2: JavaScript that adds a new paragraph to the body
String jsCode = ""
    + "var p = document.createElement('p');"
    + "p.textContent = 'Generated by JavaScript';"
    + "document.body.appendChild(p);";
```

> **Perché funziona:** `document.createElement` crea un nuovo nodo, `textContent` imposta il suo testo interno e `appendChild` lo collega all'elemento `<body>` esistente. Lo script è puro JavaScript client‑side; Aspose.HTML lo esegue in un ambiente headless.

## Passo 3 – Creare il motore di script (Il motore che **esegue JavaScript da Java**)

Aspose.HTML fornisce un `ScriptEngine` che può valutare JavaScript sul DOM caricato. È la componente che effettivamente **esegue JavaScript da Java**.

```java
import com.aspose.html.scripting.ScriptEngine;

// Step 3: Bind the engine to the document we just loaded
ScriptEngine scriptEngine = new ScriptEngine(htmlDoc);
```

> **Perché ti serve:** Il motore isola l'esecuzione dello script, impedendo che influisca sul tuo processo Java principale. Restituisce inoltre un `ScriptExecutionResult` così puoi verificare il successo.

## Passo 4 – Eseguire lo script e controllare il risultato

Con tutto pronto, finalmente avviamo lo script. Il metodo `execute` restituisce un oggetto risultato che indica se lo script è stato eseguito senza errori.

```java
import com.aspose.html.scripting.ScriptExecutionResult;

// Step 4: Run the JavaScript and verify it succeeded
ScriptExecutionResult execResult = scriptEngine.execute(jsCode);
if (execResult.isSuccess()) {
    System.out.println("Script executed successfully.");
} else {
    System.err.println("Script failed: " + execResult.getErrorMessage());
}
```

> **Caso limite:** Se lo script lancia un'eccezione (ad esempio un errore di sintassi), `isSuccess()` sarà `false` e `getErrorMessage()` fornirà i dettagli. Controlla sempre questo in codice di produzione.

## Passo 5 – Salvare il documento modificato (Completare **Modifica HTML con JavaScript**)

Dopo che lo script ha mutato il DOM, persistiamo le modifiche su disco. Il file risultante contiene ora il nuovo paragrafo.

```java
// Step 5: Write the updated DOM back to an HTML file
htmlDoc.save("YOUR_DIRECTORY/modified.html");
System.out.println("Document saved.");
```

> **Verifica del risultato:** Apri `modified.html` in qualsiasi browser. Dovresti vedere il contenuto originale più una nuova riga che recita “Generated by JavaScript”.

## Passo 6 – Esempio completo funzionante (Tutti i passaggi combinati)

Di seguito trovi il programma completo che puoi copiare‑incollare in un file `JsDomManipulation.java`. Include tutti gli import, la gestione degli errori e i commenti per chiarezza.

```java
import com.aspose.html.HTMLDocument;
import com.aspose.html.scripting.ScriptEngine;
import com.aspose.html.scripting.ScriptExecutionResult;

public class JsDomManipulation {
    public static void main(String[] args) throws Exception {

        // 1️⃣ Load the HTML document to be manipulated
        HTMLDocument htmlDoc = new HTMLDocument("YOUR_DIRECTORY/template.html");

        // 2️⃣ Define JavaScript that adds a new paragraph to the <body>
        String jsCode = "var p = document.createElement('p');"
                      + "p.textContent = 'Generated by JavaScript';"
                      + "document.body.appendChild(p);";

        // 3️⃣ Create a script engine bound to the loaded document
        ScriptEngine scriptEngine = new ScriptEngine(htmlDoc);

        // 4️⃣ Execute the JavaScript and report success
        ScriptExecutionResult execResult = scriptEngine.execute(jsCode);
        if (execResult.isSuccess()) {
            System.out.println("Script executed successfully.");
        } else {
            System.err.println("Script failed: " + execResult.getErrorMessage());
            return; // Stop if the script didn't run
        }

        // 5️⃣ Save the modified document back to HTML
        htmlDoc.save("YOUR_DIRECTORY/modified.html");
        System.out.println("Document saved.");
    }
}
```

### Output previsto

Eseguendo il programma stampa:

```
Script executed successfully.
Document saved.
```

E `modified.html` ora contiene qualcosa di simile a:

```html
<!DOCTYPE html>
<html>
<head>
    <title>Template</title>
</head>
<body>
    <h1>Original Content</h1>
    <p>Generated by JavaScript</p>
</body>
</html>
```

## Domande comuni e problemi

| Domanda | Risposta |
|----------|--------|
| **Posso eseguire più script in sequenza?** | Sì. Basta chiamare `scriptEngine.execute()` di nuovo con una nuova stringa di script. La stessa istanza di `HTMLDocument` mantiene il suo stato. |
| **Cosa succede se il file HTML manca del tag `<body>`?** | Aspose.HTML creerà automaticamente un elemento `<body>` durante il parsing, quindi lo script funziona comunque. Tuttavia, per risultati prevedibili, inizia con un modello ben formato. |
| **Questo approccio è compatibile con Java 17?** | Assolutamente. La libreria è destinata a Java 8+, quindi le versioni più recenti vanno bene. |
| **Ho bisogno di un ambiente browser completo?** | No. Aspose.HTML fornisce un motore leggero e headless che interpreta le API DOM standard senza avviare Chrome o Firefox. |
| **Come gestisco file HTML di grandi dimensioni?** | Carica il documento come stream (`new HTMLDocument(InputStream)`) per evitare di caricare l'intero file in memoria contemporaneamente. |
