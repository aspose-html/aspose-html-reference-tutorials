# Java-ban JavaScript futtatása – Teljes útmutató az elemek body-hoz való hozzáfűzéséhez

Szükséged volt már **JavaScript futtatására Java-ban**, de nem tudtad, hogyan hidalhatod át a két világot? Nem vagy egyedül. Sok Java fejlesztő elakad, amikor **JavaScript DOM manipulációt** szeretne végrehajtani anélkül, hogy teljes böngészőt indítana. A jó hír? Az Aspose.HTML segítségével futtathatsz egy szkriptet, **hozzáadhatsz egy elemet a body-hoz**, és elmentheted a módosított markupot – mindezt tisztán Java kódból.

Ebben az útmutatóban mindent végigvázolunk, amit tudnod kell: egy HTML fájl betöltése, egy apró szkript írása, amely **módosítja a HTML-t JavaScript‑el**, a szkript Java‑ból való futtatása, és végül a változások mentése. A végére egy önálló, futtatható példát kapsz, amelyet bármelyik projektbe beilleszthetsz.

## Amire szükséged lesz

- **Java 8** vagy újabb (az API Java 8+ verziókkal működik)
- **Aspose.HTML for Java** könyvtár (22.12 vagy újabb verzió)
- Egy egyszerű HTML sablonfájl (ezt a példában futás közben hozunk létre)
- IDE vagy szövegszerkesztő – semmi különleges, még egy egyszerű szövegszerkesztő is megfelel

> Pro tipp: Ha Maven‑t használsz, add hozzá az Aspose.HTML függőséget a `pom.xml` fájlodhoz. Egyébként csak helyezd a JAR‑okat a classpath‑ba.

## 1. lépés – HTML dokumentum betöltése (Java‑ban JavaScript futtatásának előkészítése)

Az első dolog, amit csinálunk, egy `HTMLDocument` példány létrehozása, amely a forrásfájlra mutat. Tekintsd ezt az objektumot a hídnak Java és a DOM‑fa között, amelyet a szkript később érinteni fog.

```java
import com.aspose.html.HTMLDocument;

// Step 1: Load the HTML document you want to manipulate
HTMLDocument htmlDoc = new HTMLDocument("YOUR_DIRECTORY/template.html");
```

> **Miért fontos:** A `HTMLDocument` osztály beolvassa a fájlt egy DOM reprezentációba, amelyet az Aspose szkriptmotor képes kezelni. Dokumentum betöltése nélkül nincs semmi, amit a szkript módosíthatna.

## 2. lépés – JavaScript írása, amely **hozzáfűz egy elemet a body‑hoz**

Most egy apró kódrészletet készítünk, amely létrehoz egy `<p>` elemet és hozzáadja a `<body>`-hoz. Ez klasszikus **JavaScript DOM manipuláció**, de Java‑ból futtatjuk.

```java
// Step 2: JavaScript that adds a new paragraph to the body
String jsCode = ""
    + "var p = document.createElement('p');"
    + "p.textContent = 'Generated by JavaScript';"
    + "document.body.appendChild(p);";
```

> **Miért működik:** A `document.createElement` új csomópontot hoz létre, a `textContent` beállítja a belső szöveget, az `appendChild` pedig csatolja a meglévő `<body>` elemhez. A szkript tisztán kliens‑oldali JavaScript; az Aspose.HTML egy fej nélküli környezetben hajtja végre.

## 3. lépés – Szkriptmotor létrehozása (A motor, amely **Java‑ból futtat JavaScript‑et**)

Az Aspose.HTML egy `ScriptEngine`‑et biztosít, amely képes JavaScript‑et értékelni a betöltött DOM‑on. Ez az a komponens, amely ténylegesen **Java‑ból futtatja a JavaScript‑et**.

```java
import com.aspose.html.scripting.ScriptEngine;

// Step 3: Bind the engine to the document we just loaded
ScriptEngine scriptEngine = new ScriptEngine(htmlDoc);
```

> **Miért van rá szükséged:** A motor elszigeteli a szkript futtatását, megakadályozva, hogy hatással legyen a fő Java folyamatodra. Emellett visszaad egy `ScriptExecutionResult`‑ot, amellyel ellenőrizheted a sikerességet.

## 4. lépés – A szkript végrehajtása és az eredmény ellenőrzése

Minden előkészítve, végre is hajtjuk a szkriptet. Az `execute` metódus egy eredményobjektust ad vissza, amely megmondja, hogy a szkript hibamentesen lefutott‑e.

```java
import com.aspose.html.scripting.ScriptExecutionResult;

// Step 4: Run the JavaScript and verify it succeeded
ScriptExecutionResult execResult = scriptEngine.execute(jsCode);
if (execResult.isSuccess()) {
    System.out.println("Script executed successfully.");
} else {
    System.err.println("Script failed: " + execResult.getErrorMessage());
}
```

> **Szél eset:** Ha a szkript kivételt dob (pl. szintaxis hiba), az `isSuccess()` **false** lesz, és a `getErrorMessage()` részletezi a problémát. Mindig ellenőrizd ezt a production kódban.

## 5. lépés – Módosított dokumentum mentése (Befejezés **HTML módosítása JavaScript‑el**)

Miután a szkript megváltoztatta a DOM‑ot, visszaírjuk a változásokat a lemezre. A keletkezett fájl már tartalmazza az új bekezdést.

```java
// Step 5: Write the updated DOM back to an HTML file
htmlDoc.save("YOUR_DIRECTORY/modified.html");
System.out.println("Document saved.");
```

> **Eredmény ellenőrzése:** Nyisd meg a `modified.html` fájlt bármelyik böngészőben. Látnod kell az eredeti tartalmat plusz egy új sort, amely azt írja: „Generated by JavaScript”.

## 6. lépés – Teljes működő példa (Minden lépés egyben)

Az alábbiakban a teljes program látható, amelyet egyszerűen beilleszthetsz egy `JsDomManipulation.java` fájlba. Tartalmazza az összes importot, hibakezelést és a tisztább megértéshez szükséges megjegyzéseket.

```java
import com.aspose.html.HTMLDocument;
import com.aspose.html.scripting.ScriptEngine;
import com.aspose.html.scripting.ScriptExecutionResult;

public class JsDomManipulation {
    public static void main(String[] args) throws Exception {

        // 1️⃣ Load the HTML document to be manipulated
        HTMLDocument htmlDoc = new HTMLDocument("YOUR_DIRECTORY/template.html");

        // 2️⃣ Define JavaScript that adds a new paragraph to the <body>
        String jsCode = "var p = document.createElement('p');"
                      + "p.textContent = 'Generated by JavaScript';"
                      + "document.body.appendChild(p);";

        // 3️⃣ Create a script engine bound to the loaded document
        ScriptEngine scriptEngine = new ScriptEngine(htmlDoc);

        // 4️⃣ Execute the JavaScript and report success
        ScriptExecutionResult execResult = scriptEngine.execute(jsCode);
        if (execResult.isSuccess()) {
            System.out.println("Script executed successfully.");
        } else {
            System.err.println("Script failed: " + execResult.getErrorMessage());
            return; // Stop if the script didn't run
        }

        // 5️⃣ Save the modified document back to HTML
        htmlDoc.save("YOUR_DIRECTORY/modified.html");
        System.out.println("Document saved.");
    }
}
```

### Várt kimenet

A program futtatása a következőt írja ki:

```
Script executed successfully.
Document saved.
```

És a `modified.html` most valami ilyesmit tartalmaz:

```html
<!DOCTYPE html>
<html>
<head>
    <title>Template</title>
</head>
<body>
    <h1>Original Content</h1>
    <p>Generated by JavaScript</p>
</body>
</html>
```

## Gyakori kérdések és buktatók

| Kérdés | Válasz |
|----------|--------|
| **Futtathatok több szkriptet egymás után?** | Igen. Csak hívd újra a `scriptEngine.execute()`‑t egy új szkriptstringgel. Az ugyanaz a `HTMLDocument` példány megtartja az állapotát. |
| **Mi van, ha a HTML fájl nem tartalmaz `<body>` elemet?** | Az Aspose.HTML automatikusan létrehozza a `<body>` elemet a feldolgozás során, így a szkript továbbra is működik. Azonban a kiszámítható eredmény érdekében érdemes jól formált sablonnal kezdeni. |
| **Ez a megközelítés kompatibilis a Java 17‑tel?** | Teljesen. A könyvtár Java 8+ célplatformra készült, így az újabb futtatókörnyezetek is támogatottak. |
| **Szükségem van teljes böngésző környezetre?** | Nem. Az Aspose.HTML egy könnyű, fej nélküli motor, amely a szabványos DOM API‑kat értelmezi anélkül, hogy Chrome‑t vagy Firefox‑ot indítana. |
| **Hogyan kezelem a nagy HTML fájlokat?** | Töltsd be a dokumentumot stream‑ként (`new HTMLDocument(InputStream)`) a teljes fájl egyszerre történő memóriába olvasása helyett. |

## Következő lépések

Miután megtanultad, hogyan **Java‑ban JavaScript‑et futtass**, tovább mélyítheted a **JavaScript DOM manipuláció** ismereteidet:

- **CSS injektálása** az újonnan hozzáadott elemek stílusozásához.
- **DOM bejárása** a meglévő csomópontok attribútumok alapján történő módosításához.
- **Külső szkriptek** futtatása URL‑ről (használd a `scriptEngine.executeFromUrl()`‑t, ha szükséges).
- Kombináld ezt a technikát **PDF konverzióval** – az Aspose.HTML egyetlen hívással képes a végleges DOM‑ot PDF‑be renderelni.

Ha érdekel a **Java‑ból JavaScript futtatása** tesztelés vagy szerver‑oldali renderelés céljából, fontold meg a logika egy segédosztályba való becsomagolását, és használd újra mikro‑szolgáltatások között.
